# 民心与稳定性系统

> 本文档整合《山河策》所有民心与稳定性机制细节，作为程序实现的唯一参考源。
> 所有数值参数从 `data/*.json` 读取，禁止硬编码。

---

## 1. 设计概述

双层制设计：国家级民心（0~100）影响税收/征兵/战斗力，城市级安定度（0~100）影响产出/叛乱。民心受税率/季节/战争/学派等宏观因素影响，安定度受建筑/驻军/文化渗透等微观因素影响。

**设计原则**：

1. **双层分离**：民心管宏观（税率、季节、战争），安定度管微观（建筑、驻军、文化）
2. **阈值驱动**：民心和安定度各设 4 档阈值，每档产生不同效果
3. **传导机制**：民心 < 30 时传导至安定度（-3/回合），形成连锁反应
4. **数据驱动**：所有数值从 `balance_params.json` 读取

## 2. 国家民心系统

### 2.1 民心范围与初始值

| 属性 | 值 | 来源 |
|:---|:---:|:---|
| 范围 | 0~100 | — |
| 初始值 | 50 | `balance_params.json` → `game_settings.starting_morale` |

### 2.2 民心每回合计算公式

```
// 民心变化 = 各项修正之和
morale_delta = base_drift                  // 基础漂移（向 50 回归）
              + tax_morale_effect          // 税率影响（5 档固定值）
              + season_morale_mod          // 季节修正
              + school_base_morale         // 学派基础修正（儒家 +2 / 法家 -1）
              + wonder_morale_buff         // 奇观民心加成
              + building_morale_buff       // 建筑民心加成（烽火台）
              + tech_morale_bonus          // 科技民心加成
              + event_morale_effect        // 事件民心效果
              + war_weariness              // 厌战情绪
              + victory_bonus              // 胜利激励

// 应用变化
morale = clamp(morale + morale_delta, 0, 100)
```

| 变量 | 说明 | 来源 |
|:---|:---|:---|
| `base_drift` | 基础漂移（每回合 +1，向 50 回归） | `balance_params.json` → `morale.base_drift` |
| `tax_morale_effect` | 税率影响（更改税率时一次性变化） | `balance_params.json` → `morale.tax_morale_values` |
| `season_morale_mod` | 季节修正 | `balance_params.json` → `morale.season_morale_mod` |
| `school_base_morale` | 学派基础修正 | `balance_params.json` → `morale.school_base_morale` |
| `wonder_morale_buff` | 奇观民心加成 | `wonders.json` → `morale_national` |

| 高 | > 80 | +20% | +30% | 暴动概率 0%，所有城市安定度 +1/回合 | `morale.threshold_high` |
| 中 | 50~80 | 正常 | 正常 | 无 | `morale.threshold_mid` |
| 低 | 30~50 | -20% | -20% | 无 | `morale.threshold_low` |
| 危机 | < 30 | -50% | -50% | 逃兵 +10%/回合，每回合安定度 -3 | — |

### 3.1 民心战斗修正

民心影响部队攻击力（决策 #74）：

```
morale_atk_mod = 0.7 + 0.3 × (民心 / 100)
```

- 0 民心 = 70% 战力
- 100 民心 = 100% 战力

### 4.2 安定度每回合计算公式

```
stability += building_bonus              // 建筑加成
           + garrison_bonus              // 驻军加成

           + city_captured               // 城市被占领惩罚
```

## 5. 安定度阈值效果

安定度分为 4 档，影响城市产出和叛乱判定（决策 #76）：

| 档位 | 安定度范围 | 产出偏移 | 特殊效果 | 来源 |
|:---|:---:|:---:|:---|:---|
| 高 | ≥ 80 | +20% | 无 | `stability.threshold_high` / `high_output_bonus` |
| 中 | 50~79 | 0% | 无 | `stability.threshold_mid` / `mid_output_penalty` |
| 低 | 30~49 | -20% | 15%/回合民变（城市变为中立） | `stability.threshold_low` / `low_output_penalty` |
| 危机 | < 10 | -50% | 30%/回合起义（城市变为中立） | `stability.threshold_critical` / `critical_output_penalty` |

### 5.1 安定度对产出的影响

安定度通过偏移量影响城市所有资源产出：

```
output = base_output × (1 + stability_output_offset)
```

| 安定度档位 | 偏移量 |
|:---|:---:|
| 高（≥ 80） | +0.20 |
| 中（50~79） | 0.00 |
| 低（30~49） | -0.20 |
| 危机（< 10） | -0.50 |

### 5.2 叛乱/起义判定

低安定度触发叛乱和起义风险（决策 #75）。叛乱和起义的结果均为**城市变为中立城市**（`neutral`），建筑保留。

| 条件 | 概率 | 效果 | 来源 |
|:---|:---:|:---|:---|
| 安定度 < 30（低） | 15%/回合 | 叛乱：城市变为中立 | `balance_params.json` → `stability.low_revolt_chance` |
| 安定度 < 10（危机） | 30%/回合 | 起义：城市变为中立 | `balance_params.json` → `stability.critical_uprising_chance` |

## 6. 民心与安定度的传导机制

### 6.1 民心传导安定度

| 条件 | 效果 | 来源 |
|:---|:---:|:---|
| 民心 < 30 | 所有城市安定度 -3/回合 | `balance_params.json` → `morale.low_stability_penalty` |
| 民心 > 80 | 所有城市安定度 +1/回合 | `balance_params.json` → `morale.high_stability_bonus` |

**设计意图**：民心是宏观指标，低民心会传导至所有城市安定度，形成连锁反应。高民心给予微量奖励，鼓励维持稳定治理。

### 6.2 文化不匹配传导

当城市主流文化与所属势力文化不一致时（详见城市经营系统.md §6.3）：

| 效果 | 值 | 来源 |
|:---|:---:|:---|
| 安定度惩罚 | -15 | `balance_params.json` → `stability.culture_mismatch_penalty` |
| 驻守部队攻击 | -20%（加法层） | `balance_params.json` → `culture.culture_mismatch_garrison_atk_penalty` |
| 驻守部队防御 | -20%（加法层） | `balance_params.json` → `culture.culture_mismatch_garrison_def_penalty` |

首都主流文化不一致时，额外触发全国民心 -10。
