# 情报力系统

> 本文档整合《山河策》视野与情报系统的所有机制细节，作为程序实现的参考源。
> 所有数值参数从 `data/*.json` 读取，禁止硬编码。
> 基于决策 #86~#89，采用点数池制 + 三层信息可见性模型。归属于外交模块。

---

## 1. 设计概述

情报力系统控制玩家对其他国家的信息可见性，涵盖两个维度：**战术视野**（地图上能看到什么）和**战略信息**（信息面板中能看到多少详情）。两者共同构成三层信息可见性模型。

**核心设计原则**：

1. **分层可见**：地理视野决定地图格子可见性，外交情报力决定信息详度
2. **取最有利**：单位信息可见性取外交状态与情报力等级中最有利的
3. **每国独立**：对每个国家的情报力独立计算
4. **被动势力豁免**：中立国和周天子自动获得完整信息

---

## 2. 三层信息可见性模型

```
┌──────────────────────────────────────────────┐
│ 第三层：单位信息详度                           │
│   取外交状态与情报力等级中最有利的             │
│   → §6 单位信息可见性                         │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│ 第二层：外交情报力（城市/国力信息）             │
│   情报力等级决定能看到多少国力详情             │
│   情报力直接在战术地图上开城市视野             │
│   → §4 情报等级 / §5 战术视野 / §7 战略信息   │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│ 第一层：地理视野（地图格子可见性）              │
│   单位视野 + 建筑视野 + 情报力开的城市视野     │
│   → 详见《地理与天堑系统.md》§10              │
└──────────────────────────────────────────────┘
```

**交互规则**：
- 地理视野决定能否在战术地图上看到某个格子（单位/建筑在附近 → 可见）
- 外交情报力决定两个事情：
  1. 能在信息面板中看到目标国家的多少详情（驻军/资源/大夫等）
  2. 直接在战术地图上开启目标城市的视野（不需要单位在旁边）
- 单位信息可见性取两者中最有利的（详见 §6）

## 3. 情报力点数计算

### 3.1 公式

```
intelligence_points = opinion + sum(modifiers)
```

- 好感度（-100 ~ +100）1:1 作为基础点数
- 修正来源直接加点数，无上限
- 宣战时点数强制归零

### 3.2 修正来源

| 来源 | 点数 | 说明 |
|:---|:---|:---|
| 驿站 | +15 | 已有 `intelligence_range` 定义 |
| 驿馆 | +30 | 需纵横家学派 1 级，已有 `intel_range` 定义 |
| 商路存在 | +10 | 商路是信息通道 |
| 送质子 | +10 | 质子在对方国内 |
| 外交大夫驻扎 | +10 | 大夫在与该国接壤的边境城市时 |
| 宣战 | 强制归零 | 战争迷雾 |
| 突袭宣战停战后 | -10 持续 10 回合 | 对方加强防间谍 |

数据来源：`diplomacy.json` → `intelligence`

### 3.3 叠加规则

所有修正直接加点数，无上限。驿站与驿馆效果可叠加。

## 4. 情报等级

### 4.1 等级定义

> 基于决策 #99，从 4 级扩展为 5 级，与外交状态五级统一。

| 等级 | 点数阈值 | 名称 |
|:---|:---|:---|
| 0 | 0 ~ 19 | 断绝 |
| 1 | 20 ~ 39 | 线索 |
| 2 | 40 ~ 59 | 基础 |
| 3 | 60 ~ 79 | 详细 |
| 4 | ≥ 80 | 完整 |

### 4.2 等级效果总览

| 等级 | 战术视野 | 战略信息 | 单位信息 |
|:---|:---|:---|:---|
| 0 断绝 | 无城市视野 | 国名、分数 | 仅基线（兵种名、血量状态） |
| 1 线索 | 1 座最近城市（仅城市本身 1 格） | 城市数量、声望等级 | 同上 |
| 2 基础 | 接壤城市正常视野 + 首都视野 -1 | 战争/结盟状态、各城驻军数量 | 攻防（与外交状态并集） |
| 3 详细 | 所有与你接壤的城市正常视野 | 资源模糊区间、科技数量、附庸关系 | 移动力、大夫名字（与外交状态并集） |
| 4 完整 | 所有城市正常视野 | 各城具体资源、大夫列表及属性、具体科技名、盟友关系 | 大夫属性（与外交状态并集） |

## 5. 战术视野机制

情报力直接在战术地图上开启目标城市的视野，不需要单位在旁边。

### 5.1 各等级视野效果

- **断绝（0）**：无法看到对方任何城市
- **线索（1）**：1 座城市（取距离你最近的对方城市），仅看到城市本身（1 格），不含周围格子
- **基础（2）**：对方所有与你接壤的城市（距离 ≤ 4 格）获得正常 `city_vision_radius` 视野；首都视野半径 -1，最少看到城市本身
- **详细（3）**：对方所有与你接壤的城市正常视野，首都正常视野
- **完整（4）**：对方所有城市获得正常 `city_vision_radius` 视野

### 5.2 与地理视野的关系

情报力开启的城市视野与单位/建筑的地理视野**叠加**：
- 如果地理视野已经覆盖某城市，情报力不额外提供新格子
- 如果地理视野未覆盖但情报力足够，情报力直接在地图上显示该城市区域
- 两种视野来源的显示效果相同（都是在战术地图上消除迷雾）

### 5.3 "接壤城市"定义

接壤城市 = 与你的城市距离 ≤ 4 格的对方城市。

### 5.4 首都视野下限

首都视野半径 = `city_vision_radius` - 1，最少为 1（即至少看到城市本身）。

数据来源：`balance_params.json` → `vision.city_vision_radius`（当前值：1）

## 6. 单位信息可见性

> 基于决策 #89、#95、#99。情报力系统从 4 级扩展为 5 级，与外交状态统一。

**前提条件**：所有单位信息可见性以**视野覆盖**为前提。不在视野范围内的单位无法查看任何信息。

### 6.1 信息类型与基线

单位信息共 6 种类型，其中 2 种始终可见（基线），4 种由外交状态和情报力共同决定。

| 信息类型 | 基线可见性 | 说明 |
|:---|:---|:---|
| 兵种名 | 始终可见 | 只要在视野内就能看到 |
| 血量状态 | 始终可见 | 血条显示，但精确数值需更高权限 |
| 攻防 | 需解锁 | 攻击力/防御力具体数值 |
| 移动力 | 需解锁 | 移动点数 |
| 大夫名字 | 需解锁 | 指挥单位的大夫姓名 |
| 大夫属性 | 需解锁 | 大夫的统御/武力/智力/政治属性 |
| 大夫技能 | 需解锁 | 大夫的主动/被动技能 |

### 6.2 外交状态贡献的信息

> 基于决策 #95 五级外交状态体系。

| 外交状态 | 独占信息 | 与情报力重叠 |
|:---|:---|:---|
| 交战 | 大夫名字 | — |
| 不友好 | — | — |
| 中立 | — | 攻防 |
| 友好 | 大夫属性 | 移动力 |
| 结盟 | 大夫技能 | — |

**说明**：
- 交战状态可通过战争接触获知对方领军大夫名字
- 不友好状态无额外信息来源
- 中立状态通过贸易/外交接触了解对方军事实力（攻防）
- 友好状态深入交流可了解大夫能力和部队机动性
- 结盟状态情报共享，可获知大夫完整能力

### 6.3 情报力贡献的信息

> 基于决策 #99，情报力扩展为 5 级。

| 情报力等级 | 独占信息 | 与外交状态重叠 |
|:---|:---|:---|
| 0 断绝 | — | — |
| 1 线索 | — | — |
| 2 基础 | — | 攻防 |
| 3 详细 | — | 移动力、大夫名字 |
| 4 完整 | — | 大夫属性 |

**说明**：
- 情报力 0~1 级无法通过间谍获取额外信息
- 情报力 2 级可通过间谍了解敌方部队攻防数据
- 情报力 3 级可渗透获取部队机动性和领军大夫身份
- 情报力 4 级可深入获取大夫个人能力情报

### 6.4 并集合并规则

最终可见信息 = 外交状态贡献的信息 ∪ 情报力贡献的信息。

**示例**：
- 交战状态（大夫名字）+ 情报力 0（无）→ 兵种名、血量状态、大夫名字
- 中立状态（攻防）+ 情报力 2（攻防）→ 兵种名、血量状态、攻防
- 友好状态（大夫属性、移动力）+ 情报力 2（攻防）→ 兵种名、血量状态、攻防、移动力、大夫属性
- 结盟状态（全部）+ 情报力 0（无）→ 全部信息（外交状态独占补全）
- 不友好状态（无）+ 情报力 4（攻防、移动力、大夫名字、大夫属性）→ 兵种名、血量状态、攻防、移动力、大夫名字、大夫属性

### 6.5 大夫技能的特殊性

大夫技能**仅外交状态为结盟时可见**，情报力系统无法获取此信息。这体现了"技能"作为大夫核心机密的特殊地位——只有真正的盟友才能完全了解。

## 7. 战略信息机制

战略信息通过外交面板/情报界面展示，不在战术地图上。

### 7.1 各等级可见信息

| 等级 | 可见信息 | 精度 |
|:---|:---|:---|
| 0 断绝 | 国名、分数 | 精确 |
| 1 线索 | 城市数量、声望等级 | 精确 |
| 2 基础 | 是否在战争、是否结盟、各城驻军数量 | 精确 |
| 3 详细 | 资源模糊区间、科技数量、附庸关系 | 模糊 |
| 4 完整 | 各城具体资源数值、大夫列表及属性、具体科技名、盟友关系 | 精确 |

### 7.2 模糊区间定义

| 资源状态 | 区间 |
|:---|:---|
| 匮乏 | < 30% |
| 一般 | 30% ~ 70% |
| 充裕 | > 70% |

### 7.3 完整情报限制

完整情报（等级 4）仅显示盟友关系和战争状态，不显示具体条约类型和剩余回合。

---

## 8. 被动势力规则

被动势力不适用情报力系统，自动获得完整信息。

### 8.1 中立国

- 非交战、非结盟的普通国家视为中立国
- 对中立国自动获得完整情报（等级 4）
- 中立国不参与情报力点数计算

### 8.2 周天子（zhou）

- 周天子为被动势力（`factions.json` → `zhou.is_passive: true`）
- 对周天子自动获得完整情报
- 洛邑始终在地图上可见
- 周天子不参与情报力点数计算

---

## 9. 战争恢复机制

| 场景 | 恢复方式 |
|:---|:---|
| 正常宣战 → 停战 | 跟随好感度自然恢复 |
| 突袭宣战 → 停战 | -10 点持续 10 回合，之后跟随好感度恢复 |

宣战时情报力强制归零（战争迷雾）。停战后按上述规则恢复。

---

## 10. 跨系统关联

### 10.1 外交 × 情报力

- 好感度是情报力的基础来源（1:1 点数）
- 宣战自动将情报力归零
- 突袭宣战停战后 -10 点持续 10 回合
- 商路、质子、外交大夫提供情报力加成

### 10.2 建筑 × 情报力

- 驿站：+15 情报力点数
- 驿馆：+30 情报力点数（需纵横家学派 1 级）
- 两者可叠加

### 10.3 地理视野 × 情报力

- 地理视野（单位/建筑）和情报力是独立的视野来源
- 两者在战术地图上叠加显示
- 情报力开启的城市视野与地理视野效果相同

### 10.4 学派 × 情报力

- 纵横家学派 1 级解锁驿馆情报效果（+30 点数）
