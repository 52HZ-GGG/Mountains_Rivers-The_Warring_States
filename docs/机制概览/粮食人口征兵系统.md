> 本文档整合《山河策》所有粮食、人口、征兵机制细节，作为程序实现的唯一参考源。
> 所有数值参数从 `data/*.json` 读取，禁止硬编码。

## 1. 设计概述

以 1 为基本单位的城市人口系统。每人口产出粮食和金钱，自耗粮食，剩余按税率充公至全国储粮。人口增长采用进度条制，受粮食盈余、季节、安定度调节。征兵从总人口中抽取，服役比例过高会惩罚经济和增长。

**核心设计原则**：

1. **人均产出模型**：粮食和金钱以城市人口为基础产出，人口是经济核心驱动力
2. **税收充公制**：城市产出的粮食和金钱按税率充公至国家级资源池
3. **进度条增长**：人口增长不直接计算，而是累积进度条，满 100% 时人口 +1
4. **征兵约束**：征兵受总人口比例约束，服役过高惩罚经济
5. **饥荒双层制**：城市级饥荒（人口+安定度惩罚）和国家级储粮不足（民心+军队惩罚）
6. **数据驱动**：所有数值从 `balance_params.json` / `units.json` / `buildings.json` / `cities.json` 读取

## 2. 粮食系统

### 2.1 粮食产出

城市粮食产出以人口为基础，受多维修正影响。

```
// 基础产出
food_base = population × 3    // balance_params.json → population.food_per_pop

// 百分比偏移叠加（所有 buff 相加后统一乘算）
food_offset = season_food_mod + farm_bonus + special_resource_mod + tech_mod + stability_offset

// 产出上限
food_gross = min(food_base × (1 + food_offset), max_food_production)

// 净产出（扣除自耗）
food_net = food_gross - population × 1    // balance_params.json → population.food_consume_per_pop
```

| 变量 | 说明 | 值 | 来源 |
|:---|:---|:---:|:---|
| food_per_pop | 人均粮食产出 | 3 | `balance_params.json` → `population.food_per_pop` |
| food_consume_per_pop | 人均粮食自耗 | 1 | `balance_params.json` → `population.food_consume_per_pop` |
| season_food_mod | 季节偏移 | 春+0.2/夏0/秋+1.0/冬-0.3 | `balance_params.json` → `resources.season_food_mod` |
| farm_bonus | 农田加成 | +10%/级 | `buildings.json` → `farm.effects.food_production_bonus` |
| stability_offset | 安定度偏移 | 高+0.2/中0/低-0.2/危机-0.5 | `balance_params.json` → `stability.*_output_*` |
| max_food_production | 最大粮食产出 | 等级决定 + 粮仓加成 | `balance_params.json` → `city_levels.*.max_food_production` + `granary.effects.max_food_production` |

**数据来源**：`balance_params.json` → `population`，`buildings.json` → `farm` / `granary`

### 2.2 粮食消耗

| 消耗场景 | 公式 | 来源 |
|:---|:---|:---|
| 城市人口自耗 | population × 1/回合 | `balance_params.json` → `population.food_consume_per_pop` |
| 军队粮食维护 | Σ(各兵种 upkeep_food) | `units.json` → `upkeep_food` |
| 征兵消耗 | 按兵种 cost_food | `units.json` → `cost_food` |

### 2.3 全国储粮

```
// 税收效率 = 基础效率 + 民心档位修正 + 腐败修正
tax_efficiency = base_efficiency + morale_tax_modifier + corruption_tax_effect

// 储粮变化
national_grain += Σ(food_net × tax_rate × tax_efficiency) - Σ(army_upkeep_food)

// 储粮上限
national_grain_cap = 200 + Σ(granary_level × 50)

// 超出上限不累积
national_grain = min(national_grain, national_grain_cap)
```

| 变量 | 说明 | 值 | 来源 |
|:---|:---|:---:|:---|
| base_efficiency | 基础税收效率 | 1.0 | `balance_params.json` → `tax.current_efficiency` |
| morale_tax_modifier | 民心档位修正 | 高+0.2/中0/低-0.2/危机-0.5 | `balance_params.json` → `morale.high_tax_bonus` / `low_tax_penalty` / `critical_tax_penalty` |
| corruption_tax_effect | 腐败修正 | 清廉+10%/正常0/腐败-6%/严重腐败-10%（线性插值） | `balance_params.json` → `corruption`，详见经营系统.md §9.2 |
| national_grain_cap_base | 储粮基础上限 | 200 | `balance_params.json` → `population.national_grain_cap_base` |
| granary_grain_cap_bonus | 粮仓加成/级 | +50 | `balance_params.json` → `population.granary_grain_cap_bonus` |
| tax_rate | 税率 | 10%~50% | `balance_params.json` → `tax` |

**数据来源**：`balance_params.json` → `population` / `tax`，`buildings.json` → `granary`

## 3. 人口系统

### 3.1 人口产出模型

每人口产出金钱（无上限），产出粮食（有上限）。

| 项目 | 基础值 | 修正项 | 来源 |
|:---|:---:|:---|:---|
| 粮食产出 | 3/人/回合 | ×(1 + 季节 + 农田 + 特产 + 科技 + 安定度)，上限 max_food_production | `balance_params.json` → `population.food_per_pop` |
| 金钱产出 | 10/人/回合 | ×(1 + 季节 + 市集 + 特产 + 科技 + 安定度)，无上限 | `balance_params.json` → `population.gold_per_pop` |
| 粮食自耗 | 1/人/回合 | 固定 | `balance_params.json` → `population.food_consume_per_pop` |

**详细产出公式**：参见城市经营系统.md §4.1。

### 3.2 人口增长（进度条制）

人口增长采用进度条制，每回合推进度，满 100% 时人口 +1（决策 #92）。

```
// 人口增长周期（回合/人）
growth_period = growth_base_period / food_surplus_factor / season_mod / stability_mod

// 粮食盈余因子
food_surplus_factor = food_surplus_per_pop / 2.0

// 每回合进度增量
growth_progress += 100.0 / growth_period

// 进度满 100 时
if growth_progress >= 100.0:
    population += 1
    growth_progress -= 100.0
```

| 修正项 | 公式 | 值域 | 来源 |
|:---|:---|:---:|:---|
| growth_base_period | 基础增长周期 | 6.67 | `balance_params.json` → `population.growth_base_period` |
| 粮食盈余 | surplus_per_pop / 2.0 | 0.5~2.0+ | 盈余 2→1.0x，盈余 3→1.5x |
| 季节 | spring 1.0 / summer 1.0 / autumn 1.2 / winter 0.7 | 0.7~1.2 | `balance_params.json` → `population.growth_season_mod` |
| 安定度 | 0.5 + stability / 100.0 | 0.5~1.5 | `balance_params.json` → `population.growth_stability_formula` |

**示例**：人口 10，粮食净产出 25，盈余 2.5/人，秋季，安定度 80：
- food_surplus_factor = 2.5 / 2.0 = 1.25
- season_mod = 1.2
- stability_mod = 0.5 + 80/100 = 1.3
- growth_period = 6.67 / 1.25 / 1.2 / 1.3 = 3.42 回合
- 每回合进度 = 100 / 3.42 = 29.2%

**数据来源**：`balance_params.json` → `population`

### 3.3 人口上限

每座城市有人口上限，由城市等级决定：

| 等级 | 最大人口 | 来源 |
|:---:|:---:|:---|
| 1（村） | 10 | `balance_params.json` → `city_levels.1.max_population` |
| 2（镇） | 15 | `balance_params.json` → `city_levels.2.max_population` |
| 3（城） | 20 | `balance_params.json` → `city_levels.3.max_population` |
| 4（府） | 25 | `balance_params.json` → `city_levels.4.max_population` |
| 5（都） | 30 | `balance_params.json` → `city_levels.5.max_population` |

人口达到上限后停止增长。

## 4. 税率系统

### 4.1 税率档位

税率决定城市产出的粮食和金钱充公至国家级资源池的比例（决策 #91）。

| 税率 | 民心影响 | 来源 |
|:---:|:---|:---|
| 10% | +15 | `balance_params.json` → `morale.tax_morale_values.0.1` |
| 20% | +5 | `balance_params.json` → `morale.tax_morale_values.0.2` |
| 30% | 0（默认） | `balance_params.json` → `morale.tax_morale_values.0.3` |
| 40% | -10 | `balance_params.json` → `morale.tax_morale_values.0.4` |
| 50% | -20 | `balance_params.json` → `morale.tax_morale_values.0.5` |

| 参数 | 值 | 来源 |
|:---|:---:|:---|
| 最低税率 | 10% | `balance_params.json` → `tax.min_rate` |
| 最高税率 | 50% | `balance_params.json` → `tax.max_rate` |
| 默认税率 | 30% | `balance_params.json` → `tax.default_rate` |
| 变更冷却 | 10 回合 | `balance_params.json` → `morale.tax_change_cooldown` |

**设计意图**：税率对民心的影响为**更改税率时一次性变化**，而非每回合累加。

### 4.2 税收效率

```
// 税收效率 = 基础效率 + 民心档位修正 + 腐败修正
tax_efficiency = base_efficiency + morale_tax_modifier + corruption_tax_effect

// 最终税收
food_tax = food_net × tax_rate × tax_efficiency
gold_tax = gold_net × tax_rate × tax_efficiency
```

- `tax_efficiency` 默认 1.0（`balance_params.json` → `tax.current_efficiency`）
- 税收效率受民心档位、腐败等级、科技、文大夫理财%、王宫税收加成影响

**税收效率修正来源**：
- 民心档位（高 +0.2 / 中 0 / 低 -0.2 / 危机 -0.5）
- 腐败等级（清廉最多 +0.10 / 正常 0 / 腐败最多 -0.06 / 严重腐败最多 -0.10，详见经营系统.md §9.2）
- 科技效果、文大夫理财%、王宫税收加成

### 4.3 民心对税收效率的影响

民心档位直接影响税收效率（`tax_efficiency`），民心越高税收效率越高，民心越低税收效率越低（决策 #73）。

| 民心档位 | 税收效率修正 | 来源 |
|:---|:---:|:---|
| 高（≥ 80） | +0.2 | `balance_params.json` → `morale.high_tax_bonus` |
| 中（50~79） | 0 | — |
| 低（30~49） | -0.2 | `balance_params.json` → `morale.low_tax_penalty` |
| 危机（< 30） | -0.5 | `balance_params.json` → `morale.critical_tax_penalty` |

**示例**：默认税收效率 1.0，民心 85（高档位），腐败值 60（腐败档）：
- tax_efficiency = 1.0 + 0.2 + (-0.02 × (60 - 50)) = 1.0 + 0.2 - 0.2 = 1.0
- 粮食税 = food_net × 0.3 × 1.0

**数据来源**：`balance_params.json` → `morale`（high_tax_bonus / low_tax_penalty / critical_tax_penalty）、`tax.current_efficiency`、`corruption`

## 6. 征兵系统

### 6.1 征兵池

征兵从全国总人口中抽取，受容量约束（决策 #94）。

| 概念 | 公式 | 来源 |
|:---|:---|:---|
| 最大征召人数 | 全国总人口 × 20% | `balance_params.json` → `population.conscription_rate` |
| 填充速率 | 最大征召 × 10%/回合 | `balance_params.json` → `population.conscription_fill_rate` |
| 约束 | 可服役 + 已服役 ≤ 最大征召 | — |

```
// 征兵池计算
max_cons = total_population × 0.20
available += max_cons × 0.10    // 每回合填充（开启征兵时）
available + active_units ≤ max_cons    // 硬约束

// 征兵消耗（按兵种）
cost_gold -= unit.cost_gold
cost_food -= unit.cost_food
cost_craftsmen -= unit.cost_craftsmen

// 达到约束上限自动停止
```

### 6.2 征兵衰减与恢复

阵亡导致征兵池永久缩减，缓慢恢复（决策 #95）：

| 事件 | 效果 | 来源 |
|:---|:---|:---|
| 阵亡 1 人 | 最大征召 -1 | — |
| 恢复 | 每 4 回合恢复 1 点 | `balance_params.json` → `population.death_recovery_period` |

### 6.3 服役惩罚

服役比例过高会惩罚经济和人口增长（决策 #96）：

```
if active / total_pop > service_penalty_threshold:    // 10%
    penalty = active / total_pop - 0.10
    food_output  *= (1 - penalty × service_penalty_food_mod)    // ×2.0
    gold_output  *= (1 - penalty × service_penalty_gold_mod)    // ×2.0
    growth_speed *= (1 - penalty × service_penalty_growth_mod)  // ×5.0
```

| 参数 | 值 | 来源 |
|:---|:---:|:---|
| 惩罚阈值 | 10% | `balance_params.json` → `population.service_penalty_threshold` |
| 粮食惩罚倍率 | ×2.0 | `balance_params.json` → `population.service_penalty_food_mod` |
| 金钱惩罚倍率 | ×2.0 | `balance_params.json` → `population.service_penalty_gold_mod` |
| 增长惩罚倍率 | ×5.0 | `balance_params.json` → `population.service_penalty_growth_mod` |

**示例**：总人口 100，已服役 15 人：
- 服役比例 = 15%
- 超过阈值 = 15% - 10% = 5%
- 粮食产出 ×(1 - 0.05×2) = ×0.90（-10%）
- 金钱产出 ×(1 - 0.05×2) = ×0.90（-10%）
- 人口增长 ×(1 - 0.05×5) = ×0.75（-25%）

## 8. 公式汇总

```
# ===== 城市产出 =====
food_base = population × 3
food_offset = season_food + farm_bonus + special_mod + tech_mod + stability_offset
food_gross = min(food_base × (1 + food_offset), max_food_production)
food_net = food_gross - population × 1

gold_base = population × 10
gold_offset = season_gold + market_bonus + special_mod + tech_mod + stability_offset
gold_net = gold_base × (1 + gold_offset)

# ===== 税收 =====
tax_efficiency = base_efficiency + morale_tax_modifier + corruption_tax_effect    // 民心高档+0.2, 中0, 低-0.2, 危机-0.5; 腐败修正见经营系统§9.2
food_tax = food_net × tax_rate × tax_efficiency
gold_tax = gold_net × tax_rate × tax_efficiency

# ===== 全国储粮 =====
national_grain += Σ(food_tax) - Σ(army_upkeep_food)
national_grain_cap = 200 + Σ(granary_level × 50)
national_grain = min(national_grain, national_grain_cap)

# ===== 人口增长 =====
growth_period = 6.67 / (food_surplus_per_pop / 2.0) / season_mod / stability_mod
growth_progress += 100 / growth_period
if growth_progress >= 100: population += 1, progress -= 100

# ===== 征兵 =====
max_cons = total_pop × 0.20
available += max_cons × 0.10 (每回合)
available + active ≤ max_cons

# ===== 服役惩罚 =====
if active / total_pop > 0.10:
    penalty = active / total_pop - 0.10
    food  *= (1 - penalty × 2.0)
    gold  *= (1 - penalty × 2.0)
    growth *= (1 - penalty × 5.0)

# ===== 军队维护 =====
army_food = Σ(unit.upkeep_food × count)
army_gold = Σ(unit.upkeep_gold × count)

# ===== 饥荒 =====
if food_gross < population:   # 城市级
    population -= 1, stability -= 10
if national_grain ≤ 0:        # 国家级
    morale -= 5, army_morale -= 10, army_atk × 0.8, army_def × 0.8
```
