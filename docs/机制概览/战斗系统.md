# 战斗系统

> 本文档整合《山河策》所有战斗机制细节，作为程序实现的唯一参考源。
> 所有数值参数从 `data/*.json` 读取，禁止硬编码。

---

## 1. 设计概述

回合制战斗，基于攻击力/防御力/地形/兵种克制/大夫加成的多维权重伤害计算。

**核心公式原则**：

1. 所有百分比 buff **相加**后统一乘算基础值
2. 克制关系在加法之后**单独乘算**
3. 扣除防御，保底伤害 1
4. 随机波动（受远程精度修正影响）
5. 崩溃态（士气 < 20）的攻防修正作为**独立乘算**，不参与加法层
6. 水军/海军战斗修正作为**最终乘算系数**，在所有计算之后应用

**程序实现状态**：`scripts/systems/combat_resolver.gd`（阶段 1 MVP）

## 2. 伤害公式

### 2.1 攻击方有效攻击力（公式 1.1）

所有百分比 buff 相加后统一乘算基础攻击力（克制关系单独乘算，不参与加法）。

```
atk_buff = 1.0
    + (terrain.atk_mod - 1.0)          // 地形修正偏移量
    + minister_bravery_pct              // 大夫勇武（如 50 -> +0.5）
    + faction_attack_bonus              // 国家攻击加成
    + morale_atk_offset                 // 单位士气修正（见第 6 节）
    + ambush_bonus                      // 伏击触发时 +0.3（与火攻互斥）
    + fire_attack_bonus                 // 火攻触发时 +0.4（与伏击互斥）
    + school_attack_bonus               // 学派攻击加成（如兵家）
    + tech_attack_bonus                 // 科技攻击加成
    + unit_ability_bonus                // 兵种特技（动能攻击等）

effective_atk = unit.attack × atk_buff
```

**崩溃态独立乘算**：若单位士气 < 20（崩溃），最终攻击力额外 ×0.5（不参与加法层）。

**数据来源**：`balance_params.json` → `combat.morale_atk_floor/ceil`，`terrain.json` → `atk_mod`

### 2.2 防御方有效防御力（公式 1.2）

```
def_buff = 1.0
    + (terrain.def_mod - 1.0)          // 地形修正偏移量
    + minister_strategy_pct             // 大夫韬略（如 40 -> +0.4）
    + faction_defense_bonus             // 国家防御加成
    + school_defense_bonus              // 学派防御加成（墨家 +0.25）
    + tech_defense_bonus                // 科技防御加成
    + building_defense_bonus            // 城墙防御加成（每级 +0.15）
    + flanking_penalty                  // 被夹击时 -0.3

effective_def = unit.defense × def_buff
```

**崩溃态独立乘算**：若单位士气 < 20（崩溃），最终防御力额外 ×0.5（不参与加法层）。

**数据来源**：`terrain.json` → `def_mod`，`buildings.json` → `wall.effects.defense_bonus`

### 2.3 净伤害计算（公式 1.3）

```
// 1. 克制乘算（在加法 buff 之后单独乘算）
counter_dmg = effective_atk × counter_multiplier

// 2. 扣除防御，保底 1
raw_dmg = max(counter_dmg - effective_def, 1)

// 3. 随机波动（受远程精度修正影响）
spread_lo = base_random_spread_lo + terrain.ranged_accuracy_mod × accuracy_spread_factor
spread_hi = base_random_spread_hi + terrain.ranged_accuracy_mod × accuracy_spread_factor
final_dmg = floor(raw_dmg × random(spread_lo, spread_hi))

// 4. 水军/海军最终乘算系数（如适用）
final_dmg = floor(final_dmg × naval_modifier)
```

**数据来源**：`balance_params.json` → `combat.base_random_spread_lo/hi`、`combat.accuracy_spread_factor`

### 2.4 反击机制

当近战单位 A 攻击单位 B 时，B 会对 A 造成反击伤害：

```
反击伤害 = B 的 effective_atk × counter_multiplier × 随机波动
```

- 反击伤害 **100%**，与主动攻击完全相同
- 反击同样适用克制矩阵、地形修正、士气修正等所有加法层和乘算层
- **不触发反击的攻击**：城市主动攻击、箭塔攻击、远程单位主动远程攻击

**数据来源**：`units.json` → `ranged_attack_mode: "no_counter"`（远程兵种标记）

### 2.5 当前代码实现（阶段 1 MVP）

`combat_resolver.gd` 的 `compute_damage()` 实际流程：

```
1. base_dmg = unit.attack
2. base_dmg ×= atk_general_mod(固定1.0) × terrain_atk_mod × atk_faction_mod(固定1.0) × morale_atk_mod
3. base_def = unit.defense
4. base_def ×= def_general_mod(固定1.0) × terrain_def_mod × def_faction_mod(固定1.0)
5. raw_dmg = max(base_dmg - base_def, 1.0)
6. dmg = raw_dmg × random(0.9, 1.1)
7. dmg ×= counter_multiplier
8. 伏击判定：p = ambush_base(0.05) + terrain.ambush_chance，上限 ambush_chance_cap(0.95)
   若触发：dmg ×= ambush_damage_multiplier(2.0)
9. 返回 floor(dmg)
```

**未接入的修正（硬编码为 1.0）**：`atk_general_mod`、`atk_faction_mod`、`def_general_mod`、`def_faction_mod`。夹击、火攻、攻城等参数已在 JSON 中定义但未在代码中实现。

## 5. 特殊战斗机制

### 5.1 伏击（被动，加法层 +0.3）

**触发公式**：
```
ambush_chance = base_ambush_chance + terrain.ambush_chance
base_ambush_chance = 0.05（5%）
上限 = ambush_chance_cap（0.95）
```

**地形伏击概率**：

| 地形 | ambush_chance |
|:---|:---:|
| 平原 | 0% |
| 森林 | 30% |
| 山地 | 20% |
| 沼泽 | 10% |
| 关隘 | 15% |
| 沙漠 | 5% |
| 冻土 | 10% |

**效果**：触发时 `atk_buff += 0.3`（攻击力 +30%）

**与火攻互斥**：主动选择火攻则跳过伏击判定。

**数据来源**：`balance_params.json` → `combat.ambush_base_chance`、`combat.ambush_atk_bonus`

### 5.2 夹击（防御层 debuff -0.3）

**触发条件**：攻击方有 **两个单位同时与同一目标接壤**（六角格相邻）。即攻击方有两个单位分别位于目标的相邻格子上，且其中至少一个单位发起攻击。

**效果**：防御方 `def_buff += -0.3`（防御力降低 30%）

**数据来源**：`balance_params.json` → `combat.flank_def_reduction`（值 0.7，实际效果为 -0.3）

### 5.3 火攻（主动，夏/秋可用，加法层 +0.4）

**触发条件**：
- 仅夏季、秋季可用
- 目标必须在森林地形
- 与伏击互斥

**效果**：
- `atk_buff += 0.4`
- 持续伤害：`burn_dot = base_dmg × 0.3`，持续 2 回合
- 兵家学派：夏秋火攻 `atk_buff` 额外 +0.2

**数据来源**：`balance_params.json` → `combat.fire_atk_bonus`（0.4）、`fire_burn_duration`（2）、`fire_burn_dot_ratio`（0.3）

### 5.4 远程精度修正

`terrain.json` 定义了 `ranged_accuracy_mod` 字段，影响远程单位的**随机波动幅度**。

**公式**：
```
spread_lo = base_random_spread_lo + accuracy × accuracy_spread_factor
spread_hi = base_random_spread_hi + accuracy × accuracy_spread_factor

// base_random_spread_lo = 0.9, base_random_spread_hi = 1.1
// accuracy_spread_factor = 0.001

// 精度 0（基准）：波动范围 0.9~1.1（±10%）
// 精度 +100：波动范围 1.0~1.2（0%~+20%，伤害偏高）
// 精度 -100：波动范围 0.8~1.0（-20%~0%，伤害偏低）
```

| 地形 | 远程精度修正 | 波动范围 |
|:---|:---:|:---|
| 平原 | 0 | 0.9~1.1（±10%，基准） |
| 森林 | -20 | 0.88~1.08 |
| 山地 | +10 | 0.91~1.11 |
| 沼泽 | -10 | 0.89~1.09 |
| 关隘 | +15 | 0.915~1.115 |
| 渡口 | -5 | 0.895~1.095 |
| 沙漠 | -5 | 0.895~1.095 |
| 冻土 | -10 | 0.89~1.09 |

**数据来源**：`balance_params.json` → `combat.base_random_spread_lo/hi`、`combat.accuracy_spread_factor`，`terrain.json` → `ranged_accuracy_mod`

### 5.5 远程进攻机制

远程单位（`special: ranged`，range ≥ 2）拥有两种攻击模式：

**主动远程攻击**（不触发反击）：
- 标记为 `ranged_attack_mode: "no_counter"`
- 攻击值与当前攻击力数值相同
- 适用远程精度修正（`ranged_accuracy_mod` 影响随机波动）
- 适用遮挡机制（高程差减射程）
- **不触发反击**

**近战反击**（被攻击时触发）：
- 当远程单位被近战单位攻击时，使用攻击力正常反击
- 反击伤害 100%，与主动攻击完全相同
- 适用克制矩阵、地形修正等所有加法层和乘算层

**不触发反击的攻击来源**：
- 城市主动攻击
- 箭塔攻击
- 远程单位主动远程攻击

**所有近战攻击都会触发反击**（攻方受到防方的反击伤害）。

**数据来源**：`units.json` → `ranged_attack_mode: "no_counter"`

### 5.6 远程遮挡机制

远程单位从低处向高处射击时，射程受地形高程差影响。

**地形高程等级**：

| 高程 | 地形 |
|:---:|:---|
| -1 | 河流、深海 |
| 0 | 平原、沙漠、冻土、浅海 |
| 1 | 森林、沼泽、山地、关隘、渡口 |

**规则**：
- 从低处向高处射击：每高 1 级高程，**射程减少 1 格**
- 从高处向低处射击：无影响
- 同高程射击：无影响

**示例**：
- 弓兵在平原(高程0, 射程2)向山地(高程1)射击：有效射程 = 2 - 1 = 1，可以攻击
- 弓兵在河流(高程-1, 射程2)向关隘(高程1)射击：有效射程 = 2 - 2 = 0，无法攻击
- 弩兵在山地(高程1, 射程2)向平原(高程0)射击：有效射程 = 2，正常攻击

**数据来源**：`terrain.json` → `elevation`

| 区间 | 状态 | 效果 |

// 低士气（20~49）：morale_atk_offset = -0.1

## 7. 城市/关隘战斗

### 7.1 部队 vs 城防（公式 1.6）

**伤害分配**：
```
if garrison_exists:
    wall_damage = final_dmg × 0.5
    garrison_damage = final_dmg × 0.5
else:
    wall_damage = final_dmg × 1.0
```

**攻城器械加成**（仅作用于城墙伤害部分）：
```
if attacker.special == "siege_bonus":
    wall_damage ×= siege_damage_multiplier  // ×3.0
```

**防御计算**：
```
city_defense = city_base_defense × (1 + wall_defense_bonus × wall_hp_ratio)
// wall_hp_ratio: 城墙 HP 100% → 1.0, HP 50% → 0.5, HP 0% → 0.5（最低 50%）
// 城墙 HP=0 时 defense_bonus 保留 50% 效果
// 城墙被占领后 defense_bonus 完全失效

wall_net = max(wall_damage - city_defense, 1)
garrison_net = max(garrison_damage - garrison_defense, 1)
```

**数据来源**：`balance_params.json` → `city_combat.damage_split_garrison/wall`（各 0.5）、`city_combat.siege_damage_multiplier`（3.0）

### 7.2 部队 vs 驻进城防（公式 1.7）

同 7.1 伤害分配规则，额外考虑：
- **驻军防御**：使用自身部队防御值（含地形、大夫、学派等 buff）
- **城墙防御**：建筑防御值 × (1 + 城墙等级加成)
- **攻城器械倍率**：仅作用于城墙伤害部分，不影响驻军伤害
- **城市主动攻击**：每回合可主动攻击范围内 1 个敌军目标，简化公式 `damage = city_attack - target_defense`（无克制矩阵，不触发反击）
- **箭塔独立攻击**：各自攻击范围内敌军，简化公式 `damage = tower_attack - target_defense`（无克制矩阵，不触发反击）

**战斗顺序**：1. 城市本体攻击 → 2. 各箭塔独立攻击 → 3. 敌军反击时伤害分配到城墙/驻军

## 10. 战斗状态机与回合机制

### 10.1 行动速度与回合顺序

每回合开始时，根据各势力的 `action_speed` 属性排序，决定行动顺序。

数据来源：`data/factions.json` → `action_speed`

| 势力 | action_speed | 说明 |
|:---|:---:|:---|
| 秦 | 80 | 虎狼之师，行动迅速 |
| 赵 | 70 | 胡服骑射，机动性强 |
| 魏 | 65 | 武卒精锐，纪律严明 |
| 燕 | 60 | 坚韧不拔，执行力强 |
| 齐 | 55 | 文风鼎盛，节奏稳健 |
| 楚 | 50 | 地广人众，调度较慢 |
| 韩 | 45 | 精工巧匠，稳扎稳打 |

**排序规则**：
1. 按 `action_speed` 降序排列
2. 相同时随机排序（每局游戏开始时固定种子）

### 10.2 单势力行动回合制

每回合内**只有一个势力**可以执行行动，包括：
- 移动单位
- 发起攻击/战斗
- 经营操作（建造、征兵、研发等）
- 外交操作

**回合流程**：
```
1. 回合开始 → 按 action_speed 排序确定行动顺序
2. 当前势力执行所有操作
3. 当前势力结束回合 → 下一个势力开始
4. 所有势力行动完毕 → 回合结束，进入下一回合
```

### 10.3 单格单部队限制

每个六角格**只能容纳一支部队**：
- 已被友军占据的格子无法进入
- 已被敌军占据的格子需要先击败守军才能占领
- 移动路径规划时自动绕过已占据格子

**程序实现**：`_occupant_id_at()` 检查格子占用状态

### 10.4 伤害溢出机制

当部队被击杀时，**多余的伤害直接溢出**，不保留：

```
实际伤害 = min(攻击力, 目标当前HP)
溢出伤害 = 攻击力 - 目标当前HP  // 丢弃，不转移
```

**示例**：
- 攻击力 50，目标 HP 30 → 造成 30 伤害，目标被歼灭，20 点溢出伤害丢弃
- 攻击力 50，目标 HP 80 → 造成 50 伤害，目标剩余 30 HP

**设计意图**：简化战斗结算，避免复杂的伤害转移逻辑

### 10.5 状态机状态定义

```enum
enum TurnState {
    TURN_START,        // 回合开始，计算行动顺序
    FACTION_ACTION,    // 势力行动中
    FACTION_END,       // 势力结束回合
    TURN_END           // 回合结束，触发回合结束效果
}
```

## 13. 补给与断粮

### 13.1 BFS 连通性检测算法

```
function check_supply(unit, friendly_cities):
    visited = set()
    queue = [unit.hex_position]
    visited.add(unit.hex_position)
    while queue is not empty:
        current = queue.pop_front()
        for neighbor in get_neighbors(current):
            if neighbor in visited: continue
            if not is_passable(neighbor, unit.faction): continue
            if neighbor in friendly_cities: return SUPPLIED
            visited.add(neighbor)
            queue.append(neighbor)
    return CUT_OFF
```

BFS 从部队位置向所有友方城市探索可通行路径。可通行判定：非不可通行地形 + 非敌方部队占据。
