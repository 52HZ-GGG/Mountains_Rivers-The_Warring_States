# 随机事件系统

> **更新日期**：2026-05-15
> **版本**：v1.2
> **数据来源**：`data/events.json`、`scripts/autoload/event_manager.gd`
> **状态**：✅ 已实现（核心引擎完成，部分条件类型待扩展）

---

## 1. 系统概述

每回合开始时触发判定，按概率触发符合条件的事件。88 个事件分 8 类，65 个有玩家选择，23 个自动生效。事件有 3 回合冷却防止重复。有选项事件支持资源消耗前置检查，资源不足时选项不可选。

**设计目标**：
- 提供每局不同的随机性和决策感
- 通过历史事件增强战国代入感
- 通过学派事件与学派系统联动
- 通过外交事件链（合纵/连横/天子礼制）增强外交策略深度
- 覆盖全部 10 种资源（含 4 种特殊资源）
- 所有资源效果通过 `GameManager` 接口应用，外交效果通过 `DiplomacySystem` 公共接口应用

## 3. 触发机制

### 3.1 触发时机

所有事件在 `turn_start` 时触发（玩家回合开始时）。`event_manager.gd:22-25`。

### 3.2 触发流程

```
回合开始信号
  → 遍历所有事件
    → 检查 trigger.type == "turn_start"
    → 检查冷却（_is_on_cooldown）
    → 检查条件（_check_conditions）
    → 概率判定（randf <= probability）
    → 触发事件（_trigger_event）
```

### 3.3 冷却机制

- 默认冷却：3 回合（`balance_params.json` → `event_cooldown_turns`）
- 冷却在 `turn_ended` 时递减
- 冷却中的事件不会被判定触发

### 3.4 双路径触发

| 路径 | 条件 | 行为 |
|:---|:---|:---|
| 无选项事件 | `options == null` | 直接应用 `effects`，发出 `event_resolved` 信号 |
| 有选项事件 | `options != null` | 发出 `event_triggered` 信号，等待 UI 处理玩家选择 |

### 5.2 效果结构

**无选项事件**：效果直接定义在 `effects` 字段中。

```json
{
  "effects": {
    "food_delta": 200,
    "gold_delta": 0,
    "morale_delta": 10,
    "population_delta": 0,
    "troops_delta": 0,
    "wood_delta": 0
  },
  "options": null
}
```

**有选项事件**：效果定义在每个选项的 `outcomes` 字段中。

```json
{
  "effects": null,
  "options": [
    {
      "id": "A",
      "text": "选项A描述",
      "cost": {"gold": 80},
      "outcomes": { "food_delta": -150, "morale_delta": 15, ... }
    },
    {
      "id": "B",
      "text": "选项B描述",
      "cost": {},
      "outcomes": { "food_delta": 50, "morale_delta": -10, ... }
    }
  ]
}
```

## 6. 消耗系统

### 6.1 cost 机制

有选项事件的 `cost` 字段定义选择该选项需要消耗的资源。`event_manager.gd` 在应用效果前先检查资源是否足够：

- **前置检查**：`_can_afford(cost)` 检查玩家资源是否满足所有 cost 项
- **资源扣除**：`_deduct_cost(cost)` 在效果应用前扣除 cost 中的资源
- **UI 查询**：`can_afford_option(event_id, choice_id)` 供 UI 判断选项是否可选

### 6.2 cost 支持的资源类型

| 资源 | cost 字段名 |
|:---|:---|
| 粮食 | `food` |
| 金钱 | `gold` |
| 木材 | `wood` |
| 工匠 | `craftsmen` |
| 建材 | `building_materials` |
| 马匹 | `horse` |
| 精铁 | `refined_iron` |

### 6.3 cost 设计原则

- cost 数值约为对应 outcomes 中资源消耗的 50~80%
- 历史战役事件的"历史选项"通常有较高 cost（体现战略投入）
- 经济/民心事件中涉及投入的选项有 gold/food cost
- 学派事件中涉及拨款的选项有 gold cost
- "不做"选项（B 选项中较保守的）通常无 cost

## 8. 代码实现

### 8.1 文件结构

| 文件 | 职责 |
|:---|:---|
| `data/events.json` | 事件数据定义（88 个事件） |
| `scripts/autoload/event_manager.gd` | 事件引擎（触发/条件/冷却/cost检查/效果应用/外交效果） |
| `scripts/autoload/diplomacy_system.gd` | 外交公共接口（change_opinion/change_reputation/事件链标记） |
| `scripts/autoload/signal_bus.gd` | 信号定义（event_triggered/event_resolved/school_exp_gained） |
| `scripts/autoload/data_manager.gd` | 数据加载接口（get_all_events/get_event） |

### 8.2 event_manager.gd 接口

| 方法 | 返回 | 说明 |
|:---|:---|:---|
| `resolve_event_choice(event_id, choice_id)` | bool | 处理玩家选项选择，资源不足返回 false |
| `can_afford_option(event_id, choice_id)` | bool | 检查选项是否可承担（供 UI 调用） |
| `reset()` | void | 重置所有状态（测试/重开） |
| `get_cooldowns()` | Dictionary | 获取当前冷却状态（测试用） |

### 8.3 信号

| 信号 | 参数 | 时机 |
|:---|:---|:---|
| `event_triggered` | `event_data: Dictionary` | 有选项事件触发时 |
| `event_resolved` | `event_id: String, choice_id: String` | 事件结算后 |
| `school_exp_gained` | `amount: int` | 事件获得学派经验时 |

## 10. 数据 Schema

### events.json 顶层结构

```json
{
  "schema_version": "1.0",
  "events": [ ... ]
}
```

### 事件对象结构

```json
{
  "id": "evt_harvest_bumper",        // 唯一标识
  "title": "五谷丰登",               // 显示标题
  "description": "...",               // 事件描述
  "category": "economy",              // 分类（8种）
  "trigger": {
    "type": "turn_start",             // 触发时机
    "probability": 0.15,              // 触发概率（0~1）
    "conditions": { ... }             // 触发条件（可选）
  },
  "effects": { ... } | null,          // 无选项事件效果
  "options": [ ... ] | null           // 有选项事件选项列表
}
```

### 条件对象字段

```json
{
  "season": ["autumn"],               // 限定季节数组
  "morale_min": 50,                   // 民心下限
  "morale_max": 40,                   // 民心上限
  "turn_min": 3,                      // 最早回合
  "turn_max": 2,                      // 最晚回合
  "faction": "qin",                   // 限定势力
  "at_war_with": "wei",              // 与指定势力交战
  "tech_researched": "tech_id",      // 已研究科技
  "at_war": true,                     // 是否战争状态（⏳）
  "school": "confucianism",          // 当前学派（⏳）
  "has_alliance": true,               // 有同盟（⏳）
  "allies_min": 1,                    // 最少同盟数（⏳）
  "troops_in_hex_range": {            // 部队范围检查（⏳）
    "center": [8, 6],
    "radius": 3,
    "both_factions": true
  }
}
```

### 选项对象结构

```json
{
  "id": "A",                          // 选项标识（A/B/C）
  "text": "选项描述文本",              // 显示文本
  "cost": {                           // 选择消耗（前置检查）
    "gold": 80,                       // 金钱消耗
    "food": 100,                      // 粮食消耗
    "wood": 30                        // 木材消耗
  },
  "outcomes": { ... }                 // 选择效果
}
```

### 效果对象字段（11 种）

```json
{
  "food_delta": 200,                  // 粮食变化
  "gold_delta": -100,                 // 金钱变化
  "wood_delta": 0,                    // 木材变化
  "craftsmen_delta": 5,              // 工匠变化
  "building_materials_delta": 3,     // 建材变化
  "morale_delta": 10,                 // 民心变化
  "population_delta": 0,             // 人口变化
  "troops_delta": 0,                 // 兵力变化
  "horse_delta": 8,                  // 马匹变化
  "refined_iron_delta": 6,           // 精铁变化
  "school_exp": 15                   // 学派经验（学派事件专用）
}
```
