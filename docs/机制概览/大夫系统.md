# 大夫系统

> 本文档整合《山河策》所有大夫系统机制细节，作为程序实现的唯一参考源。
> 所有数值参数从 `data/*.json` 读取，禁止硬编码。

---

## 1. 设计概述

大夫系统完全替代旧将军系统（决策 #2），是独立于部队和建筑的角色系统，覆盖城市管理、军队统帅、外交使节三大职能。三类大夫（文/武/外交）各有独立容量上限、二维百分比属性、技能槽和升级路径。

**核心设计原则**：

1. **三类分治**：文大夫治城、武大夫统兵、外交大夫出使，覆盖游戏三大维度
2. **容量限制**：每类大夫有独立容量上限，迫使玩家做人才配置取舍
3. **档次梯度**：普通/稀有传说三档，传说级绑定历史名人，提供收集欲
4. **数据驱动**：所有数值从 `ministers.json` / `balance_params.json` 读取
5. **系统联动**：大夫作为中间层，连接城市/战斗/外交/学派四大系统

## 2. 大夫分类

### 2.1 三类大夫总览

| 类型 | 容量上限 | 任命位置 | 核心功能 | 属性 |
|:---|:---:|:---|:---|:---|
| 文大夫 | 5 | 城市 | 城市营收 + 稳定度 | 理财% + 安民% |
| 武大夫 | 5 | 部队 | 攻防 buff + 特殊技能 | 勇武% + 韬略% |
| 外交大夫 | 2 | 指定目标国家 | 降低外交成本 + 被动友好度 | 辩才% + 亲和% |

### 2.2 容量扩充

| 途径 | 扩充量 | 说明 |
|:---|:---:|:---|
| 科技 | 每类最多 +2 | 如「郡县制」→ 文大夫 +1，「军功爵制」→ 武大夫 +1 |
| 建筑（招贤馆） | 全局最多 +1 | 所有类型各 +1 |
| 事件 | 每类最多 +1 | 特殊事件奖励（如「稷下学宫」事件） |

**最大容量**：文 9 / 武 9 / 外交 6

**数据来源**：`balance_params.json` → `minister.capacity`

## 3. 属性系统

### 3.1 属性定义

每类大夫有二维百分比属性，影响其核心功能：

| 类型 | 属性 1 | 属性 2 | 属性 1 效果 | 属性 2 效果 |
|:---|:---|:---|:---|:---|
| 文大夫 | 理财% | 安民% | 城市金钱产出 +N% | 城市稳定度 +N |
| 武大夫 | 勇武% | 韬略% | 部队攻击力 +N% | 部队防御力 +N% |
| 外交大夫 | 辩才% | 亲和% | 外交行动资源消耗 -N% | 目标国友好度 +N/回合 |

### 3.2 属性公式

```
文大夫：
  city_gold_output  *= (1 + 理财% / 100)
  city_stability    += 安民%

武大夫：
  army_attack       *= (1 + 勇武% / 100)
  army_defense      *= (1 + 韬略% / 100)

外交大夫：
  diplomacy_cost    *= (1 - 辩才% / 100)    // 最低降至 50%
  target_opinion    += 亲和% / 10            // 每回合，四舍五入
```

### 3.3 档次属性范围

| 档次 | 属性范围（每个属性独立随机） |
|:---:|:---|
| 普通 | 10~50 |
| 稀有 | 20~70 |
| 传说 | 40~100 |

**数据来源**：`balance_params.json` → `minister.stat_ranges`

## 7. 获取系统

### 7.1 四种获取途径

| 途径 | 说明 | 产出类型 |
|:---|:---|:---|
| 建筑产出 | 学院→文大夫，兵营→武大夫，驿站→外交大夫 | 按建筑固定 |
| 事件触发 | 随机事件中出现招募选项 | 随机 |
| 战斗获得 | 部队击败敌军后概率获得 | 武大夫为主 |
| 科技加成 | 研发特定科技增加获取概率或容量上限 | 不直接产出 |

### 7.2 建筑产出机制

- 建筑每 8 回合产出 1 个大夫（基础速率）
- 产出类型由建筑决定：学院→文大夫，兵营→武大夫，驿站→外交大夫
- 产出时从概率池中抽取档次

### 7.3 战斗获得机制

- 战斗胜利后 15% 概率获得 1 个武大夫

### 7.4 概率池

| 档次 | 基础概率 |
|:---:|:---:|
| 普通 | 60% |
| 稀有 | 30% |
| 传说 | 10% |

可通过科技、建筑、学派调整。

### 7.5 学派影响获取

| 学派 | 影响 |
|:---|:---|
| 儒家 | 文大夫获取概率 +20% |
| 法家 | 文大夫获取概率 +10%，武大夫获取概率 +10% |
| 墨家 | 文大夫获取概率 +10% |
| 道家 | 所有大夫获取概率 -10%（道家崇尚无为） |
| 兵家 | 武大夫获取概率 +20% |
| 纵横家 | 外交大夫获取概率 +20% |

**数据来源**：`balance_params.json` → `minister.acquisition`

## 8. 命运机制

### 8.1 武大夫（部队被歼灭时）

| 结果 | 概率 |
|:---:|:---:|
| 死亡 | 30% |
| 被俘 | 40% |
| 逃脱 | 30% |

### 8.2 文大夫（城破时）

| 结果 | 概率 |
|:---:|:---:|
| 死亡 | 30% |
| 被俘 | 40% |
| 逃脱 | 30% |

### 8.3 外交大夫

- 不因城破/部队被歼触发命运判定
- 固定回合寿命：30 回合（自然死亡）
- 寿命到期后自然死亡，无法避免

### 8.4 被俘处置

- 被俘后可花费金币赎回（赎金 = 档次基础 + 等级 × 50）
  - 普通基础 100 金，稀有基础 250 金，传说基础 500 金
  - 例：传说 5 级 = 500 + 5×50 = 750 金
- 可通过外交谈判要回（消耗外交行动）
- 若未达成赎回/谈判，被俘大夫有概率：
  - 自杀（40%）：永久失去
  - 归顺对方（60%）：加入对方阵营
- 若对方大夫容量已满，被俘大夫自动死亡

### 8.5 战斗保护规则

- 城市或部队上一回合遭遇进攻时，该位置的大夫不能卸任
- 防止玩家通过临时卸任规避命运判定

**数据来源**：`balance_params.json` → `minister.fate` / `minister.ransom`

## 11. 跨系统交互

### 11.1 交互链路总览

```
大夫系统
  ├─→ 城市经营：文大夫理财%/安民% 加成城市产出和稳定度
  ├─→ 战斗系统：武大夫勇武%/韬略% 加成部队攻防
  ├─→ 外交系统：外交大夫辩才% 降低行动成本，亲和% 增长友好度
  ├─→ 科技树：科技解锁容量扩充和获取概率
  ├─→ 学派系统：领袖指派、授学、传道任务、获取概率修正
  └─→ 随机事件：事件触发大夫招募
```

### 11.2 与城市系统

- 文大夫任命到城市后，`CityManager` 计算产出时读取大夫的理财%/安民% 加成
- 文大夫在任期间，城市稳定度每回合额外恢复（安民% 效果）

### 11.3 与战斗系统

- 武大夫任命到部队后，`CombatResolver` 读取大夫的勇武%/韬略% 加成
- 替代旧的 `atk_general_mod` 和 `def_general_mod` 占位符
- 战斗胜利后触发大夫获取判定和经验获取

### 11.4 与外交系统

- 外交大夫指定目标国家后，`DiplomacySystem` 读取大夫的辩才%/亲和% 加成
- 辩才% 降低外交行动资源消耗（最低降至 50%）
- 亲和% 每回合自动增加目标国友好度

### 11.5 与科技系统

- 特定科技解锁容量扩充（如「郡县制」→ 文大夫 +1）
- 特定科技增加获取概率

### 11.6 与学派系统

- 学派影响大夫获取概率（详见第 7.5 节）
- 大夫可被指派为学派领袖或授学导师（详见第 10 节）

### 11.7 与事件系统

- 随机事件可触发大夫招募（如「商鞅入秦」「张仪入楚」）
- 特殊事件可扩充容量

## 12. 数据文件索引

| 文件 | 大夫相关字段 |
|:---|:---|
| `data/ministers.json` | 大夫池（civil/military/diplomat × common/rare/legendary）、姓名模板（surnames/given_names）、技能定义（civil/military/diplomat/universal） |

| `data/schools.json` | 学派领袖品质倍率、授学经验产出、学派获取概率修正 |
| `data/factions.json` | 各势力 `default_school`（影响学派获取概率） |
