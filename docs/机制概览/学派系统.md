# 学派系统

> 本文档整合《山河策》所有学派系统机制细节，作为程序实现的唯一参考源。
> 所有数值参数从 `data/*.json` 读取，禁止硬编码。

---

## 1. 设计概述

六大学派（儒家、法家、墨家、道家、兵家、纵横家）为玩家提供差异化的国家战略方向。玩家开局自由选择学派，通过完成学派任务积累经验、提升等级，解锁更强的被动 buff 和政策槽位。学派系统拥有独立的事件池、政策系统、领袖机制，并与外交、民心、文化、大夫等系统深度联动。

**核心设计原则**：

1. **数据驱动**：所有数值从 `schools.json` / `balance_params.json` / `events.json` 读取，禁止硬编码
2. **等级递进**：3 级体系，每级叠加 buff，提供持续成长感
3. **策略抉择**：学派经验用于升级还是激活政策，玩家需权衡资源分配
4. **系统联动**：学派与外交、民心、文化、大夫、AI 行为等 6 个系统交互
5. **自由切换**：玩家可切换学派，但三重代价保证切换有战略分量

## 4. 学派经验系统

### 4.1 经验来源

| 来源 | 经验值 | 条件 | 说明 |
|:---|:---:|:---|:---|
| 完成学派任务 | 15~30 | 任务完成 | 主要来源 |
| 学派事件触发 | 5~10 | 事件触发 | 次要来源 |
| 游戏行为自动获得 | 5 | 特定行为 | 如兵家赢得战斗 +5，儒家提高民心 +5 |
| 大夫授学 | 1~3/回合 | 大夫被指派为学派导师 | 持续来源 |
| 学派领袖加成 | 2/回合 | 领袖被指派 | 持续来源 |

### 4.2 经验用途（共享资源）

学派经验是共享资源，用于两个目的：

1. **升级**：积累到阈值后自动升级，消耗对应经验
2. **激活政策**：消耗经验激活学派政策（详见第 6 节）

玩家需在"攒经验升级"和"花经验激活政策"之间权衡。

### 4.3 数据定义

经验来源的具体数值定义在 `balance_params.json` → `school` 段：

```json
{
  "school": {
    "exp_to_level_2": 50,
    "exp_to_level_3": 120,
    "quest_exp_base": 15,
    "quest_exp_bonus_multiplier": 1.5,
    "event_exp_base": 5,
    "auto_exp_per_action": 5,
    "mentor_exp_per_turn": 1,
    "leader_exp_per_turn": 2
  }
}
```

### 4.4 游戏行为自动获得经验值

玩家在游戏过程中触发特定行为时，自动为当前学派获得经验。数据来源：`balance_params.json` → `school.auto_exp_triggers`。

| 学派 | 触发行为 | 经验值 | 条件 |
|:---|:---|:---:|:---|
| 儒家 | 民心达到阈值 | +5/次 | 70/80/90/100/110/120，每阈值仅触发一次 |
| 法家 | 科技研究完成 | +2 | 无条件，每完成一个科技即触发 |
| 墨家 | 建造城墙/箭塔/瓮城完成 | +3 | 仅城墙类防御建筑 |
| 道家 | 单城人口达到 15 | +5 | 单城总人口达到 15 时触发 |
| 兵家 | 攻城胜利 | +5 | 不要求武大夫在场 |
| 纵横家 | 签订条约 | +5 | 外交行动成功 |

## 5. 学派任务系统

### 5.1 任务刷新机制

| 属性 | 值 | 来源 |
|:---|:---:|:---|
| 刷新周期 | 每 10 回合 | `balance_params.json` → `school.quest_refresh_interval` |
| 每次刷新数量 | 2~3 个 | `balance_params.json` → `school.quest_refresh_count` |
| 旧任务处理 | 被替换（不累积） | — |
| 任务来源 | 当前学派的 `quest_pool` | `schools.json` |

### 5.2 任务类型

| 类型 | 说明 | 示例 |
|:---|:---|:---|
| 通用任务 | 任何玩家可完成 | "治下民心达到 70" |
| 传道任务 | 需对应学派大夫执行 | "由儒家大夫治理的城市达到 5 座" |

传道任务经验奖励为通用任务的 1.5 倍。

### 5.3 任务条件类型

| `condition_type` | 说明 | 条件字段 |
|:---|:---|:---|
| `threshold` | 数值达标 | `{ "target": "national_morale", "operator": ">=", "value": 70 }` |
| `action` | 完成特定行为 | `{ "target": "win_battle", "count": 3 }` |
| `possession` | 拥有特定资源/建筑 | `{ "target": "building", "building_id": "academy", "count": 2 }` |

### 5.4 任务数据结构

定义在 `schools.json` → `quest_pool`：

```json
{
  "id": "conf_q1",
  "type": "general",
  "name": "仁政之始",
  "description": "治下民心达到 70",
  "condition_type": "threshold",
  "condition": { "target": "national_morale", "operator": ">=", "value": 70 },
  "reward_exp": 20
}
```

```json
{
  "id": "conf_q2",
  "type": "missionary",
  "name": "礼乐教化",
  "description": "由儒家大夫治理的城市达到 5 座",
  "condition_type": "threshold",
  "condition": { "target": "confucian_governed_cities", "operator": ">=", "value": 5 },
  "reward_exp": 30,
  "requires_minister_school": "confucianism"
}
```

### 5.5 信号

```
SignalBus.school_quest_refreshed(school_id: String, quests: Array)
SignalBus.school_quest_completed(school_id: String, quest_id: String, reward_exp: int)
```

## 9. 学派切换机制

### 9.1 切换条件

- 玩家在学派面板主动选择"改换门庭"
- AI 默认不切换（走历史路线），濒临灭亡时有小概率切换

### 9.2 三重代价

| 代价 | 效果 | 来源 |
|:---|:---|:---|
| 等级损失 | 新学派从 1 级开始（旧学派等级清零） | — |
| 民心惩罚 | 切换当回合民心 -X（X = 旧学派等级 × 5） | `balance_params.json` → `school.switch_morale_penalty_per_level` |
| 过渡期 | 切换后 3 回合内，新学派效果完全不生效（×0.0）；旧学派未完成任务全部失效；无法接取新学派任务 | `balance_params.json` → `school.switch_transition_turns`、`school.switch_transition_effect_multiplier`、`school.switch_transition_quest_lock` |

### 9.3 切换后状态重置

- 学派经验保留（不因切换损失）
- 已激活的政策全部失效
- 学派领袖自动卸任
- 大夫授学自动终止

### 9.4 过渡期机制详细说明

| 属性 | 值 | 来源 |
|:---|:---|:---|
| 过渡期时长 | 3 回合 | `balance_params.json` → `school.switch_transition_turns` |
| 效果倍率 | ×0.0（完全不生效） | `balance_params.json` → `school.switch_transition_effect_multiplier` |
| 任务锁定 | 无法接取新学派任务 | `balance_params.json` → `school.switch_transition_quest_lock: true` |
| 旧任务处理 | 旧学派未完成任务全部失效 | — |
| 政策限制 | 无法激活新学派政策 | — |

过渡期内，新学派的 `level_effects` 全部乘以 0.0，即不产生任何效果。玩家需在切换前评估 3 回合"真空期"的战略风险。

## 16. 与旧设计的差异

| 维度 | 旧设计（阶段 3） | 新设计 |
|:---|:---|:---|
| 效果结构 | 扁平 `global_effects` | 递进 `level_effects`（3 级），唯一效果来源 |
| 等级系统 | 无 | 3 级，独立于时代 |
| 效果完整性 | 仅 2-3 个效果维度 | 每级包含学派全部效果维度（民心/文化/战斗/经济/外交/季节） |
| 字段命名 | 部分字段与现有系统不匹配 | 全部对齐现有系统字段（`research_speed_bonus`、`defense_bonus` 等） |
| 法家代价 | 民心 -1/回合（过于严苛） | 取消民心惩罚，改为外交惩罚（`diplomacy_penalty`）和安定度加成 |
| 季节联动 | 独立 `season_bonus` 字段 | 整合进 `level_effects` 对应等级 |
| 外交修正 | 独立 `diplomacy_modifiers` 字段 | 整合进 `level_effects` 对应等级 |
| 任务系统 | 无 | 每 10 回合刷新 2-3 个可选任务 |
| 政策系统 | 1 个固定特殊政策 | 通用 + 专属政策池，每级 1 槽位，消耗经验激活 |
| 领袖机制 | 无 | 2 级解锁，同学派大夫，品质倍率 |
| 大夫授学 | 无 | 同学派大夫每回合提供经验 |
| 切换机制 | 无 | 三重代价（等级-1 + 民心罚 + 过渡期） |
| 专属建筑 | 无 | 每学派 1 个，与等级联动 |
| AI 权重 | 无 | 6 维行为权重叠加 |
| 事件触发 | 无法触发 | 扩展 EventManager 条件支持 |
| 数据字段 | `special_policy`、`season_bonus`、`diplomacy_modifiers` 独立 | 移除冗余字段，全部整合进 `level_effects` |
