## 1. 设计概述

城市是游戏核心实体，承载资源产出、征兵、战斗防御、文化扩散四大职能。全图 50 座城市分布在 7 个势力和中立阵营中，每城有等级、建筑槽位、人口、特产、HP、攻击力等属性。

**势力分布**：秦 8 / 楚 9 / 齐 7 / 赵 7 / 魏 6 / 韩 5 / 燕 5 / 中立 3

**核心设计原则**：

1. **城市等级驱动**：5 级体系（村/镇/城/府/都）决定 HP、攻击力、建筑槽位、产出上限
2. **建筑辖区制**：建筑建在城市周围的六角格上，有实体空间位置
3. **人均产出模型**：人口是粮食和金钱的产出基础，受季节/建筑/特产/安定度多维修正
4. **安定度影响产出**：城市级安定度（0~100）直接影响该城所有资源产出
5. **文化渗透**：邻城文化扩散影响安定度和城市归属
6. **城市地形**：城市所在六角格强制视为平原（plains），不享受原地块地形的战斗修正；城市及建筑不影响原地块地形效果
7. **数据驱动**：所有数值从 `balance_params.json` / `buildings.json` / `cities.json` / `wonders.json` 读取

## 2. 城市属性与等级

### 2.1 城市等级体系（5 级：村/镇/城/府/都）

| 等级 | 名称 | HP | 攻击力 | 攻击范围 | 城防 | 建筑槽位 | 基础粮产 | 基础金产 | 最大人口 | 最大粮食产出 |
|:---:|:---|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| 1 | 村 | 300 | 5 | 1 | 10 | 1 | 3 | 5 | 10 | 30 |
| 2 | 镇 | 600 | 10 | 1 | 25 | 2 | 6 | 10 | 15 | 45 |
| 3 | 城 | 1000 | 15 | 1 | 45 | 3 | 10 | 15 | 20 | 60 |
| 4 | 府 | 1500 | 20 | 2 | 70 | 4 | 15 | 25 | 25 | 75 |
| 5 | 都 | 2000 | 25 | 2 | 100 | 5 | 20 | 35 | 30 | 90 |

**数据来源**：`balance_params.json` → `city_levels`

**设计意图**：等级越高，城市的军事、经济、建筑容量全面增强。府和都（等级 4~5）拥有远程攻击能力（攻击范围 2）。

### 2.2 首都加成

首都城市额外获得：

| 属性 | 加成 |
|:---|:---:|
| HP | +500 |
| 攻击力 | +5 |

**数据来源**：`balance_params.json` → `city_levels.capital_bonus`

**示例**：5 级都城为首都时，HP = 2000 + 500 = 2500，攻击力 = 25 + 5 = 30。

### 2.3 发展度

每座城市有发展度属性（1~100），影响 AI 迁都选择和城市综合实力评估。

| 城市类型 | 发展度范围 | 示例 |
|:---|:---:|:---|
| 首都 | 55~75 | 咸阳/临淄 75，郢 70，邯郸 70，大梁 65，新郑 60，蓟 55 |
| 大城 | 30~50 | 蜀郡 35，晋阳 40，彭城 35，洛邑 45 |
| 中城 | 15~30 | 栎阳 25，濮阳 25，上党 20 |
| 小城/边城 | 5~15 | 陇西 15，辽西 10，渔阳 10 |
| 中立城 | 10~45 | 洛邑 45（周室旧都），邢台 10，定陶 15 |

**数据来源**：`cities.json` → `development`

### 2.4 城市特产

每座城市可拥有 1 种特殊资源特产，影响资源产出。无特产的城市不获得额外加成。

| 特产 | 效果 | 城市数量 | 示例 |
|:---|:---|:---:|:---|
| wood（木材） | 木材产出 +30%，木材消耗 -20% | 7 | 咸阳、陈仓、宛城、晋阳、安邑、上党、宜阳 |
| horse（马匹） | 马匹产出 +30% | 9 | 陇西、邯郸、代郡、云中、雁门、辽阳、辽西、上谷、渔阳 |
| salt（盐） | 金钱产出 +20% | 1 | 临淄 |
| craftsmen（工匠） | 工匠产出 +30% | 2 | 彭城、南阳 |
| building_materials（建材） | 建材产出 +30%，建材消耗 -20% | 2 | 寿春、洛邑 |
| 无 | — | 29 | — |

**数据来源**：`cities.json` → `special_resource`，`balance_params.json` → `resources.special_resource_mod`

**特产机制说明**：
- 特产加成以偏移量形式叠加到城市产出计算中（`balance_params.json` → `resources.special_resource_mod`）
- 部分特产城拥有该资源的基础产出（`balance_params.json` → `resources.special_resources.*.city_base_production`），无需建造对应建筑即可产出
- 木材特产城的基础木材产出由城市基础产出提供（`balance_params.json` → `resources.city_base_wood` = 10）

### 2.5 城市地形规则

- 城市所在六角格**强制视为平原**（plains），不享受原地块地形的战斗修正（如森林防御 +20%、山地攻击 -10% 等）
- 城市及建筑**不影响**原地块地形效果（原地形的移动力消耗、视野等属性不变）
- 建筑辖区内的地形效果独立于城市本身（城墙/箭塔建在森林格上仍享受森林防御加成）

**升级费用公式**：

```
upgrade_cost = base_cost × upgrade_cost_multiplier ^ current_level
```

- `upgrade_cost_multiplier` = 1.5（所有建筑统一）
- 部分建筑高等级升级需额外建材（`upgrade_building_materials`）：城墙 lv3 需 30，箭塔 lv3 需 25，王宫 lv2 需 20、lv3 需 40

### 3.3 建造/升级/拆除规则

**建造验证**（7 项，`city_manager.gd:78-160`）：

1. 城市存在且有效
2. 建筑类型有效
3. 该城未建造此建筑（可重复建筑除外）
4. 不在建造队列中
5. 建筑槽位未满（城市等级决定上限）
6. 国家建筑数量上限未超（`max_national_count`）
7. 资源充足（gold + wood + building_materials）

**升级验证**（6 项）：

1. 城市存在且有效
2. 建筑已建造
3. 未达 max_level
4. 不在升级队列中
5. 资源充足（base_cost × 1.5^level）
6. 建材充足（如有 `upgrade_building_materials` 要求）

**拆除退款**（难度相关）：

| 难度 | 退款比例 |
|:---|:---:|
| 简单 | 75% |
| 普通 | 50% |
| 困难 | 25% |
| 地狱 | 0% |

**建造队列**：全城共用 1 个队列，每回合最多 1 个建筑在建/升级。

### 3.4 建筑辖区（六角格环）

- **辖区范围**：以城市为中心的 6 个相邻六角格（决策 #123）
- **辖区半径**：`cities.json` → `jurisdiction_radius`（默认 1）
- **建筑实体**：建筑建在辖区六角格上，有实体位置
- **建筑槽位**：城市等级决定可建建筑数量（1~5），每格最多 1 个建筑
- **城墙位置**：必须建在辖区格子上
- **船坞限制**：必须建在水军可通行的地形上（河流/深海/浅海/渡口），船坞建筑不阻碍水军通行

## 4. 人口机制

### 4.1 城市资源产出

人口是城市经济的基础单位。每人口产出粮食和金钱，同时消耗粮食。所有百分比 buff **相加**后统一乘算基础值。

**粮食产出**：

```
// 基础产出
food_base = population × 3    // balance_params.json → population.food_per_pop

// 百分比偏移叠加
food_offset = season_food_mod + farm_bonus + special_resource_mod + tech_mod + stability_offset

// 产出上限
food_gross = min(food_base × (1 + food_offset), max_food_production)

// 净产出（扣除自耗）
food_net = food_gross - population × 1    // balance_params.json → population.food_consume_per_pop
```

| 变量 | 说明 | 来源 |
|:---|:---|:---|
| population | 城市当前人口 | cities.json → initial_population（运行时状态） |
| food_per_pop | 人均粮食产出 | balance_params.json → population.food_per_pop（3） |
| food_consume_per_pop | 人均粮食自耗 | balance_params.json → population.food_consume_per_pop（1） |
| season_food_mod | 季节偏移 | balance_params.json → resources.season_food_mod（春+0.2/夏0/秋+1.0/冬-0.3） |
| farm_bonus | 农田加成 | buildings.json → farm.effects.food_production_bonus × 等级（+10%/级） |
| special_resource_mod | 特产偏移 | balance_params.json → resources.special_resource_mod（木材特产无粮食加成） |
| stability_offset | 安定度偏移 | 见 §5.3（高+0.2/中0/低-0.2/危机-0.5） |
| max_food_production | 最大粮食产出 | balance_params.json → city_levels.*.max_food_production + 粮仓加成 |

**金钱产出**：

```
// 基础产出
gold_base = population × 10    // balance_params.json → population.gold_per_pop

// 百分比偏移叠加
gold_offset = season_gold_mod + market_bonus + special_resource_mod + tech_mod + stability_offset

// 无上限
gold_net = gold_base × (1 + gold_offset)
```

| 变量 | 说明 | 来源 |
|:---|:---|:---|
| gold_per_pop | 人均金钱产出 | balance_params.json → population.gold_per_pop（10） |
| season_gold_mod | 季节偏移 | balance_params.json → resources.season_gold_mod |
| market_bonus | 市集加成 | buildings.json → market.effects.gold_production_bonus × 等级（+10%/级） |
| special_resource_mod | 特产偏移 | balance_params.json → resources.special_resource_mod（盐特产 +0.2） |

**木材产出**：

```
// 城市基础产出（所有城市）
wood_base = city_base_wood    // balance_params.json → resources.city_base_wood（10）

// 伐木场产出
wood_mill = lumbermill_production × level    // buildings.json → lumbermill.effects.wood_production（20/级）

// 百分比偏移叠加
wood_offset = season_wood_mod + special_resource_mod + tech_mod + stability_offset

// 特产城有基础产出（无需建筑）
wood_special = special_resource_mod.wood_production    // balance_params.json → resources.special_resource_mod

// 净产出
wood_net = (wood_base + wood_mill) × (1 + wood_offset)
```

| 变量 | 说明 | 来源 |
|:---|:---|:---|
| city_base_wood | 城市基础木材产出 | balance_params.json → resources.city_base_wood（10） |
| lumbermill_production | 伐木场每级产出 | buildings.json → lumbermill.effects.wood_production（20） |
| season_wood_mod | 季节偏移 | balance_params.json → resources.season_wood_mod |

**特殊资源产出**（马匹/精铁/工匠/建材）：

```
// 基础产出（特产城自动产出，无需建筑）
special_base = special_resources.*.city_base_production    // balance_params.json（2/回合）

// 建筑产出
special_building = building_production × level    // buildings.json → 对应建筑.effects

// 百分比偏移叠加
special_offset = season_mod + special_resource_mod + tech_mod + stability_offset

// 净产出
special_net = (special_base + special_building) × (1 + special_offset)
```

| 特产 | 基础产出 | 建筑来源 | 季节偏移 | 来源 |
|:---|:---:|:---|:---:|:---|
| 马匹 | 2 | 马厩（5/级） | — | balance_params.json → special_resources.horse |
| 精铁 | 2 | 精铁坊（3/级） | — | balance_params.json → special_resources.refined_iron |
| 工匠 | 2 | 工匠坊（3/级） | 秋+0.2/冬-0.2 | balance_params.json → special_resources.craftsmen |
| 建材 | 2 | 建材坊（3/级） | 春+0.2/冬-0.3 | balance_params.json → special_resources.building_materials |

**税收**（城市产出 → 国家资源池）：

```
// 税收效率 = 基础效率 + 民心档位修正 + 腐败修正 + 其他修正
tax_efficiency = base_efficiency + morale_tax_modifier + corruption_tax_effect + tech_bonus + minister_bonus + palace_bonus

// 粮食税
food_tax = food_net × tax_rate × tax_efficiency

// 金钱税
gold_tax = gold_net × tax_rate × tax_efficiency
```

- `tax_rate`：10%~50%，默认 30%（`balance_params.json` → `tax`）
- `base_efficiency`：默认 1.0（`balance_params.json` → `tax.current_efficiency`）
- `morale_tax_modifier`：民心高档位 +0.2 / 中 0 / 低 -0.2 / 危机 -0.5（`balance_params.json` → `morale.high_tax_bonus` / `low_tax_penalty` / `critical_tax_penalty`）
- `corruption_tax_effect`：清廉最多 +0.10 / 正常 0 / 腐败最多 -0.06 / 严重腐败最多 -0.10（`balance_params.json` → `corruption`，详见经营系统.md §9.2）
- 详见经营系统.md §6.2 和 §9.2

**详细产出公式**：参见经营系统.md §2 和 §5。

### 4.2 人口增长（进度条制）

人口增长采用进度条制，每回合推进度，满 100% 时人口 +1（决策 #92）。

```
// 人口增长周期（回合/人）
growth_period = 6.67 / (food_surplus_per_pop / 2.0) / season_mod / stability_mod

// 每回合进度增量
growth_progress += 100.0 / growth_period

// 进度满 100 时
if growth_progress >= 100.0:
    population += 1
    growth_progress -= 100.0
```

| 修正项 | 公式 | 值域 | 来源 |
|:---|:---|:---:|:---|
| 粮食盈余 | surplus_per_pop / 2.0 | 0.5~2.0+ | 盈余 2→1.0x，盈余 3→1.5x |
| 季节 | spring 1.0 / summer 1.0 / autumn 1.2 / winter 0.7 | 0.7~1.2 | `balance_params.json` → `population.growth_season_mod` |
| 安定度 | 0.5 + stability / 100.0 | 0.5~1.5 | `balance_params.json` → `population.growth_stability_formula` |

**数据来源**：`balance_params.json` → `population`

## 5. 城市安定度

### 5.1 安定度阈值

城市安定度（0~100）分为 4 档，直接影响该城资源产出（决策 #75）：

| 档位 | 安定度范围 | 产出偏移 | 来源 |
|:---|:---:|:---:|:---|
| 高 | ≥ 80 | +20% | `balance_params.json` → `stability.threshold_high` / `high_output_bonus` |
| 中 | 50~79 | 0% | `balance_params.json` → `stability.threshold_mid` / `mid_output_penalty` |
| 低 | 30~49 | -20% | `balance_params.json` → `stability.threshold_low` / `low_output_penalty` |
| 危机 | < 30 | -50% | `balance_params.json` → `stability.threshold_critical` / `critical_output_penalty` |

**初始安定度**：50（`balance_params.json` → `stability.initial`）

### 5.2 安定度修正来源

| 修正来源 | 效果 | 来源 |
|:---|:---:|:---|
| 祠堂 | +9 | `balance_params.json` → `stability.building_effect.shrine` |
| 学宫 | +6 | `balance_params.json` → `stability.building_effect.academy` |
| 王宫 | +15 | `balance_params.json` → `stability.building_effect.palace` |
| 驻军 | +6/单位 | `balance_params.json` → `stability.garrison_effect_per_unit` |
| 文大夫安民 | +安民% | 大夫系统 §13 |
| 文化不匹配 | -15 | `balance_params.json` → `stability.culture_mismatch_penalty` |
| 城市被占领 | -30 | `balance_params.json` → `stability.city_captured_penalty` |
| 低民心传导 | -3/回合（民心 < 30 时） | `balance_params.json` → `stability.low_morale_stability_penalty` |
| 战争恢复 | +3/回合 | `balance_params.json` → `stability.war_recovery_per_turn` |

**安定度公式**（每回合计算）：

```
stability += building_bonus + garrison_bonus + minister_bonus
stability += culture_mismatch + captured_penalty
if morale < 30:
    stability += low_morale_penalty    // -3/回合
if at_war:
    stability += war_recovery           // +3/回合
stability = clamp(stability, 0, 100)
```

### 5.3 安定度对产出的影响

安定度通过偏移量影响城市所有资源产出：

```
output = base_output × (1 + stability_output_offset)
```

| 安定度档位 | 偏移量 |
|:---|:---:|
| 高（≥ 80） | +0.20 |
| 中（50~79） | 0.00 |
| 低（30~49） | -0.20 |
| 危机（< 30） | -0.50 |

### 5.4 叛乱/起义判定

低安定度触发叛乱和起义风险（决策 #75）。叛乱和起义的结果均为**城市变为中立城市**（`neutral`），建筑保留。

| 条件 | 概率 | 效果 | 来源 |
|:---|:---:|:---|:---|
| 安定度 < 30（低） | 15%/回合 | 叛乱：城市变为中立 | `balance_params.json` → `stability.low_revolt_chance` |
| 安定度 < 10（危机） | 30%/回合 | 起义：城市变为中立 | `balance_params.json` → `stability.critical_uprising_chance` |

## 6. 城市文化

### 6.1 文化扩散模型（邻格扩散）

每座城市拥有文化值分布，文化从高浓度城市向低浓度邻城扩散（决策 #68）。

```
// 文化扩散计算
spread_amount = source_culture_value × spread_rate × distance_decay

// 基础扩散率
spread_rate = 0.05    // 5%/回合

// 距离衰减（六角格距离）
distance_decay = {
    1: 1.0,    // 相邻格 100%
    2: 0.7,    // 2 格 70%
    3: 0.4,    // 3 格 40%
    4: 0.2     // 4 格 20%
}
```

| 参数 | 值 | 来源 |
|:---|:---:|:---|
| 基础扩散率 | 0.05 | `balance_params.json` → `culture.spread_rate` |
| 距离衰减 | 1.0 / 0.7 / 0.4 / 0.2 | `balance_params.json` → `culture.distance_decay` |

**传播途径**（5 种）：
1. 邻城自然扩散（基础途径）
2. 贸易路线加速
3. 军事占领强制传播
4. 学宫建筑主动输出
5. 奇观全国加成

### 6.2 主流文化判定

当某文化在城市中的占比超过 50% 时，该文化成为该城的主流文化。

```
mainstream_culture = argmax(culture_values) if max(culture_values) / sum(culture_values) > 0.5
```

**数据来源**：`balance_params.json` → `culture.flip_threshold`

### 6.3 文化不匹配惩罚

当城市主流文化与所属势力文化不一致时：

| 效果 | 值 | 来源 |
|:---|:---:|:---|
| 安定度惩罚 | -15 | `balance_params.json` → `stability.culture_mismatch_penalty` |

### 6.4 文化翻转

当非归属势力的文化在城市中占比超过 50% 时，城市可能叛离至该文化所属势力。

- 翻转阈值：50%（`balance_params.json` → `culture.flip_threshold`）
- 翻转后城市归属变更，建筑保留

### 6.5 文化胜利条件

通过文化影响力覆盖全图城市可获得文化胜利（决策 #68）：

| 条件 | 值 | 来源 |
|:---|:---:|:---|
| 城市覆盖比例 | ≥ 70% | `balance_params.json` → `culture.victory_ratio` |
| 维持回合数 | 10 回合 | `balance_params.json` → `culture.victory_turns` |

## 8. 大夫驻城

### 8.1 文大夫效果

文大夫驻扎城市时，通过两个百分比属性加成城市（决策 #101~#111）：

```
// 金钱产出加成（乘算）
city_gold_output *= (1 + 理财% / 100)

// 安定度加成（加算）
city_stability += 安民%
```

| 属性 | 效果 | 说明 |
|:---|:---|:---|
| 理财% | 金钱产出乘算 | 5~100%，档次越高属性越高 |
| 安民% | 安定度加算 | 5~100%，直接加到安定度值上 |

**大夫档次**：

| 档次 | 属性范围 | 技能槽数 | 概率 |
|:---|:---:|:---:|:---:|
| 普通 | 5~30 | 1 | 60% |
| 稀有 | 15~60 | 2 | 30% |
| 传说 | 30~100 | 3 | 10% |

**获取途径**：学院每 8 回合产出 1 个文大夫；战斗胜利 15%~25% 概率获得；随机事件触发。

### 8.2 大夫俘虏/死亡

城市被攻陷时，驻扎的文大夫触发命运判定（决策 #101~#111）：

| 结果 | 概率 | 效果 |
|:---|:---:|:---|
| 死亡 | 30% | 大夫永久移除 |
| 被俘 | 40% | 可赎回（等级 × 100 金）或谈判要回 |
| 逃脱 | 30% | 返回己方其他城市 |

被俘未赎回时：40% 自杀 / 60% 归顺对方。

**详细机制**：参见游戏机制总览.md §13 大夫系统。
