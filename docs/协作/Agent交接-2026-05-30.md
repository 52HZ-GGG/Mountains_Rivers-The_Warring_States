## 1. Latest diagnosis for missing unit sprites
This is important: the problem is no longer believed to be just "missing `unit_` fallback paths".

The more likely root cause is **mixed unit ID naming conventions**.

The project currently mixes two styles:
- bare IDs used by data/runtime, for example: `infantry`, `archer`, `cavalry`
- prefixed IDs used in some scripts/default exports, for example: `unit_infantry`

The actual asset directories are named like this:
- `assets/sprites/units/base/unit_infantry`
- `assets/sprites/units/base/unit_archer`
- `assets/sprites/units/base/unit_cavalry`

That means:
- if the incoming ID is `infantry`, it should eventually resolve to `unit_infantry`
- if the incoming ID is already `unit_infantry`, the code must **not** generate `unit_unit_infantry`

### Confirmed facts
- `data/units.json` uses bare IDs such as `infantry`, `archer`, `cavalry`
- `data/tactical_skirmish_mvp.json` also uses bare `unit_type_id` values such as `infantry`
- `scripts/units/unit.gd` currently has a default export value of `@export var unit_type: String = "unit_infantry"`

### Correct fix direction
Do not keep adding more raw path fallbacks.
Normalize the unit ID first, then build path candidates from the normalized value.

Relevant files:
- `scripts/units/unit.gd`
- `scenes/ui/skirmish/skirmish_mvp_panel.gd`

Recommended helper:
```gdscript
func _normalized_unit_id(unit_id: String) -> String:
	return unit_id.trim_prefix("unit_")
```

Recommended derived values:
- original value: `unit_type` / `unit_type_id`
- normalized bare ID: `normalized_id`
- normalized directory name: `unit_dir = "unit_%s" % normalized_id`

Recommended candidate order:
1. faction/original
2. faction/normalized `unit_dir`
3. base/original
4. base/normalized `unit_dir`

Important:
- do **not** keep writing `"unit_%s" % unit_type`
- if `unit_type` is already `unit_infantry`, that produces the broken path `unit_unit_infantry`

## 2. Diplomacy panel extra background
This is the same class of issue as the event popup and city panel.

Relevant file:
- `scenes/ui/diplomacy/diplomacy_panel.gd`

Root cause:
- `_build_ui()` creates a fullscreen runtime `TextureRect` background
- it uses `SkirmishTileTextures.panel_texture("diplomacy")`
- it uses `TextureRect.STRETCH_KEEP_ASPECT_COVERED`
- this causes an extra stacked background image

The block that should be removed is:
```gdscript
var bg_tex: Texture2D = SkirmishTileTextures.panel_texture("diplomacy")
if bg_tex != null:
	var bg := TextureRect.new()
	bg.name = "Background"
	bg.texture = bg_tex
	bg.stretch_mode = TextureRect.STRETCH_KEEP_ASPECT_COVERED
	bg.set_anchors_preset(Control.PRESET_FULL_RECT)
	bg.mouse_filter = Control.MOUSE_FILTER_IGNORE
	add_child(bg)
```
