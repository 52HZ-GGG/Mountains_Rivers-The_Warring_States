# 《山河策》开发看板

> 用途：记录当前阶段程序、美术、策划的统一状态。更新时只改状态和下一步，不重写整份文档。
> 更新口径：以当前仓库内代码、资源和数据为准；若文档与实现不一致，以代码现状为准并记录差异。

## 1. 状态说明

| 状态 | 含义 |
|---|---|
| 已完成 | 已落地到仓库，可运行或可查看 |
| 待验收 | 已交付或已接入，但还没做最终检查 |
| 待接入 | 文件或设计已到位，程序还没接 |
| 待交付 | 已明确需求，等待策划或美术输出 |
| 待定稿 | 设计仍未冻结，程序不应直接实现 |
| 暂缓 | 不在当前阶段优先级内 |

| P0 | 补齐缺失资源清单并指定负责人 | 不然程序和美术继续错位 |

| P1 | 补齐深海/浅海与冻土相关资源接入 | 现有资源和地图系统可顺手收尾 |

| P2 | 整理旧引用与历史遗留文件 | 减少目录噪音 |
