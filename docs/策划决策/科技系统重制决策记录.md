# 科技系统重制决策记录

> **日期**：2026-05-16
> **版本**：v2.0
> **状态**：✅ JSON 数据已落地，⏳ 程序逻辑待实现

---

## 1. 背景与动机

原科技系统（v1.0）结构较为传统：66 项科技，4 类 ×3 时代，纯金币成本，线性前置链，无负面效果，无深度养成。缺乏策略深度和历史沉浸感。

## 2. 设计决策

### 2.1 八大新机制

| 机制 | 决策 | 理由 |
|:---|:---|:---|
| 互斥科技路线 | 硬锁定，按政策方向分组 | 增加路线抉择重量感，模拟战国各国不同的治国方略 |
| 多元研究资源 | 现有 7 种资源 + 新增帛书 | 将科技研究与资源经营深度绑定 |
| 科技事件 | 研究中随机触发 | 增加研究过程的叙事性和不确定性 |
| 奇观垄断 | 特定奇观解锁特定科技 | 奇观成为科技路线的关键节点 |
| 负面效果 | 部分科技有代价 | 增加决策张力，非纯正收益 |
| 科技协同 | 研究后揭示 | 鼓励探索不同科技组合 |
| 精通等级 | 全局精通，可升级 | 给予"精研一技"vs"博采众长"的策略选择 |
| 科技百科 | 全部 72 项填写完整知识卡片 | 增加文化教育价值 |

### 2.2 关键设计选择

- **互斥分组标准**：按政策方向（治国之道/选贤之制/刑德之辩/拓土方略/文化方略），而非按学派
- **帛书产出途径**：仅通过藏书阁建筑产出，无特产城加成
- **精通范围**：全局精通（国家级），非局部精通
- **协同可见性**：研究后才揭示，增加探索感
- **百科定位**：先做纯展示，后续考虑机制联动

## 4. Schema 变更

### 4.1 tech_tree.json（1.0 → 2.0）

顶层新增：
- `mutual_exclusion_groups`：互斥组定义数组
- `mastery_global_config`：精通全局配置

每项科技新增字段（全部可选）：
- `cost_resources`：多元资源成本（覆盖 cost_gold）
- `mutual_exclusion_group`：互斥分组 ID
- `malus_effects`：负面效果数组
- `mastery`：精通等级配置
- `knowledge_card`：科技百科内容（全部 72 项必填）
- `requires_wonder`：奇观垄断前置
- `research_events`：研究事件 ID 列表

### 4.2 balance_params.json

tech 节点新增 13 个参数（mastery/research_event/mutual_exclusion/cost_resources/knowledge_card/malus 相关）。

resources.special_resources 新增 silk_books 定义。
