## 决策 #176：建筑升级机制重构 —— 每级独立效果与独立成本

### 176.1 背景

现有建筑升级采用统一公式：`升级成本 = base_cost × 1.5^level`，效果 = `effect_value × level` 线性增长。这导致：
1. 所有建筑增长曲线相同，无法差异化设计
2. 高等级建筑的效果类型无法扩展（如 1 级只有粮食加成，3 级额外获得安定度加成）
3. 升级成本无法按等级差异化（如城墙 lv3 需建材，但 lv2 不需要）

### 176.2 新方案：`levels` 数组

**核心变更**：移除 `effects`、`upgrade_cost_multiplier`、`upgrade_building_materials` 字段，改为 `levels` 数组，每个元素定义该等级的完整属性。

**JSON Schema**：

```json
{
  "id": "farm",
  "name": "农田",
  "description": "开垦荒地，增产粮食",
  "category": "economic",
  "max_level": 3,
  "upkeep_gold": 5,
  "max_national_count": null,
  "build_requires": null,
  "levels": [
    {
      "cost_gold": 100,
      "cost_wood": 0,
      "build_turns": 1,
      "effects": {
        "food_production_bonus": 0.10
      }
    },
    {
      "cost_gold": 150,
      "cost_wood": 0,
      "build_turns": 1,
      "effects": {
        "food_production_bonus": 0.25
      }
    },
    {
      "cost_gold": 225,
      "cost_wood": 0,
      "build_turns": 2,
      "effects": {
        "food_production_bonus": 0.45
      }
    }
  ]
}
```

**字段说明**：
- `levels[0]` = 从无到 lv1 的建造成本和效果
- `levels[1]` = 从 lv1 升到 lv2 的成本和效果（效果是该等级的**完整效果**，不是增量）
- `levels[n]` 的 `effects` 字典结构与现有 `effects` 完全一致，程序读取时直接取当前等级对应的 effects 字典
- `cost_gold` / `cost_wood` / `build_turns` 为该级升级的独立成本
- `cost_building_materials`（可选）为该级升级额外需要的建材

**程序影响**：
- `can_build()` → 读取 `levels[0]` 的成本进行校验
- `start_build()` → 扣除 `levels[0]` 的成本
- `can_upgrade()` → 读取 `levels[current_level]` 的成本进行校验（current_level 为 0-based 下一级索引）
- `start_upgrade()` → 扣除 `levels[current_level]` 的成本
- `_calculate_upgrade_cost()` → 不再使用 `upgrade_cost_multiplier` 公式，直接读取 levels 数组
- `get_city_production()` → 读取 `levels[current_level - 1].effects`（已建等级的 effects）

**兼容性**：旧格式（`effects` + `upgrade_cost_multiplier`）的建筑仍可被 DataManager 正常加载，但所有建筑将统一迁移到新格式。

### 176.3 设计意图

- 支持递增曲线（如粮食加成：10% → 25% → 45%）
- 支持效果类型扩展（如 lv1 仅粮食加成，lv3 额外获得安定度加成）
- 支持成本差异化（如 lv3 需建材，lv1/lv2 不需要）
- 每个建筑的每个等级都是独立可调的"策划旋钮"

## 决策 #177：建筑拆除机制重构

### 177.1 背景

现有拆除为瞬时操作，无时间成本。这使得建筑决策缺乏战略深度——玩家可以随时拆除换退款，没有沉没成本。

### 177.2 新方案

| 属性 | 旧值 | 新值 |
|:---|:---|:---|
| 拆除耗时 | 即时 | 1 回合（占用建筑队列） |
| 退款 | 按难度 75%/50%/25%/0% | 无退款 |
| 效果停止时机 | 立即 | 拆除完成时（1 回合后） |
| 队列冲突 | 不占用队列 | 与建造/升级共用队列，拆除期间不能对该建筑进行其他操作 |

**程序变更**：
- `demolish()` 改为 `start_demolish()`：校验建筑已建 → 加入 build_queue（标记 `is_demolish: true`）→ 不扣资源、不退资源
- `process_turn()` 处理 build_queue 时，识别 `is_demolish` 条目 → 倒计时归零时从 buildings 中移除
- 拆除期间建筑效果**持续生效**（直到拆除完成）
- 拆除中的建筑不能被升级（已在队列中）
- 拆除中的建筑不能被取消（与 cancel_build 区分：cancel_build 仅取消建造/升级）

**新增字段**：
- `build_queue` 条目新增 `is_demolish: bool`（默认 false）
- `balance_params.json` → `city_combat.demolish_turns` = 1

### 177.3 设计意图

- 拆除有时间成本（1 回合），不能在紧急时刻临时拆建筑换资源
- 无退款，拆除是纯粹的战略权衡（腾槽位 vs 损失已投入资源）
- 效果持续到拆除完成，避免"拆了农田立刻饥荒"的反直觉情况

## 决策 #180：建筑效果计算程序变更

### 180.1 产出计算变更

`get_city_production()` 修改逻辑：

```
旧：for effects → prod[key] += effects[key] × level
新：prod[key] += levels[level - 1].effects[key]（直接取当前等级的完整效果值）
```

### 180.2 升级计算变更

`_calculate_upgrade_cost()` 修改逻辑：

```
旧：cost = base_cost × multiplier^current_level
新：cost = levels[current_level].cost_gold / cost_wood
```

### 180.3 拆除流程变更

```
旧：demolish() → 立即移除 + 退款
新：start_demolish() → 加入 build_queue（is_demolish=true）
    process_turn() → is_demolish 条目倒计时归零 → 移除建筑（不退款）
```

### 180.4 新增安定度建筑效果

在安定度计算中新增建筑加成：

```
stability += building_effect.shrine      // 9（已有）
stability += building_effect.academy     // 6（已有）
stability += building_effect.palace      // 15（已有）
stability += building_effect.altar       // 5×level（新增）
stability += building_effect.prison      // 3×level（新增）
```

## 决策 #181：建筑分类扩展为 5 类 + 养马场

### 181.1 背景

原建筑分为 4 类（经济/军事/政治/特殊），城墙和箭塔归入军事类。但城墙、箭塔、瓮城的功能（防御血量、城防加成）与兵营、马厩等生产类军事建筑差异显著，混在一起不利于 UI 分类展示和玩家理解。

### 181.2 新方案

**5 类建筑体系**：

| 类别 | 数量 | 说明 |
|:---|:---:|:---|
| 经济 | 10 | 资源产出、存储、贸易 |
| 军事 | 7 | 兵种生产、训练、视野 |
| 防御 | 3 | 城防血量、防御加成（城墙/箭塔/瓮城） |
| 政治 | 6 | 民心、安定度、文化、腐败 |
| 特殊 | 1 | 王宫 |

**防御类建筑**（从 military 移出）：
- wall（城墙）
- arrow_tower（箭塔）
- inner_gate（瓮城）

### 181.3 新增建筑：养马场（horse_farm）

| 属性 | 值 |
|:---|:---|
| 类别 | economic |
| 效果 | lv1 +5 / lv2 +12 / lv3 +20 马匹产出 |
| max_level | 3 |
| 维护费 | 8 |
| 全国限建 | 3 座 |
| 建造前置 | 需马匹特产城市 |

**设计意图**：原马厩同时承担"产马匹资源"和"产骑兵单位"两个职能，拆分后养马场负责资源产出，马厩专注兵种解锁，职责清晰。

## 决策 #182：军事生产建筑改造 —— 等级解锁兵种

### 182.1 背景

原军事建筑的效果为简单的数值加成（如兵营 recruit_discount +10%/级），produces_category 仅标记可生产的兵种类别，未区分同类别内的高低级兵种。玩家只需建 1 级兵营就能生产所有步兵，缺乏升级动力。

### 182.2 新方案：建筑等级解锁兵种

**双重门控**：生产一个兵种需要同时满足：
1. 对应科技已解锁（tech_tree 中的 unlock_unit）
2. 对应建筑达到要求等级

**5 种生产建筑的兵种解锁表**：

**兵营（barracks）— 步兵类**：

| 等级 | 解锁兵种 | 建造成本 |
|:---:|:---|:---|
| 1 | 民兵、步兵 | 150 金 + 30 木 |
| 2 | +枪刺兵、斥候小队 | 225 金 + 45 木 |
| 3 | +铁甲兵 | 338 金 + 68 木 |

**马厩（stable）— 骑兵类**：

| 等级 | 解锁兵种 | 建造成本 |
|:---:|:---|:---|
| 1 | 斥候骑兵、战车 | 150 金 + 20 木 |

**工坊（workshop）— 攻城类**：

| 等级 | 解锁兵种 | 建造成本 |
|:---:|:---|:---|
| 1 | 冲车 | 200 金 + 40 木 |
| 2 | +投石车 | 300 金 + 60 木 |
| 3 | +弩炮 | 450 金 + 90 木 |

**船坞（dock）— 水军类**：

| 等级 | 解锁兵种 | 建造成本 |
|:---:|:---|:---|
| 1 | 蒙冲 | 180 金 + 35 木 |
| 2 | +大翼 | 270 金 + 53 木 |

**射营（archery_camp）— 远程类**：

|:---:|:---|:---|
| 1 | 弓兵 | 120 金 + 25 木 |
| 2 | +弩兵 | 180 金 + 38 木 |
| 3 | +弓骑兵 | 270 金 + 57 木 |

### 182.3 JSON 效果格式

```json
{
  "effects": {
    "unlock_infantry": ["militia", "infantry", "spear", "scout_team", "iron_armored"]
  }
}
```

每个等级的 `unlock_*` 数组列出该等级**可生产的全部兵种**（包含低等级可生产的），程序取当前等级对应的数组作为可用兵种列表。

### 182.4 城市默认可生产兵种（按城市等级）

无生产建筑时，城市按等级可生产基础兵种（决策 #127）：

| 城市等级 | 可生产兵种 |
|:---:|:---|
| 1 村 | 民兵 |
| 2 镇 | 民兵、斥候小队 |
| 3 城 | 民兵、斥候小队、步兵 |
| 4 府 | 民兵、斥候小队、步兵、枪刺兵、弓兵 |
| 5 都 | 民兵、斥候小队、步兵、枪刺兵、弓兵、冲车 |

高级兵种（铁甲兵、骑兵类、弩兵、弓骑兵、投石车、楼船等）必须有对应建筑才能生产。

### 182.5 兵种来源汇总

| 来源 | 可生产兵种类别 |
|:---|:---|
| 城市（无建筑） | 民兵（militia） |
| 兵营 | 步兵类 |

| 工坊 | 攻城类 |

征兵机制中，士兵来源即以上 6 种途径。

### 182.6 马厩变更

马厩不再产出马匹资源（马匹产出移至养马场），专职为骑兵兵种解锁建筑。建造前置仍需马匹特产城市。

### 182.7 练兵场保留

练兵场（training_ground）保留，效果调整为 `garrison_atk_bonus` + `garrison_def_bonus`（驻军攻防加成），移除 `recruit_discount`。

### 182.8 射营（archery_camp）新增

| 属性 | 值 |
|:---|:---|
| 类别 | military |
| 效果 | 3 级解锁弓兵/弩兵/弓骑兵 |
| max_level | 3 |
| 维护费 | 10 |
| 建造前置 | 无 |

## 决策 #183：奇观系统全面重构

### 183.1 背景

原有奇观系统存在以下问题：
1. 效果键与建筑系统不兼容（`recruit_speed_bonus`、`research_speed_bonus` 在建筑系统中未定义）
2. 奇观建造机制与普通建筑无差异（无全局唯一约束、占用建筑槽位）
3. 缺少奇观竞赛机制（所有国家可同时建造同一奇观）
4. 奇观种类不足（仅6座，未覆盖所有机制领域）

### 183.2 奇观分类

奇观分为两类：

| 类型 | 说明 | 建造方式 |
|:---|:---|:---|
| **单城奇观** | 在城市内建造，效果全国生效 | 在城市建筑面板中建造 |

### 183.3 奇观竞赛机制

- **全局唯一**：每种奇观全世界只能存在1座
- **竞赛机制**：所有国家可同时建造同一奇观，先建成者获得
- **失败返还**：被抢建后100%返还金、木、建材
- **不占槽位**：奇观不占用城市建筑槽位

### 183.4 建造时间

- 单城奇观：统一20回合
- 地图奇观：按长度计算（每格2回合）
