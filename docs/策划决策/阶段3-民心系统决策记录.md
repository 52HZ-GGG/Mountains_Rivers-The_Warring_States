## 关键决策

### 决策 71：民心采用双层制（国家级民心 + 城市级安定度）

**选定方案**：
- 国家级：民心（0~100），影响全国税收效率、征兵速度、战斗修正
- 城市级：安定度（0~100），影响该城市产出修正、叛乱判定
- 最终城市产出 = 基础产出 × 民心修正 × 安定度修正

**理由**：
- 原国家级单一民心无法体现建筑/城市差异（建筑按城市配置，但民心是全国共享）
- 双层制让「建祠堂稳民心」的策略在具体城市上生效，增加决策深度
- 国家级民心管宏观（税率、季节、战争），城市级安定度管微观（建筑、驻军、文化渗透）

**已落地**：`docs/decisions/阶段3-民心系统决策记录.md`

### 决策 72：国家级民心公式扩展

**选定方案**：
- 公式：`民心变化 = base_drift + tax_effect + season_effect + school_base + wonder_effect + tech_effect + event_effect + war_weariness + victory_bonus`
- 初始值：50

**各项明细**：

| 项 | 公式 | 默认值 |
|:---|:---|:---|
| base_drift | +1 | 每回合自然回升 |
| tax_effect | 固定值（见税率表） | 税率30%为基准 |
| season_effect | 春+5/夏-5/秋+10/冬-10 | 季节修正 |
| school_base | 学派基础修正/回合 | 儒家+2，法家-1，其余0 |
| wonder_effect | 奇观民心加成 | 兵马俑+10，章华台+15（全国） |
| tech_effect | 科技民心加成 | 6个科技节点累加 |
| event_effect | 事件指定值 | 随机事件 |
| war_weariness | 战争超20回合后每回合-2 | 厌战情绪 |
| victory_bonus | 占领首都+5，持续3回合 | 胜利激励 |

**税率对民心的影响（固定值）**：

| 税率 | 民心影响/回合 |
|:---|:---|
| 10% | +15 |
| 20% | +5 |
| 30% | 0 |
| 40% | -10 |
| 50% | -20 |

**战后恢复**：战争结束后，厌战debuff每回合+2恢复，直到恢复原值。

**理由**：
- 在原有公式基础上新增 war_weariness、victory_bonus、wonder_effect、tech_effect
- 厌战门槛设为20回合（较长），避免小规模冲突过度惩罚
- 胜利激励仅在占领首都时触发（稀缺），+5持续3回合（有限）
- 战后恢复与厌战对称（-2/+2），形成自然消退
- 税率采用固定值而非倍率，简单直接
- 奇观和科技民心加成生效，让已有数据发挥作用

**已落地**：`docs/数值公式.md` §3（需更新）

### 决策 73：民心阈值效果

**选定方案**：

| 民心范围 | 税收效率 | 征兵速度 | 其他效果 |
|:---|:---|:---|:---|
| >80 | +20% | +30% | 暴动概率0%，所有城市安定度+1/回合 |
| 50~80 | 正常 | 正常 | 无 |
| 30~50 | -20% | -20% | 无 |
| <30 | -50% | -50% | 逃兵+10%/回合，每回合城市安定度-3 |

**理由**：
- 阈值效果已在 balance_params.json 中定义，保持不变
- 高民心（>80）新增安定度+1/回合，奖励稳定治理
- 低民心（<30）征兵速度-50%，体现民心低落时百姓不愿从军
- 低民心（<30）对城市安定度的传导：每回合-3

**已落地**：`data/balance_params.json` morale 段

### 决策 74：民心战斗修正沿用现有公式

**选定方案**：
```
morale_atk_mod = 0.7 + 0.3 × (民心 / 100)
```
0民心=70%战力，100民心=100%战力。

**理由**：
- 公式已在数值公式.md中定义，battle_params.json中有morale_atk_floor/ceil参数
- 保持不变，不增加额外复杂度

**已落地**：`docs/数值公式.md` §1

### 决策 75：城市安定度公式（影响值放大3倍）

**选定方案**：
- 初始值：50
- 公式：`安定度变化 = building_effect + garrison_effect + culture_effect + morale_drift + high_morale_bonus + city_captured + war_recovery + event_effect`
- 每回合自动计算，clamp 到 [0, 100]

**各项明细**：

| 项 | 公式 | 默认值 |
|:---|:---|:---|
| building_effect | 祠堂+9/级，学宫+6/级，王宫+15/级 | 建筑加成（累加） |
| garrison_effect | 驻军每队+6 | 无驻军=0（不下降） |
| culture_effect | 主流文化≠归属国时-15 | 文化渗透定值 |
| morale_drift | 民心<30时每回合-3 | 低民心传导 |
| high_morale_bonus | 民心>80时+1/回合 | 高民心奖励 |
| city_captured | 攻占时-30 | 占领惩罚 |
| war_recovery | 停战后+3/回合 | 战后恢复 |
| event_effect | 事件指定值 | 随机事件（暂不实现） |

**理由**：
- 安定度0~100范围较大，放大3倍让玩家感受到建筑和驻军的价值
- 无驻军时安定度不变（不惩罚），驻军是正向加成而非维持
- 文化渗透给固定-15而非持续累加
- 高民心奖励+1/回合，鼓励维持高民心
- 攻占城市-30，体现占领后的不稳定
- 停战后+3/回合恢复，让安定度逐步回升
- 事件影响预留接口，暂不实现

**已落地**：`data/balance_params.json` 新增 stability 段

### 决策 76：城市安定度阈值效果

**选定方案**：

| 安定度范围 | 城市产出 | 惩罚 |
|:---|:---|:---|
| >80 | +20% | 无 |
| 50~80 | 正常 | 无 |
| 30~50 | -20% | 征兵受阻 |
| <30 | -50% | 15%/回合民变，<10时20%/回合起义（城市叛变） |

**理由**：
- 与民心阈值设计对称，玩家容易理解
- 民变是概率事件（产出暂时归零），起义是确定性后果（城市叛变），形成阶梯惩罚
- 安定度<10时起义概率20%，给玩家最后的挽救窗口

**已落地**：`data/balance_params.json` stability 段

### 决策 77：税率系统设计（税收效率暂定1.00）

**选定方案**：
- 税率范围：10%~50%，默认30%
- 税收效率：暂定全部为 1.00，只留接口
- 交税包括金钱和粮食（双资源征收）
- 税率可每回合调整（UI滑块）

**税率对民心的影响（固定值）**：

| 税率 | 民心影响/回合 | 税收效率 |
|:---|:---|:---|
| 10% | +15 | 1.00（预留） |
| 20% | +5 | 1.00（预留） |
| 30% | 0 | 1.00（预留） |
| 40% | -10 | 1.00（预留） |
| 50% | -20 | 1.00（预留） |

**税收公式**：
```
实际金钱收入 = 基础金钱收入 × 税收效率
实际粮食收入 = 基础粮食收入 × 税收效率
```

（后续阶段可启用税收效率与税率的线性关系：`效率 = 税率 / 0.3`）

**理由**：
- 税收效率暂定1.00，避免税率系统与经济系统耦合过深
- 只留接口，后续阶段可轻松启用效率公式
- 税率对民心采用固定值，简单直接
- 交税包括钱和粮食，符合战国时期实物税的历史背景

**已落地**：`data/balance_params.json` 新增 tax 段

### 决策 78：学派对民心采用基础+政策双轨制

**选定方案**：
- 基础修正：每回合微量民心变化（如儒家+2/回合，法家-1/回合）
- 政策触发：特殊政策一次性大幅调整（如祭祀大典+15）

**各学派基础修正**：

| 学派 | 每回合民心修正 | 说明 |
|:---|:---|:---|
| 儒家 | +2 | 仁政教化，民心自然归附 |
| 法家 | -1 | 严刑峻法，民心略有抵触 |
| 墨家 | 0 | 兼爱非攻，中性 |
| 道家 | 0 | 无为而治，中性 |
| 兵家 | 0 | 重军事轻民生，中性 |
| 纵横家 | 0 | 重外交轻内政，中性 |

**政策触发**（已有数据，保持不变）：
- 儒家「祭祀大典」：cost_gold 500, 持续3回合, 民心+15
- 法家「什伍连坐」：cost_gold 300, 持续5回合, 民变概率-50%

**理由**：
- 基础修正体现学派的长期影响（儒家天然亲民，法家天然压民）
- 政策触发体现主动决策（花资源换民心）
- 双轨制让学派选择有持续影响而非仅靠政策

**已落地**：`data/schools.json`（需新增 school_base_morale 字段）
