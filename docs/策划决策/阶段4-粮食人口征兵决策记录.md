## 关键决策

### 决策 88：人口单位重构

**选定方案**：
- 完全废弃现有 `base_population`（3000-14000 历史尺度），改用 1 为基本单位
- `cities.json` 新增 `initial_population` 字段，废弃 `base_population`
- 初始人口分配：

| 城市等级 | 初始人口 | 数量 |
|:---|:---:|:---:|
| 小城（4槽） | 3 | 36 |
| 中城（5槽） | 5 | 7 |
| 大城（6槽） | 7 | 7（均为首都） |
| 首都加成 | 10 | 7 |

**理由**：
- 以 1 为基本单位简化计算，目标均值 15 人口/城
- 首都初始 10，距离上限 25 仍有增长空间，体现起步差异
- 后续可逐城微调 `initial_population` 值

---

### 决策 89：城市等级系统

**选定方案**：
- 新增 `city_level` 字段替代 `max_building_slots`
- 等级定义：

| city_level | 名称 | 建筑槽位 | max_population | max_food_production |
|:---:|:---|:---:|:---:|:---:|
| 1 | 小城 | 2 | 10 | 30 |
| 2 | 中城 | 3 | 15 | 45 |
| 3 | 大城 | 4 | 20 | 60 |
| 首都 | 首都 | 5 | 25 | 75 |

**理由**：
- 建筑槽位减少（原4-6→现2-5）增加建筑取舍的策略深度
- max_population 和 max_food_production 作为双上限，制约城市过度膨胀
- 首都额外+1槽位和更高上限，体现核心城市地位

---

### 决策 90：人口经济产出

**选定方案**：
- 每人口每回合基础产出：粮食 3、金钱 10
- 每人口每回合自耗：粮食 1
- 修正链：

```
粮食：base(3) × season × (1 + farm_bonus) × special_mod → 上限 max_food_production
金钱：base(10) × season × (1 + market_bonus) × special_mod × tech_mod → 无上限
```

| 修正项 | 粮食 | 金钱 |
|:---|:---|:---|
| 季节 | 春1.2/夏1.0/秋2.0/冬0.7 | 春1.0/夏1.0/秋1.5/冬0.8 |
| 建筑 | 农田每级+10% | 市集每级+10% |
| 特产 | 铁矿城无直接效果 | 盐池城×1.2 |
| 科技 | — | 科技树效果累加 |
| 上限 | max_food_production（硬上限） | 无 |

**理由**：
- 人均产出模型替代原固定基础值（city_base_food=100），让人口成为核心经济驱动力
- 粮食有硬上限防止人口无限产出粮食，金钱无上限鼓励经济发展
- 保留季节/建筑/特产修正链，与现有系统兼容

---

### 决策 91：税收与储粮

**选定方案**：
- 每人剩余粮食 = 产粮(3) - 自耗(1) = 2（基础值，修正后可能不同）
- 税收 = 剩余 × 税率 → 全国储粮池
- 金钱全额按税率征收 → 全国金钱池
- 税率沿用现有 5 档制（决策 77）：

| 税率 | 民心影响/回合 |
|:---:|:---:|
| 10% | +15 |
| 20% | +5 |
| 30% | 0（默认） |
| 40% | -10 |
| 50% | -20 |

- 全国储粮上限 = 200（基础）+ 粮仓加成（每座粮仓每级+50）
- 城市无粮食池，只看当回合盈余

**理由**：
- 城市无粮食池简化管理，粮食盈余直接用于判定饥荒和计算人口增长
- 全国储粮池可累积，作为战争消耗的缓冲，有上限防止无限囤积
- 沿用现有税率避免重复设计

---

### 决策 92：人口增长公式

**选定方案**：
- 进度条制：每城市有增长进度（0-100%），每回合累加
- 基准速率：每人盈余粮食=2 时，每回合 15%（约 6.67 回合增长 1 人）

**公式**：
```
food_surplus_per_pop = city_food_net / population
growth_period = 6.67 / (food_surplus_per_pop / 2.0) / season_mod / stability_mod
growth_progress += 100 / growth_period   （每回合）
if growth_progress >= 100:
    population += 1
    growth_progress = 0
```

**修正系数**：

| 修正项 | 公式 | 值域 |
|:---|:---|:---|
| 粮食盈余 | surplus_per_pop / 2.0 | 盈余2→1.0x，盈余3→1.5x，盈余1→0.5x |
| 季节 | 春1.0/夏1.0/秋1.2/冬0.7 | 0.7~1.2 |
| 安定度 | 0.5 + stability/100 × 1.0 | 0.5~1.5（城市级安定度） |

**示例**：
- 10人口城市，产粮30（基准），自耗10，盈余20，每人盈余2 → period=6.67，每回合15%
- 秋季：period=6.67/1.2=5.56，每回合18%
- 冬季安定度30：period=6.67/0.7/0.8=11.9，每回合8.4%

**理由**：
- 进度条制比概率制更可预测，玩家能规划人口增长
- 粮食盈余作为核心调节因子，鼓励发展农业
- 季节和安定度提供额外调节维度

### 决策 93：饥荒与国家储粮不足

**选定方案**：

**城市饥荒**（当回合城市产粮 < 城市消耗）：
- 该城人口 -1/回合
- 该城安定度 -10/回合
- 判定依据：当回合净产出（非存量）

**国家储粮不足**（全国储粮 ≤ 0）：
- 民心 -5/回合（国家级民心，非城市安定度）
- 军队士气 -10/回合
- 军队攻击 -20%、防御 -20%

**理由**：
- 城市饥荒直接影响人口和安定度，形成恶性循环（人口减少→产出更低→更容易饥荒）
- 国家储粮不足影响全国民心和军队战斗力，迫使玩家重视粮食储备
- 攻防-20%的惩罚让缺粮状态下的战争极为不利，体现"兵马未动粮草先行"
- 使用民心（决策71-72的国家级变量）而非安定度（城市级），保持双层制一致性

### 决策 94：征兵池设计

**选定方案**：
- 最大征召人数 = 全国总人口 × 20%（固定上限）
- 约束：可服役人数 + 已服役人数 ≤ 最大征召人数
- 开启征兵后，每回合最大征召人数的 10% 进入可服役池
- 可服役池为全局资源，进入后可一回合全部消耗
- 征兵消耗：按 `units.json` 中兵种的 cost_gold/cost_food/cost_iron
- 达到约束上限后自动停止征兵

**理由**：
- 双池设计（可服役池+已服役）模拟兵源从民间到军队的流动
- 20%上限限制穷兵黩武，10%/回合填充速率让征兵需要时间
- 可服役池可一回合全部消耗，允许玩家在紧急时快速动员
- 约束条件防止征兵超过人口承受能力

---

### 决策 95：征兵衰减与恢复

**选定方案**：
- 阵亡 1 人 → 最大征召人数临时 -1
- 恢复速率：每 4 回合恢复 1 点最大征召人数
- 恢复不能超过总人口×20%的原始上限

**示例**：
- 总人口100，最大征召20人
- 征召20人，战死5人 → 最大征召降为15
- 需要 5×4=20 回合才能恢复到20

**理由**：
- 1:1衰减直接反映战争对人口的伤害
- 4回合恢复（=1年）体现人口恢复的缓慢性
- 激励玩家珍惜兵力，避免无意义消耗

---

### 决策 96：服役惩罚

**选定方案**：
- 阈值：服役比例 > 10% 总人口时触发
- 线性惩罚：每超 1% → 粮食产出 -2%、金钱产出 -2%、人口增长速率 -5%

**公式**：
```
service_ratio = active_service / total_population
if service_ratio > 0.10:
    penalty = service_ratio - 0.10
    food_production  *= (1 - penalty × 2.0)
    gold_production  *= (1 - penalty × 2.0)
    growth_rate      *= (1 - penalty × 5.0)
```

**示例**：

| 服役比例 | 粮食惩罚 | 金钱惩罚 | 增长惩罚 |
|:---:|:---:|:---:|:---:|
| 10% | 无 | 无 | 无 |
| 15% | -10% | -10% | -25% |
| 20% | -20% | -20% | -50% |

**理由**：
- 10%阈值允许小规模征兵无惩罚，鼓励初期扩军
- 增长惩罚系数（5.0）高于经济惩罚（2.0），体现劳动力流失对人口增长的更大影响
- 线性惩罚可预测，玩家能计算征兵的经济代价

---

### 决策 97：军队维护

**选定方案**：
- 粮食维护：按兵种 `upkeep_food`，从全国储粮扣除
- 金钱维护：统一 5 金/人/回合（暂定值，后续按兵种调整）
- 新增 `units.json` 字段 `upkeep_gold`（所有兵种统一=5）

**理由**：
- 粮食维护按兵种差异化（民兵2/步兵5/骑兵8/战车12），体现兵种消耗差异
- 金钱维护暂统一，避免初期过度复杂化
- 为后续按兵种调整预留字段

### 决策 98：建筑效果调整

**选定方案**：

| 建筑 | 原效果 | 新效果 | 分类变化 |
|:---|:---|:---|:---|
| 农田(farm) | +50 粮食/级 | +10% 城市粮食产出/级 | 不变（经济） |
| 市集(market) | +40 金钱/级 | +10% 城市金钱产出/级 | 不变（经济） |
| 粮仓(granary) | +200 粮食存储/级 | +5 max_food_production/级 + +50 全国储粮上限/级 | 政治→经济 |

**理由**：
- 百分比加成与人均产出模型匹配，比固定值更有成长性
- 粮仓双重效果（产粮上限+储粮上限）使其在新系统中仍有价值
- 粮仓改为经济类建筑，与农田/市集同属经济建筑组

### 决策 100：公式汇总

**城市每回合产出**：
```
food_base = population × 3
food_modified = min(food_base × season_food_mod × (1 + farm_bonus) × special_mod, max_food_production)
food_consumed = population × 1
food_net = food_modified - food_consumed          # 当回合盈余（城市无存量）
food_tax = food_net × tax_rate                    # → 全国储粮

gold_base = population × 10
gold_modified = gold_base × season_gold_mod × (1 + market_bonus) × special_mod × tech_mod
gold_tax = gold_modified × tax_rate               # → 全国金钱
```

**人口增长（进度条制）**：
```
food_surplus_per_pop = food_net / population
growth_period = 6.67 / (food_surplus_per_pop / 2.0) / season_mod / stability_mod
# 基准值：surplus=2, season=1.0, stability=1.0 → period=6.67, 每回合15%
growth_progress += 100 / growth_period   （每回合）
if growth_progress >= 100:
    population += 1
    growth_progress = 0
```

**全国储粮**：
```
national_grain += 各城 food_tax 之和 - 军队 upkeep_food 之和
national_grain_cap = 200 + Σ(粮仓等级 × 50)
national_grain = clamp(national_grain, 0, national_grain_cap)
```

**征兵**：
```
max_conscription = total_population × 0.20
available_pool += max_conscription × 0.10   （每回合，开启征兵时）
约束：available_pool + active_service ≤ max_conscription
征兵消耗：cost_gold/cost_food/cost_iron per unit type
```

**服役惩罚**：
```
service_ratio = active_service / total_population
if service_ratio > 0.10:
    penalty = service_ratio - 0.10
    food_production  *= (1 - penalty × 2.0)
    gold_production  *= (1 - penalty × 2.0)
    growth_rate      *= (1 - penalty × 5.0)
```

**饥荒与储粮不足**：
```
if city_food_net < 0:
    city.population -= 1
    city.stability  -= 10

if national_grain ≤ 0:
    morale             -= 5    # 国家级民心
    army_morale        -= 10
    army_attack        *= 0.80
    army_defense       *= 0.80
```
