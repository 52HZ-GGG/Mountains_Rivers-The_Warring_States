## 重制背景

### 现有系统问题

1. `factions.json` 中 `diplomacy_modifier` 字段全部为空 `{}`，从未使用 → 删除
2. `_apply_building_diplomacy_effects()` 是空函数（驿站/王宫/长城效果未接入）
3. `get_power_score()` 中 culture/tech 硬编码为 0
4. 宣战后自动断商路的逻辑未确认是否实现
5. 声望阈值中 20-30 段无单独效果
6. 游士外交动作只有文档设计，代码未实现
7. `difficulty` 字段错误放置在 `diplomacy.json` 中
8. 历史事件（合纵连横等）只消耗资源，不实际改变国家关系
9. 停战谈判条件类型不足，缺乏俘虏释放等战国特色条件

### 重制目标

- 整理现有外交系统全部内容，形成完整的决策记录
- 填补已确认的缺口（建筑效果、power_score、商路断开等）
- 新增三种战国特色机制：质子、纵横家、朝贡册封
- 以"三层外交模型"重新组织整个系统
- 更新所有相关 JSON 数据定义

## 关键决策

### 决策 65：采用三层外交模型

**选定方案**：
- 第一层（双边关系层）：好感度 -100~+100，双向独立，8种外交动作
- 第二层（国家声望层）：声望 0~100，五档模糊指示器，影响贸易/联盟
- 第三层（天子权威层）：朝贡度 0~100，每国独立，与周天子交互
- 三层之间有联动：朝贡度→声望→好感度门槛

**理由**：
- 三层模型比扁平的好感度+声望更有层次感
- 朝贡册封机制体现战国"尊王攘夷"的历史特色
- 天子层为游戏增加了一个非军事的外交维度
**影响**：`diplomacy.json` 新增 `tribute` 节，`factions.json` 新增 `initial_tribute`

### 决策 66：删除 diplomacy_modifier 字段

**选定方案**：
- 从 `factions.json` 中删除所有国家的 `diplomacy_modifier` 字段
- 外交修正通过学派系统（`schools.json`）和建筑系统（`buildings.json`）处理

**理由**：
- 该字段从未使用，全部为空 `{}`
- 学派系统已提供 `diplomacy_opinion_multiplier`、`alliance_cost_multiplier` 等修正
- 建筑系统已提供 `diplomacy_bonus`、`diplomacy_reputation` 等效果
- 保持数据模型简洁，避免冗余字段
**影响**：`factions.json` 每个国家对象删除 `diplomacy_modifier`

---

### 决策 67：宣战惩罚全面加重

**选定方案**：

| 动作 | 原值 | 新值 |
|:---|:---|:---|
| 宣战-目标好感 | -30 | **-50** |
| 宣战-其他国家好感 | -10 | **-15** |
| 宣战-声望 | -20 | **-25** |
| 突袭宣战-目标好感 | -30 | **-70** |
| 突袭宣战-其他国家好感 | -10 | **-25** |
| 突袭宣战-声望 | -20 | **-40** |

新增动作效果：

| 动作 | 声望 | 好感度(目标) | 说明 |
|:---|:---|:---|:---|
| 送质子 | 0 | +20 | 新增 |
| 释放俘虏 | +5 | +15 | 新增 |
| 召回质子 | -5 | -15 | 新增 |

**理由**：
- 原宣战惩罚过轻，玩家可以频繁宣战而代价不大
- 突袭宣战应比正式宣战惩罚更重，体现"背信弃义"的代价
- 加重惩罚让外交决策更有分量，宣战成为重大抉择
**影响**：`diplomacy.json` 的 `action_effects` 节更新

### 决策 68：结盟增加声望门槛

**选定方案**：
- 结盟条件：好感度≥30 **且** 双方声望≥**40**
- 声望<40的一方无法发起结盟
- 声望≥80时结盟成功率额外+30%（来自声望效果）

**理由**：
- 仅靠好感度结盟过于简单，缺乏战略深度
- 声望代表国际信誉，低声望国家不被信任为可靠盟友
- 40声望门槛要求玩家维护基本的外交声誉
**影响**：`diplomacy_system.gd` 的 `form_alliance()` 需新增声望检查

### 决策 69：停战谈判条件扩展

**选定方案**：

原有条件保留：
- 赔款（金钱）、割地（城市）、称臣（附庸）

新增条件：
- **释放俘虏**：释放所有被俘大夫回归原国，分数2分，好感度+15，声望+5
- **开放关隘**：指定关隘N回合内可自由穿越（部队可跨越该关隘格移动，但不可在关隘格上结束回合），分数1.5分
- **互不侵犯**：停战自动附带互不侵犯条约，分数1分
- **解除商路封锁**：恢复被切断的商路，分数0.5分

谈判分数系统更新：

| 条件 | 分数 | 说明 |
|:---|:---|:---|
| 赔款 | 金额×0.01 | 保留 |
| 割地 | 每城2分 | 保留 |
| 称臣 | 3分 | 保留 |
| 释放俘虏 | 2分 | 新增 |
| 开放关隘 | 1.5分 | 新增 |
| 互不侵犯 | 1分 | 新增 |
| 解除商路封锁 | 0.5分 | 新增 |

**理由**：
- 战国时期的停战谈判远比赔款割地复杂
- 俘虏释放是重要的外交筹码（如渑池之会）
- 开放关隘体现了关隘在战国军事中的重要性
- 多种条件组合让谈判更有策略深度
**影响**：`diplomacy.json` 的 `negotiation` 节更新，`diplomacy_system.gd` 和 `negotiation_dialog.gd` 需扩展

### 决策 70：俘虏机制与外交系统整合

**选定方案**：
- 俘虏定义：战斗中被俘的武大夫和文大夫（已有设计：40%被俘概率）
- 释放俘虏作为停战谈判条件（决策69已定义）
- 拥有对方被俘大夫 → 对方好感度每回合-1（持续仇恨）
- 释放被俘大夫 → 对方好感度+15，声望+5
- 被俘大夫叛变 → 原国好感度-20

**理由**：
- 大夫被俘机制已有完整设计（见大夫系统.md §8），但未与外交系统联动
- 俘虏是战国时期重要的外交筹码
- 释放俘虏的"仁义"效果鼓励玩家进行外交博弈而非简单处决
**影响**：`diplomacy_system.gd` 需新增俘虏相关查询接口

### 决策 71：声望五档阈值重新设计

**选定方案**：

| 等级 | 阈值 | 效果 |
|:---|:---|:---|
| 极低 | <20 | 触发反玩家联盟 + 商路收入-50% + 结盟成功率-30% |
| 低 | 20~39 | 商路收入-25% + 结盟成功率-20% |
| 中 | 40~60 | 无特殊效果（基准区间） |
| 高 | 61~79 | 商路收入+10% + 结盟成功率+15% |
| 极高 | ≥80 | 商路收入+20% + 结盟成功率+30% + 可发起朝贡册封 |

**理由**：
- 原设计中 20-30 段无单独效果，现在补齐
- 低声望增加结盟惩罚，高声望增加结盟奖励
- 极高声望解锁朝贡册封，形成声望→朝贡的正向循环
**影响**：`diplomacy.json` 的 `reputation_thresholds` 和 `reputation_effects` 更新

### 决策 72：difficulty 字段移出 diplomacy.json

**选定方案**：
- 将 `difficulty` 字段从 `diplomacy.json` 移至 `balance_params.json`
- `diplomacy.json` 只保留外交相关参数

**理由**：
- difficulty 是全局游戏难度设置，不属于外交系统
- 放在 `diplomacy.json` 中容易造成误解
- `balance_params.json` 是更合适的位置
**影响**：`diplomacy.json` 删除 `difficulty` 节，`balance_params.json` 确认已有该节

### 决策 73：质子机制采用大夫 HOSTAGE 状态

**选定方案**：
- 质子 = 大夫的 HOSTAGE 状态，复用现有大夫系统
- 新增 `MinisterState.HOSTAGE` 枚举值
- 任何空闲大夫（武/文/外交）都可以被送作质子
- 质子状态的大夫在对方国家，不能执行任何行动
- 每个国家最多向1个国家送质子，最多接收1个质子

**理由**：
- 复用现有大夫系统，不需要新增资源类型
- 消耗大夫名额增加策略决策（送质子 = 牺牲一个大夫的生产力）
- 与被俘机制形成对称：被俘是被动失去大夫，送质子是主动送出大夫
**影响**：`ministers.json` 需新增 HOSTAGE 状态定义

### 决策 74：质子动作参数设计

**选定方案**：

| 动作 | 条件 | 效果 |
|:---|:---|:---|
| 送质子 | 好感度≥20，双方和平，有空闲大夫 | 好感度+20，对方不可对你宣战（质子存在期间） |
| 召回质子 | 有质子在对方 | 好感度-15，声望-5，和平保证解除 |
| 对方归还质子 | 对方有你的质子 | 好感度+10 |

质子状态事件：

| 状态 | 触发 | 效果 |
|:---|:---|:---|
| 正常 | 送质子成功 | 持续提供和平保证 + 对方好感度每回合+1 |
| 逃跑 | 每回合5%概率 | 好感度-25，声望-10，和平保证解除 |
| 死亡 | 每回合2%概率 | 好感度-40，声望-15，和平保证解除，可能引发战争 |

**理由**：
- 送质子的核心价值是"和平保证"——对方不能对你宣战
- 逃跑/死亡概率让质子不是无风险的，增加戏剧性
- 好感度每回合+1的持续加成鼓励长期维持质子关系
**影响**：`diplomacy.json` 新增 `hostage` 节

### 决策 75：朝贡度系统设计

**选定方案**：
- 朝贡度范围 0~100，每个国家对周天子独立
- 周天子为中立权威实体，不参与战争和领土争夺
- 朝贡度通过朝贡行为提升，不朝贡则自然衰减

初始朝贡度（基于历史）：

| 国家 | 初始朝贡度 | 历史依据 |
|:---|:---|:---|
| 齐 | 60 | 稷下学宫，礼乐传统 |
| 燕 | 55 | 姬姓诸侯，与周同宗 |
| 韩 | 50 | 中原小国，礼制尚可 |
| 魏 | 50 | 中原大国 |
| 赵 | 45 | 胡服骑射，务实 |
| 楚 | 30 | 自称王号，蔑视周室 |
| 秦 | 20 | 变法强国，不重礼制 |

**理由**：
- 齐国礼乐传统最强，初始朝贡度最高
- 秦国变法后不重礼制，初始朝贡度最低
- 楚国曾自称王号，对周室蔑视
**影响**：`factions.json` 新增 `initial_tribute` 字段

### 决策 76：朝贡行为参数设计

**选定方案**：

| 行为 | 朝贡度变化 | 消耗 | 冷却 |
|:---|:---|:---|:---|
| 小朝贡 | +5 | 100金 | 每回合限1次 |
| 大朝贡 | +15 | 300金 | 每3回合限1次 |
| 祭祀大典 | +20 | 200金+50粮 | 每5回合限1次 |
| 不朝贡（每10回合） | -5 | 无 | 自然衰减 |
| 攻击周天子领地 | -50 | 无 | 严重惩罚 |
| 僭越称王 | -30 | 无 | 特殊事件触发 |

朝贡度与声望联动：
- 朝贡度每10点 → 声望+1（被动加成）
- 朝贡度<20 → 声望-5（被视为不敬周室）
- 朝贡度≥80 → 声望+5（被视为忠臣）

**理由**：
- 三种朝贡档位提供不同投入产出比
- 自然衰减防止"一次朝贡永久收益"
- 攻击天子和僭越称王是高风险高收益的特殊选择
**影响**：`diplomacy.json` 新增 `tribute` 节

### 决策 77：册封系统设计

**选定方案**：

| 册封类型 | 条件 | 效果 |
|:---|:---|:---|
| 诸侯册封 | 朝贡度≥40 | 声望+10，好感度(天子)+5 |
| 方伯册封 | 朝贡度≥60 | 声望+15，对低声望国家宣战惩罚-50% |
| 霸主册封 | 朝贡度≥80 | 声望+20，结盟费用-30%，可号令诸侯（触发特殊事件） |

册封流程：
1. 玩家/AI朝贡度达标后，可向周天子请求册封
2. 周天子根据朝贡度和声望决定是否册封
3. 册封成功 → 应用效果，全服公告
4. 册封失败 → 朝贡度-5（天子不满）
5. 每种册封每局游戏只能获得一次

**理由**：
- 三种册封对应不同阶段的战略需求
- 方伯的"征伐不臣"效果与声望系统联动
- 霸主的"号令诸侯"效果创造独特的外交优势
- 每局只能册封一次防止反复刷册封
**影响**：`diplomacy.json` 新增 `enfeoffment` 节

### 决策 78：朝贡特殊事件设计

**选定方案**：

| 事件 | 触发条件 | 效果 |
|:---|:---|:---|
| 天子求援 | 朝贡度最高国被攻击 | 周天子号召其他国家援助，好感度+10 |
| 九鼎迁徙 | 朝贡度≥80且声望≥80 | 可请求九鼎迁入本国，声望+30，特殊胜利条件 |

**理由**：
- 天子求援让高朝贡度国家获得被动保护
- 九鼎迁徙和天子禅让提供两条基于朝贡的胜利路径
- 僭越称王是低声望国家的冒险选择
**影响**：`data/events.json` 新增朝贡相关事件

### 决策 79：纵横家事件系统设计

**选定方案**：
- 纵横家作为独立事件系统，自动触发
- 触发条件：回合数 + 国家关系状态 + 学派条件
- 触发概率：基础概率 × 学派修正 × 关系修正
- 纵横家学派存在时，事件概率+50%

**理由**：
- 纵横家是战国最具特色的外交现象
- 作为事件系统而非动作系统，增加游戏的随机性和历史感
- 与学派系统联动，鼓励玩家选择纵横家学派
**影响**：`data/events.json` 新增纵横家事件链

### 决策 80：合纵事件链设计

**选定方案**：

| 事件 | 触发条件 | 效果 |
|:---|:---|:---|
| 合纵之议 | 某国声望≥70且正在扩张（3回合内攻占2城以上） | 好感度最低的3国自动结盟，对扩张国宣战 |

- 触发条件模拟"一国独大引发围攻"的历史模式
- 瓦解和内讧事件防止联盟永久存在
**影响**：`data/events.json` 新增合纵事件链

### 决策 82：纵横家个人事件设计

**选定方案**：

| 事件 | 触发条件 | 效果 |
|:---|:---|:---|
| 苏秦出山 | 回合≥10，齐国存在，齐国声望≥50 | 齐国获得"合纵"能力，可主动发起合纵（消耗300金+20声望） |

**理由**：
- 苏秦、张仪是战国最著名的纵横家
- 个人事件让特定国家获得主动发起合纵/连横的能力
- 公孙衍游说增加随机性
**影响**：`data/events.json` 新增纵横家个人事件

### 决策 83：纵横家与三层模型联动

**选定方案**：
- **双边关系层**：纵横家事件直接改变多国好感度（±10~30）
- **声望层**：合纵成功提升参与者声望+5，连横成功提升发起者声望+10
- **天子层**：合纵可获得周天子支持（朝贡度+5），连横可能被天子视为不义（朝贡度-5）

**理由**：
- 纵横家事件影响所有三层，体现其在战国政治中的核心地位
- 合纵获得天子支持体现"尊王"的传统
- 连横被天子视为不义体现"霸道"vs"王道"的冲突
**影响**：事件效果需同时更新好感度、声望、朝贡度

### 决策 85：power_score 补全文化/科技权重

**选定方案**：
- 原公式：城市×50 + 军队×3 + 科技×15 + 民心×1 + 文化×1
- 文化和科技当前硬编码为0，需接入实际值
- 新公式：城市×50 + 军队×3 + 已研究科技数×15 + 民心×1 + 文化值×1

**理由**：
- 文化和科技已有实际数值（通过文化系统和科技树追踪）
- 硬编码为0导致国力评分不准确
- 补全后均势联盟触发更精确
**影响**：`diplomacy_system.gd` 的 `get_power_score()` 需接入实际值

### 事件管理器条件扩展（event_manager.gd）

当前事件管理器支持 8 种条件（season/morale_min/morale_max/turn_min/turn_max/faction/at_war_with/tech_researched），新增外交事件需要以下条件支持：

- [ ] `reputation_min` / `reputation_max`：检查玩家声望阈值（调用 `DiplomacySystem.get_reputation()`）
- [ ] `tribute_min` / `tribute_max`：检查玩家朝贡度阈值（调用 `DiplomacySystem.get_tribute()`）
- [ ] `faction_exists`：检查指定国家是否存在（调用 `GameManager.faction_exists()`）
- [ ] `faction_reputation_min`：检查指定国家的声望（调用 `DiplomacySystem.get_reputation(faction_id)`）
- [ ] `hezong_active`：检查合纵联盟是否处于激活状态（调用 `DiplomacySystem.is_hezong_active()`）
- [ ] `lianheng_active`：检查连横联盟是否处于激活状态（调用 `DiplomacySystem.is_lianheng_active()`）
- [ ] `expanding_fast`：检查玩家是否正在快速扩张（3回合内攻占2城以上，调用 `GameManager.get_recent_conquests()`）
- [ ] `has_strong_enemy`：检查是否存在国力超过玩家1.2倍的敌国
