# 阶段5 奇观系统与旱灾决策记录

## 决策 #181：奇观系统全面重构

### 背景

原有奇观系统存在以下问题：
1. 效果键与建筑系统不兼容（`recruit_speed_bonus`、`research_speed_bonus` 在建筑系统中未定义）
2. 奇观建造机制与普通建筑无差异（无全局唯一约束、占用建筑槽位）
3. 缺少奇观竞赛机制（所有国家可同时建造同一奇观）
4. 奇观种类不足（仅6座，未覆盖所有机制领域）

### 决策内容

#### 1. 奇观分类

奇观分为两类：

| 类型 | 说明 | 建造方式 |
|:---|:---|:---|
| **单城奇观** | 在城市内建造，效果全国生效 | 在城市建筑面板中建造 |
| **地图奇观** | 在地图上跨格建造，效果覆盖路径沿线 | 玩家选择放置位置 |

#### 2. 奇观竞赛机制

- **全局唯一**：每种奇观全世界只能存在1座
- **竞赛机制**：所有国家可同时建造同一奇观，先建成者获得
- **失败返还**：被抢建后100%返还金、木、建材
- **不占槽位**：奇观不占用城市建筑槽位

#### 3. 建造时间

- 单城奇观：统一20回合
- 地图奇观：按长度计算（每格2回合）

#### 4. 效果键映射

修正现有奇观效果键，确保与建筑系统兼容：

| 原效果键 | 修正后 | 说明 |
|:---|:---|:---|
| `recruit_speed_bonus` | `recruit_discount_national` | 对应建筑 `recruit_discount` |
| `research_speed_bonus` | `corruption_reduction_national` | 替换为已有效果 |
| `diplomacy_opinion_neighbor` | `diplomacy_reputation` | 对应建筑 `diplomacy_reputation` |
| `wood_production_national` | 保留 | 百分比加成，与建筑固定值区分用途 |
| `culture_production_national` | 保留 | 百分比加成，与建筑固定值区分用途 |
| `drought_immune` | `drought_mitigation` | 改为减灾效果（概率×0.5，效果×0.5） |

## 决策 #182：旱灾系统

### 背景

需要一个自然灾害系统增加游戏随机性和策略深度。旱灾作为首个自然灾害，影响粮食产出和安定度。

### 决策内容

#### 1. 触发条件

- **触发季节**：仅春季
- **触发概率**：每城每回合独立判定，基础概率 3%~5%
- **概率公式**：
  ```
  drought_base_chance = 0.03 + 0.02 × (city_count / 50)
  drought_chance = drought_base_chance × wonder_modifier × (1 - granary_count × 0.01)
  ```

#### 2. 旱灾效果

- **粮食产出 debuff**：-70%（加法层，单回合）
- **安定度影响**：-10（单回合，下一回合自动恢复）

#### 3. 减灾手段

| 手段 | 概率修正 | 效果修正 | 来源 |
|:---|:---|:---|:---|
| 都江堰/芍陂 | ×0.5 | ×0.5 | `wonders.json` → `drought_mitigation: 0.5` |
| 粮仓 | 每座 -1% | — | `buildings.json` → `granary` |
| 御史台 | — | 安定度影响 ×0.5 | `buildings.json` → `censor_office` |

#### 4. 效果计算

```
// 触发判定
if random() < drought_chance:
    // 粮食产出 debuff（加法层）
    food_debuff = -0.70 × (1 - wonder_drought_mitigation)

    // 安定度影响
    stability_penalty = -10 × (1 - wonder_drought_mitigation) × (1 - censor_office_modifier)
```

#### 5. 数据定义

**balance_params.json 新增**：
```json
{
  "drought": {
    "base_chance_min": 0.03,
    "base_chance_max": 0.05,
    "food_debuff": -0.70,
    "stability_penalty": -10,
    "granary_chance_reduction": 0.01,
    "censor_stability_reduction": 0.5
  }
}
```

### 待实现功能

1. **旱灾触发判定**：春季回合结算时，每城独立判定是否触发旱灾
2. **粮食产出 debuff**：在粮食产出公式中应用旱灾 debuff
3. **安定度影响**：触发旱灾时减少安定度，下一回合自动恢复
4. **减灾效果计算**：计算奇观、粮仓、御史台的减灾效果
5. **旱灾事件通知**：触发旱灾时显示事件通知

### 决策依据

- 旱灾增加游戏随机性和策略深度
- 奇观减灾效果（而非免疫）保持旱灾的威胁性
- 粮仓和御史台提供额外减灾手段
- 单回合效果避免长期负面体验
