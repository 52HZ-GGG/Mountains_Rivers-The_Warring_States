## 决策 #138：战斗公式重构（百分比加法叠加）

**状态**：✅ 已定稿

**背景**：原战斗公式中所有百分比修正（地形、将领、派系、士气）采用连乘方式，导致修正项之间相互放大，数值难以控制。需要改为百分比相加后统一乘算，克制关系单独乘算。

**决策**：
- 攻击方：所有百分比 buff 相加后乘以基础攻击力（地形偏移 + 大夫勇武 + 派系 + 士气 + 伏击/火攻 + 学派 + 科技 + 兵种特技）
- 防御方：所有百分比 buff 相加后乘以基础防御力（地形偏移 + 大夫韬略 + 派系 + 学派 + 科技 + 城墙 + 夹击 debuff）
- 克制关系：在加法叠加之后、扣除防御之前单独乘算
- 崩溃态（士气 < 50）的 -50% 攻防作为独立乘算，不参与加法层

**影响范围**：数值公式.md §1.1~§1.5、游戏机制总览.md 战斗系统 Section 1

## 决策 #143：远程攻击机制设计（双模式+遮挡+精度）

**状态**：✅ 已定稿

**背景**：原远程攻击仅简单定义为"range ≥ 2"，缺乏主动攻击/反击区分、遮挡判定和精度系统。需要完整设计远程战斗子系统。

**决策**：

**双模式攻击**：
- 远程单位拥有两种攻击模式：主动远程攻击（标记 `ranged_attack_mode: "no_counter"`，不触发反击）和近战反击（被近战攻击时使用攻击力 100% 反击）
- 所有近战攻击触发反击；不触发反击的攻击：城市主动攻击、箭塔攻击、远程单位主动攻击

**远程遮挡（高程差减射程）**：
- 地形高程等级：平原/沙漠/冻土(0) < 森林/沼泽/渡口(1) < 山地/关隘(2)，河流(-1)不可通行
- 从低处向高处射击：每高 1 级高程，射程减少 1 格
- 从高处向低处或同高程：无影响
- 示例：弓兵在平原(射程2)向山地射击，高程差2，有效射程=0，无法攻击

**远程精度**：
- `ranged_accuracy_mod` 不再是命中率修正，而是改变随机波动幅度
- 基准精度 0：波动范围 ±10%（0.9~1.1）
- 精度 +100：波动范围 0%~20%（1.0~1.2，伤害偏高）
- 精度 -100：波动范围 -20%~0%（0.8~1.0，伤害偏低）
- 线性插值：`spread_lo = 0.9 + accuracy / 1000`，`spread_hi = 1.1 + accuracy / 1000`

**影响范围**：units.json（ranged_attack_mode）、terrain.json（elevation）、balance_params.json（combat.base_random_spread_*）、战斗系统.md §5.4~§5.6

## 决策 #161：夹击/包围机制移入士气系统

**状态**：✅ 已定稿

**背景**：原夹击效果为防御层 debuff -0.3（降低防御力 30%），与士气系统分离。夹击触发条件为"两个单位同时接壤"，未限定方向关系，缺乏战术深度。此外缺少包围机制。将夹击/包围统一移入士气系统，可使士气管理成为更核心的战斗要素。

**决策**：

### 夹击重设计
- **触发条件**：两个敌方部队从**六角形相对的两个格子**同时与目标接壤。六角格相对方向：东↔西（0↔3）、东北↔西南（1↔4）、西北↔东南（2↔5）。其中至少一个部队发起攻击时触发。
- **效果**：目标士气 **-20**（原为防御层 debuff -0.3）
- **JSON 字段**：`unit_morale.flanking_morale_loss: -20`

### 新增包围
- **触发条件**：目标单位周围 **6 个相邻格子全部被敌方部队占据**
- **效果**：目标士气 **-50**
- **JSON 字段**：`unit_morale.encirclement_morale_loss: -50`

### 优先级规则
- 包围与夹击**不叠加**，包围触发时仅生效包围效果（-50），不再额外计算夹击
- 从防御层公式中移除 `flanking_penalty` 项

**影响范围**：
- 战斗系统.md §2.2（移除防御公式夹击项）、§5.2（重写夹击/包围定义）、§6.2（士气事件表新增）、§6.3（士气参数表新增）
- balance_params.json `unit_morale` 段（新增 `flanking_morale_loss`、`encirclement_morale_loss`）
- `combat_resolver.gd`（待实现：夹击方向检测、包围检测、士气扣减逻辑）

- 效果：敌方单位进入 ZOC 范围时，移动力额外消耗 +1
- 叠加：多个 ZOC 重叠时效果不叠加，仅触发一次 +1
- 水军免疫陆地 ZOC

### 162.4 寻路集成
- ZOC 额外消耗加入 Dijkstra/A* 寻路边权重计算

## 决策 #169：腐败值机制

**状态**：✅ 已定稿

**背景**：经营系统缺乏帝国规模惩罚机制，玩家扩张无负面反馈。需要引入腐败系统，使大规模帝国有额外的治理成本，增加"扩张 vs 治理"的策略抉择。

**决策**：

### 169.1 腐败值模型
- **固定值模型**：腐败值每回合由帝国规模**重新计算**，非累加
- 公式：`corruption_base = C_cities × log2(city_count + 1) + C_pop × log2(total_pop + 1) + C_border × border_city_count`
- 边界城市定义：距离首都 > 8 格的城市（`corruption.growth.border_distance_threshold`）
- 系数：C_cities=12, C_pop=8, C_border=5（偏激进，小势力~40，大势力~70，超大帝国~97）
- 最终值 = clamp(base × (1 - tech_reduction) - Σ(countermeasures), 0, 100)

### 169.2 四档效果（档内线性插值）
| 等级 | 区间 | 税收效率修正 | 民心变化/回合 |
|:---|:---:|:---|:---|
| 清廉 | 0~20 | +0% ~ +10% | +5 ~ 0 |
| 正常 | 21~50 | 0 | 0 |
| 腐败 | 51~80 | 0 ~ -6% | 0 ~ -15 |
| 严重腐败 | 81~100 | -6% ~ -10% | -15 ~ -31 |

### 169.3 治理手段
1. **御史台**（政治建筑）：max_level 2，全国限 2 座，每级 -3 腐败，维护 15 金/级
2. **科技**：减少腐败基础值（百分比），上限 20%（具体科技待科技树设计）
3. **文大夫"肃贪"技能**：每级 -2 全国腐败，max level 3

### 169.4 数据变更
- `balance_params.json` 新增 `corruption` 段（growth/tech_reduction_max/thresholds/effects/censor）
- `buildings.json` 新增 `censor_office`（御史台，政治类，200 金 + 20 木，维护 15 金）

**影响范围**：
- `balance_params.json`（新增 `corruption` 段）
- `buildings.json`（新增 `censor_office`）
- `经营系统.md`（新增 §9 腐败系统，§10→§11、§11→§12 重编号）
- `城市经营系统.md`（§3.1/§3.2 新增御史台、§10 跨系统交互、§11 数据索引、决策索引）
