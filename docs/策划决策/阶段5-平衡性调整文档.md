## 决策 #137：新增沙漠/冻土地形

**状态**：✅ 已定稿

**背景**：现有 7 种地形缺乏极端环境地形。沙漠（西北干旱区）和冻土（北方严寒区）可增加地理策略维度，使补给管理和行军路线规划更具深度。

**决策**：
- 新增沙漠（desert）和冻土（tundra）两种地形
- 移动力消耗均为 2
- 补给消耗 +20%（新增 terrain.json 字段 `supply_cost_modifier: 1.2`）
- 战斗修正：沙漠 atk 0.95 / def 0.9，冻土 atk 0.9 / def 1.0
- 伏击概率：沙漠 5%，冻土 10%

**影响范围**：terrain.json、游戏机制总览.md、skirmish_tile_textures.gd

## 决策 #138：战斗公式重构（百分比加法叠加）

**状态**：✅ 已定稿

**背景**：原战斗公式中所有百分比修正（地形、将领、派系、士气）采用连乘方式，导致修正项之间相互放大，数值难以控制。需要改为百分比相加后统一乘算，克制关系单独乘算。

**决策**：
- 攻击方：所有百分比 buff 相加后乘以基础攻击力（地形偏移 + 大夫勇武 + 派系 + 士气 + 伏击/火攻 + 学派 + 科技 + 兵种特技）
- 防御方：所有百分比 buff 相加后乘以基础防御力（地形偏移 + 大夫韬略 + 派系 + 学派 + 科技 + 城墙 + 夹击 debuff）
- 克制关系：在加法叠加之后、扣除防御之前单独乘算
- 崩溃态（士气 < 50）的 -50% 攻防作为独立乘算，不参与加法层

**影响范围**：数值公式.md §1.1~§1.5、游戏机制总览.md 战斗系统 Section 1

## 决策 #139：城防战斗公式完善

**状态**：✅ 已定稿

**背景**：阶段4决策记录定义了城市战斗机制（决策 126），但未与新的加法叠加公式对齐，且缺少部队 vs 驻进城防的完整公式。

**决策**：
- 部队 vs 城防：有驻军时 50/50 伤害分配，无驻军时 100% 对城墙
- 攻城器械倍率 ×3.0 仅作用于城墙伤害部分
- 驻军使用自身防御值（含 buff），城墙使用建筑防御值 × (1 + 等级加成)
- 城市主动攻击：每回合攻击范围内 1 个敌军
- 箭塔独立攻击：各自攻击范围内敌军

**影响范围**：数值公式.md §1.6~§1.7、游戏机制总览.md 战斗系统

## 决策 #140：单位士气系统重设计（三段式）

**状态**：✅ 已定稿

**背景**：原士气系统崩溃阈值过低（20），崩溃效果仅为随机撤退概率，缺乏战略深度。需要重新设计为三段式系统，使士气管理成为战斗核心要素。

**决策**：
- 三段式士气：崩溃 < 50 / 正常 50~129 / 高士气 ≥ 130
- 基础士气 100，上限 130，自然恢复最多到 100
- 崩溃效果：每回合 -20% HP，攻击/防御/移动力 ×0.5
- 士气变化：友军阵亡 -5，友军击杀 +5，本单位击杀额外 +15
- 高士气（≥130）：攻击力 +10%
- 自然恢复 +3/回合（城内 +8），上限 100

**影响范围**：数值公式.md §1.8、游戏机制总览.md 单位士气系统、balance_params.json unit_morale
