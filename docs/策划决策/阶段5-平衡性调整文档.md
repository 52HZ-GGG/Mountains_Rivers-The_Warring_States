## 决策 #138：战斗公式重构（百分比加法叠加）

**状态**：✅ 已定稿

**背景**：原战斗公式中所有百分比修正（地形、将领、派系、士气）采用连乘方式，导致修正项之间相互放大，数值难以控制。需要改为百分比相加后统一乘算，克制关系单独乘算。

**决策**：
- 攻击方：所有百分比 buff 相加后乘以基础攻击力（地形偏移 + 大夫勇武 + 派系 + 士气 + 伏击/火攻 + 学派 + 科技 + 兵种特技）
- 防御方：所有百分比 buff 相加后乘以基础防御力（地形偏移 + 大夫韬略 + 派系 + 学派 + 科技 + 城墙 + 夹击 debuff）
- 克制关系：在加法叠加之后、扣除防御之前单独乘算
- 崩溃态（士气 < 50）的 -50% 攻防作为独立乘算，不参与加法层

**影响范围**：数值公式.md §1.1~§1.5、游戏机制总览.md 战斗系统 Section 1

## 决策 #143：远程攻击机制设计（双模式+遮挡+精度）

**状态**：✅ 已定稿

**背景**：原远程攻击仅简单定义为"range ≥ 2"，缺乏主动攻击/反击区分、遮挡判定和精度系统。需要完整设计远程战斗子系统。

**决策**：

**双模式攻击**：
- 远程单位拥有两种攻击模式：主动远程攻击（标记 `ranged_attack_mode: "no_counter"`，不触发反击）和近战反击（被近战攻击时使用攻击力 100% 反击）
- 所有近战攻击触发反击；不触发反击的攻击：城市主动攻击、箭塔攻击、远程单位主动攻击

**远程遮挡（高程差减射程）**：
- 地形高程等级：平原/沙漠/冻土(0) < 森林/沼泽/渡口(1) < 山地/关隘(2)，河流(-1)不可通行
- 从低处向高处射击：每高 1 级高程，射程减少 1 格
- 从高处向低处或同高程：无影响
- 示例：弓兵在平原(射程2)向山地射击，高程差2，有效射程=0，无法攻击

**远程精度**：
- `ranged_accuracy_mod` 不再是命中率修正，而是改变随机波动幅度
- 基准精度 0：波动范围 ±10%（0.9~1.1）
- 精度 +100：波动范围 0%~20%（1.0~1.2，伤害偏高）
- 精度 -100：波动范围 -20%~0%（0.8~1.0，伤害偏低）
- 线性插值：`spread_lo = 0.9 + accuracy / 1000`，`spread_hi = 1.1 + accuracy / 1000`

**影响范围**：units.json（ranged_attack_mode）、terrain.json（elevation）、balance_params.json（combat.base_random_spread_*）、战斗系统.md §5.4~§5.6
