## 关键决策

### 决策 126：水军可移动地形 — 河流 + 渡口

**选定方案**：
- 水军可移动单元格定义为：河流（river）、渡口（ford）两种地形
- 渡口的 `is_navigable` 字段从 `false` 改为 `true`
- 在水面单元格修建建筑后，水军依然可以正常通行

**理由**：
- 渡口作为河流的浅水区域，历史上是水军重要的停靠和补给点
- 水军需要更多可通行地形来增加战术灵活性
- 建筑不阻碍水军通行符合物理逻辑（码头、港口等建筑本就是为水军建造的）

**已落地**：
- `data/terrain.json` → ford 条目：`is_navigable: true`
- `docs/机制概览/战斗系统.md` → §8.2 水军移动规则

### 决策 127：行动速度与回合顺序机制

**选定方案**：
- 在 `factions.json` 中为七国定义 `action_speed` 字段（范围 3-5）
- 每回合开始时按 `action_speed` 降序排列，决定行动顺序
- 每回合内**只有一个势力**可以执行所有操作（移动、攻击、经营、外交）
- 相同 `action_speed` 的势力随机排序（游戏开始时固定种子）

**行动速度分配**：

| 势力 | action_speed | 设计理由 |
|:---|:---:|:---|
| 秦 | 80 | 虎狼之师，商鞅变法后军纪严明，行动迅速 |
| 赵 | 70 | 胡服骑射改革，骑兵机动性强 |
| 魏 | 65 | 武卒精锐，训练有素 |
| 燕 | 60 | 坚韧不拔，执行力强 |
| 齐 | 55 | 文风鼎盛，节奏稳健 |
| 楚 | 50 | 地广人众，调度较慢 |
| 韩 | 45 | 精工巧匠，稳扎稳打 |

**理由**：
- 行动速度机制创造先后手策略差异，高速势力可先发制人
- 单势力行动回合制简化状态管理，避免多势力同时操作的复杂性
- 数值设计体现历史特征（秦赵尚武，齐楚重文/地广）

**已落地**：
- `data/factions.json` → 各势力新增 `action_speed` 字段
- `docs/机制概览/战斗系统.md` → 新增 §10 战斗状态机与回合机制

### 决策 128：单格单部队限制

**选定方案**：
- 每个六角格只能容纳一支部队
- 已被友军占据的格子无法进入
- 已被敌军占据的格子需要先击败守军才能占领
- 移动路径规划时自动绕过已占据格子

**理由**：
- 单格单部队简化战斗结算逻辑
- 避免堆叠导致的数值爆炸问题
- 促使玩家合理规划部队阵型和进攻路线

**已落地**：
- `scripts/autoload/tactical_skirmish_manager.gd` → `_occupant_id_at()` 检查
- `docs/机制概览/战斗系统.md` → §10.3 单格单部队限制

### 决策 129：伤害溢出机制

**选定方案**：
- 当部队被击杀时，多余的伤害直接溢出，不保留
- 溢出伤害不转移至其他单位
- 公式：`实际伤害 = min(攻击力, 目标当前HP)`

**示例**：
- 攻击力 50，目标 HP 30 → 造成 30 伤害，目标被歼灭，20 点溢出伤害丢弃
- 攻击力 50，目标 HP 80 → 造成 50 伤害，目标剩余 30 HP

**理由**：
- 简化战斗结算，避免复杂的伤害转移逻辑
- 符合直觉（击杀后多余的动能消散）
- 防止通过伤害转移实现过于强力的连锁击杀

**已落地**：
- `scripts/systems/combat_resolver.gd` → 添加溢出机制文档说明
- `scripts/autoload/tactical_skirmish_manager.gd` → 现有代码已实现此行为
- `docs/机制概览/战斗系统.md` → §10.4 伤害溢出机制

## 状态机设计

### 回合状态枚举

```gdscript
enum TurnState {
    TURN_START,        # 回合开始，计算行动顺序
    FACTION_ACTION,    # 势力行动中
    FACTION_END,       # 势力结束回合
    TURN_END           # 回合结束，触发回合结束效果
}
```

### 回合流程

```
1. TURN_START → 按 action_speed 排序确定行动顺序
2. FACTION_ACTION → 当前势力执行所有操作
3. FACTION_END → 当前势力结束回合
4. 下一个势力 → 回到 FACTION_ACTION
5. 所有势力完毕 → TURN_END → 触发回合结束效果
6. 回到 TURN_START
```
