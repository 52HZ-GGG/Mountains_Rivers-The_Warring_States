## 0. 公式重构：减法模型 → 乘法模型

**原公式**：`raw_dmg = max(effective_atk - effective_def, 1)`（减法模型，攻防接近时伤害趋近 0）

**新公式**：`raw_dmg = max(counter_dmg × COEFF / (COEFF + effective_def), 1)`（乘法模型）

**COEFF = 20**（防御系数）：
- 5 防（弓兵）→ 减伤 20%
- 10 防（步兵）→ 减伤 33%
- 18 防（铁甲）→ 减伤 47%

**兵种攻击力全面重算**：目标同类互打 5-6 回合，所有兵种 attack 值按乘法模型重新计算。

**防御建筑**（`buildings.json`，需花资源建造，占用建筑槽）：

| 建筑 | 等级 | structure_hp | 效果 | 维护 |
|:---|:---:|:---:|:---|:---:|

### 1.2 当前攻城伤害公式

**打城墙**（`skirmish_attack_pipeline.gd:275-310`）：
```
wall_dmg = eff_atk × split_wall(0.5) × siege_mult(3.0)
```
- 无防御值，50% 分流导致伤害偏低
- 步兵(ATK=27)：27×0.5=13.5/回合，打 550HP 城墙需 **41 回合**

**打城市本体**（减法模型）：
```
city_dmg = max(effective_atk - city_defense, 1)
```
- 步兵(ATK=27) vs 城(def=45)：27-45<0 → **保底 1 伤害**
- 即使攻城器械(ATK=45) vs 镇(def=25)：45-25=20，600HP 需 **30 回合**

**箭塔攻击**（`tactical_skirmish_manager.gd:1549-1552`）：
```
tower_dmg = max(1, tower_atk - target_defense)
```
- 仍为减法模型

### 1.3 核心问题

1. **城市 HP 与城墙 HP 混淆**：`_init_city_data()` 用 `city_levels.hp` 作为城墙 HP，建筑物的 `structure_hp` 未被使用
2. **city_defense 过高**（10~100）：减法模型下新攻击力无法穿透
3. **城墙公式无防御值**：打墙伤害与攻城方攻击力线性相关，无策略深度
4. **箭塔使用减法模型**：与野战公式不统一
5. **城市防御值单调**：仅随等级线性增长，缺乏特色

## 2. 设计目标

1. **3 支军队**（2 步兵 + 1 冲车，中等品质）攻打 **Lv3 城**→ **3~4 回合**攻下（无守军）
2. 单个步兵独自攻城 → 极慢（几乎不可能）
3. 城市等级有明确区分：村脆弱、都坚不可摧
4. 防御建筑（城墙/箭塔/瓮城）各有清晰战术作用
5. 所有攻城公式统一为乘法模型

---

## 3. 防御建筑体系整理

### 3.1 三种防御建筑的定位

| 建筑 | 战术作用 | 机制说明 |
|:---|:---|:---|
| **城墙** | HP 缓冲层 + 防御加成 | 提供独立的城墙 HP 池（必须先破墙才能攻城）；根据城墙等级提供 `defense_bonus` 百分比加成，乘算到城市有效防御上；城墙 HP 越低，防御加成越弱 |
| **箭塔** | 主动伤害输出 | 每回合自动攻击范围内敌军 1 次；城墙被毁后箭塔失效；仅 Lv4+ 城市自带箭塔，其他城市需建造 |
| **瓮城** | 额外 HP + 攻击加成 | 瓮城的 `structure_hp` **叠加到城墙总 HP** 上（增加破墙难度）；`city_attack_bonus` 提升城市反击伤害 |

### 3.2 城墙 HP 构成

战术层城墙总 HP = 城墙建筑物 `structure_hp` + 瓮城建筑物 `structure_hp`（如有）

**不再使用** `city_levels.hp` 作为城墙 HP。城市本体 HP 仅在城墙破后生效。

### 3.3 城墙防御加成（与 HP 挂钩）

城墙提供百分比防御加成，随 HP 损耗而衰减：

```
wall_ratio = current_wall_hp / max_wall_hp
effective_ratio = max(wall_ratio, wall_defense_min_ratio(0.5))
wall_def_bonus = wall_defense_bonus × effective_ratio
```

- 城墙满血 → 100% 防御加成
- 城墙半血 → 50% 防御加成
- 城墙残血 → 最低保底 50%（`wall_defense_min_ratio`）

### 3.4 箭塔工作机制

- **触发条件**：箭塔 HP > 0 且城墙 HP > 0
- **攻击范围**：城市 Lv3 以下 = 1 格，Lv4+ = 2 格
- **伤害公式**（统一为乘法模型）：`tower_dmg = tower_atk × COEFF / (COEFF + target_def)`
- **限制**：每座箭塔每回合仅攻击 1 个目标
- **建造 vs 自带**：Lv4+ 城市自带箭塔，其他等级需建造箭塔建筑物

## 5. 攻城公式

### 5.1 打城墙（新公式）

```
wall_dmg = eff_atk × siege_mult × COEFF / (COEFF + wall_struct_def)
```

- `COEFF = 20`（复用野战防御系数）
- `siege_mult`：攻城器械 ×3.0，普通单位 ×1.0
- `wall_struct_def`：城墙建筑物的防御值（随等级递增：Lv1=5, Lv2=8, Lv3=12）

**与旧公式对比**：

| | 旧公式 | 新公式 |
|:---|:---|:---|
| 步兵(ATK=27) vs Lv3 城墙 | 27×0.5×1.0 = **13.5** | 27×1.0×20/32 = **16.9** |
| 冲车(ATK=15, ×3) vs Lv3 城墙 | 15×0.5×3.0 = **22.5** | 15×3.0×20/32 = **28.1** |
| 3 单位合计/回合 | 49.5 | **61.9** |

> 新公式移除了 50% 分流限制，引入结构防御值作为平衡杠杆。

### 5.2 打城市本体（新公式）

```
city_dmg = max(1, eff_atk × COEFF / (COEFF + city_def))
```

- 城墙必须先被摧毁，才能对城市本体造成伤害
- city_def 值较低（2~10），不会像旧数值（10~100）那样让攻击无效

### 5.3 箭塔攻击（新公式）

```
tower_dmg = max(1, tower_atk × COEFF / (COEFF + target_def))
```

- 统一为乘法模型
- 箭塔在城墙被毁后失效

### 5.4 关隘结构伤害（新公式）

```
struct_dmg = max(1, eff_atk × siege_mult × COEFF / (COEFF + pass_struct_def))
```

### 5.5 伤害分流（保持不变）

有守军时攻击城市：
- 50% 伤害 → 城墙 HP
- 50% 伤害 → 守军 HP
- 攻城器械的 ×3 仅作用于城墙部分

无守军时：
- 100% 伤害 → 城墙 HP（`no_garrison_wall_ratio = 1.0`）

## 7. 防御建筑战术价值总结

| 建筑 | 无此建筑时 | 有此建筑时 | 战术意义 |
|:---|:---|:---|:---|
| 城墙 | 城市裸露，直接打本体 | 提供 HP 缓冲层 + 防御加成 | **必须先破墙**，是攻城的核心障碍 |
| 箭塔 | 攻城方无损耗 | 每回合自动攻击敌军 | 迫使攻城方**速战速决**或带攻城器械 |
| 瓮城 | 城墙 HP 较低 | 城墙 HP +200~400，城攻 +10~20% | 增加破墙难度，守军反击更强 |

**组合效果**：
- 仅城墙：标准防御
- 城墙 + 箭塔：攻城方每回合被箭塔消耗，心理压力大
- 城墙 + 瓮城：城墙更厚，破墙时间延长
- 城墙 + 箭塔 + 瓮城：顶级防御，需要充分准备才能攻下

## 10. 追加调整：防御建筑数值修正（2026-06-22）

根据 `balance-workflow` 流程分析 `数值平衡参考手册.md` 后发现 4 个异常，本轮修正。

### 10.1 异常与修复

**异常 1：城墙防御加成 buff 对城市减伤影响过小**

根因：city_def 基础值太低（2~10），在 COEFF=20 乘法模型下 +45% 防御加成只带来 4~9% 减伤变化。

修复：提高 city_def 基础值，同时降低 city_HP。

| 城市 | 旧 HP/def | 新 HP/def | 无墙减伤 | 有墙Lv3(+45%) | 差值 |
|:---:|:---:|:---:|:---:|:---:|:---:|
| 村 | 150/2 | 100/3 | 13% | 18% | +5% |
| 镇 | 250/4 | 200/7 | 26% | 34% | +8% |
| 城 | 400/6 | 350/12 | 38% | 46% | +8% |
| 府 | 600/8 | 500/18 | 47% | 56% | +9% |
| 都 | 800/10 | 700/25 | 56% | 64% | +8% |

**异常 2：瓮城性价比远超城墙**

修复：瓮城 structure_hp 降低（Lv1: 200→120, Lv2: 400→250），HP/金 降至 0.67~0.93，低于城墙。

**异常 3：箭塔对重甲几乎无感**

修复：箭塔攻击提高（Lv1: 10→12, Lv2: 15→18, Lv3: 20→24）。

**异常 5：城墙 Lv3 升级效率低**

经讨论，阶梯升级建筑效率递减属正常设计（"锦上添花"成本更高），不修正。

## 19. 总结

阶段7战斗系统数值重构涉及以下核心调整：

1. **战斗公式**：减法模型 → 乘法模型（COEFF=20）
2. **攻城系统**：城市HP/防御值重制，防御建筑体系整理
3. **关隘战斗**：地形修正简化，HP调整
4. **大夫系统**：数值缩放50%
5. **士气系统**：极端因素调整
6. **兵种造价**：金币+40-50%，粮食+100-200%，限制快速扩军
7. **特殊资源**：特产城+7座，建筑产出提升
8. **维护费**：低级兵种降低，高级兵种提升，平衡维护/造价比
9. **特色兵种**：七国特色兵种数值完善

**设计原则**：
- 性价比天然不平衡，但不可相差太大让某些兵种具有无脑优势
- 低级兵种性价比高，高级兵种性价比低，通过征兵人口限制等因素平衡
- 不希望玩家进行快速的大规模扩军
