# 《山河策》文档总入口

> 最后更新：2026-06-19
> 用途：作为 `docs/` 目录统一入口，区分正式文档、协作文档与归档材料，降低查找成本。

---

## 1. 先看什么

按使用场景，建议从这里进入：

| 场景 | 入口文件 | 说明 |
|---|---|---|
| 快速了解代码结构 | `docs/CODEMAP.md` | 代码、数据、场景、测试与主要依赖关系索引 |
| 快速了解机制全貌 | `docs/机制概览/游戏机制总览.md` | 当前机制设计总索引 |
| 查看当前程序落地情况 | `docs/程序进度/程序已实现功能.md` | 以现有代码为准的实现状态 |
| 查看协作规范 | `docs/协作/协作入口.md` | 程序 / 策划 / 美术协作入口 |
| 查看阶段性策划结论 | `docs/策划决策/` | 各阶段决策记录 |
| 查看美术交付记录 | `docs/美术进度/` | 已交付素材与程序接入说明 |

## 4. 当前建议

后续维护建议遵守这几个规则：

1. 正式文档优先放入 `机制概览 / 策划决策 / 程序进度 / 协作 / 美术进度`
2. 一次性任务提示、AI 过程稿不要再直接放进 `docs/程序进度/`
3. 新文档命名尽量避免 `文件1/文件2/文件3` 这类无语义命名
4. 若已有中文正式版，英文草稿优先放入 `docs/归档/英文协作参考/`
5. 目录结构调整后，同步更新 `docs/CODEMAP.md` 与本文件
