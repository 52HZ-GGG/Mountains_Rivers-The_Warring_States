## Phase 1：经济循环（~150 行新代码，~10 个测试）

**依赖：** 无新增依赖，修改 `city_manager.gd` + `game_manager.gd`

### 1.1 建筑队列递减

**入口：** `game_manager.gd` 的 `_process_turn_economy()` 或等效回合推进函数

**逻辑：**
```gdscript
# city_manager.gd 新增
func tick_build_queue(city_id: String) -> void:
    var city := get_city_state(city_id)
    if city.is_empty():
        return
    var queue: Array = city.get("build_queue", [])
    var completed: Array = []
    for i in range(queue.size() - 1, -1, -1):
        queue[i]["turns_remaining"] = int(queue[i]["turns_remaining"]) - 1
        if int(queue[i]["turns_remaining"]) <= 0:
            completed.append(i)
    # 先完成的先处理（倒序遍历已保证索引安全）
    for idx in completed:
        var entry := queue[idx] as Dictionary
        _apply_building(city_id, entry)
        queue.remove_at(idx)
    _cities[city_id] = city
```

**`_apply_building` 逻辑：** 将建筑加入 `city.buildings` 数组，emit `SignalBus.building_completed`。

**测试点：**
- 队列有1项，tick后 turns_remaining 减1
- turns_remaining 到0时自动完成，移到 buildings
- 空队列 tick 不报错
- 多项队列同时完成

### 1.2 城市产出计算

**公式来源：** `docs/数值公式.md §经济公式`

```
城市产出 = 人口基础 + 建筑加成 + 特产加成

粮食产出 = pop × food_per_pop × (1 + Σ farm_bonus × level) × season_mod × special_mod
金币产出 = pop × gold_per_pop × (1 + Σ market_bonus × level) × tax_rate × special_mod
木材产出 = Σ building.wood_production × level
工匠产出 = Σ building.craftsmen_production × level
建材产出 = Σ building.building_materials_production × level
```

**关键参数（`balance_params.json`）：**
- `economy.food_per_pop` = 3
- `economy.gold_per_pop` = 10
- `economy.food_consumed_per_pop` = 1

**实现位置：** `city_manager.gd` 新增 `get_city_production(city_id) -> Dictionary`

**测试点：**
- 空城产出为0
- 有farm时粮食+10%/级
- 有market时金币+10%/级
- 特产木材加成×1.3
- 季节修正生效

### 1.3 人口增长

**公式来源：** `docs/数值公式.md §人口公式`

```
增长周期 = base_growth_period × (base_food_ratio / actual_food_ratio)
         × season_mod × stability_mod

实际粮食比 = 粮食产出 / (人口 × food_per_pop)
base_food_ratio = 1.0（平衡状态）
```

**关键参数：**
- `population.growth_base_period` = 6.67 回合
- `population.food_per_pop` = 3
- `population.min_growth_period` = 2
- `population.max_growth_period` = 20

**实现：** 进度条制——`city.population_growth_progress += 1/period`，满1则 pop += 1，进度归零。

**测试点：**
- 粮食平衡（ratio=1）时 6.67 回合增长
- 粮食充裕加速（ratio=2 时约3.33回合）
- 粮食不足减速（ratio=0.5 时约13.3回合）
- 达到人口上限停止增长
- 进度条跨回合累积

### 1.4 饥荒检测

**逻辑：** 粮食产出 < 人口消耗 → 饥荒
```
if food_production < pop × food_consumed_per_pop:
    # 饥荒：每回合减少人口，数量 = floor(deficit / food_per_pop)
    var deficit := food_production - pop * food_consumed_per_pop  # 负值
    var pop_loss := int(floor(-deficit / food_per_pop))
    city.current_population = max(1, city.current_population - pop_loss)
```

**测试点：**
- 粮食刚好够 → 不减口
- 粮食缺口 3 → 减1口
- 粮食缺口 9 → 减3口
- 不会减到0以下（最低1）

## Phase 2：民心安定度 + 税收（~120 行，~12 个测试）

**依赖：** Phase 1 完成（需要产出数据）

### 2.1 税率系统

**5档税率（策划案）：**

| 档位 | 税率 | 金币乘数 | 民心影响 |
|------|------|----------|----------|
| 轻税 | 10% | ×0.33 | +2/回合 |
| 低税 | 20% | ×0.67 | +1/回合 |
| 标准 | 30% | ×1.0  | 0 |
| 重税 | 40% | ×1.33 | -1/回合 |
| 苛税 | 50% | ×1.67 | -2/回合 |

**实现：** `game_manager.gd` 新增 `_tax_rate: int = 30`，提供 `set_tax_rate(rate)` / `get_tax_rate()`。税收乘数在产出计算时应用。

**测试点：**
- 默认30%税率
- 设置5档税率生效
- 超出范围拒绝
- 税率影响金币产出

### 2.2 国家粮仓

**逻辑：** 全国粮食净产出汇总 → 存入粮仓 → 维持军队消耗

```
national_grain_pool += Σ(每个城市粮食产出 - 该城市人口消耗)
national_grain_pool -= 军队粮食消耗
```

**实现：** `game_manager.gd` 新增 `_national_grain_pool: int = 0`。

### 2.3 民心计算

**公式来源：** `docs/数值公式.md §民心公式`

```
民心 = base_morale + tax_modifier + stability + event_modifier + building_bonus
民心 ∈ [0, 100]
```

**安定度（stability）计算：**
```
stability = f(城池发展度, 驻军数量, 建筑民心加成, 在位回合数)
安定度 ∈ [0, 100]
```

**民心阈值效果（策划案）：**

| 民心区间 | 效果 |
|----------|------|
| 80-100 | 产出+10%，征兵速度+20% |
| 60-79  | 正常 |
| 40-59  | 产出-10%，逃兵+5% |
| 20-39  | 产出-25%，暴动概率 |
| 0-19   | 叛乱，城池可能脱离 |

**测试点：**
- 标准税率民心不变
- 苛税每回合-2民心
- 轻税每回合+2民心
- 民心0触发叛乱检测
- 民心100封顶
- 阈值效果正确触发

## Phase 3：征兵 + 军队维护（~100 行，~6 个测试）

**依赖：** Phase 1（人口数据）、Phase 2（民心数据）

### 3.1 征兵池

**公式来源：** `docs/数值公式.md §征兵公式`

```
征兵池上限 = 城市人口 × conscription_rate（默认0.2）
每回合补充 = 征兵池上限 × fill_rate（默认0.1）
实际征兵 = min(征兵池当前, 请求量)
征兵后人口减少 = 征兵量 × service_penalty（默认0.5）
```

**关键参数（`balance_params.json`）：**
- `conscription.max_rate` = 0.2
- `conscription.fill_rate` = 0.1
- `conscription.service_penalty` = 0.5

**实现：** `city_manager.gd` 新增征兵池字段 + `conscribe(city_id, amount) -> int`。

### 3.2 军队维护费

**公式：**
```
每回合粮食消耗 = Σ(单位数量 × unit.upkeep_food)
每回合金币消耗 = Σ(单位数量 × unit.upkeep_gold)
```

**关键数据（`units.json`）：**
- 步兵：upkeep_food=5, upkeep_gold=5
- 弓兵：upkeep_food=4, upkeep_gold=5
- 骑兵：upkeep_food=8, upkeep_gold=5
- 攻城器械：upkeep_food=12, upkeep_gold=5

**测试点：**
- 征兵池正确填充
- 征兵消耗人口
- 征兵池不足时只征到上限
- 维护费正确计算

## Phase 4：城市战斗闭环（~150 行，~10 个测试）

**依赖：** Phase 3（需要军队数据）

### 4.1 城池 HP 系统

**公式来源：** `docs/数值公式.md §城池战斗`

```
城池HP = base_hp × city_level × (1 + 防御建筑加成)
城池攻击 = base_attack × city_level × (1 + 军事建筑加成)
```

**关键参数（`balance_params.json → city_levels`）：**
- Level 1: hp=100, attack=10
- Level 2: hp=200, attack=20
- Level 3: hp=400, attack=40
- Level 4: hp=700, attack=70
- Level 5: hp=1000, attack=100

### 4.2 攻城战斗解析

**流程：**
1. 进攻方军队到达目标城池
2. 计算攻城伤害 = 进攻方攻击力 vs 城池防御
3. 城池反击伤害
4. 城池HP降到0 → 占领
5. 占领后城池归属变更，HP恢复50%

**调用现有系统：** `combat_resolver.gd` 已有战斗公式，扩展支持城池战。

### 4.3 驻军系统

**逻辑：** 单位驻扎在城池内，城池被攻击时参与防守。

**测试点：**
- 城池HP正确计算
- 攻城伤害公式
- 城池HP归零占领
- 占领后归属变更
- 驻军参与防守
- 攻城失败撤退

```
Phase 1（经济循环） → Phase 2（民心税收） → Phase 3（征兵维护） → Phase 4（城池战斗）
```
