## 一、核心战斗系统

### 1.1 战斗伤害结算器 `combat_resolver.gd`（225 行）

**文件**：`scripts/systems/combat_resolver.gd`

纯函数模块，所有状态通过参数传入，无副作用。公式流程：攻防加法层 → 克制乘算 → 扣减防御 → 随机波动 → 崩溃态乘算。

| 已实现函数 | 说明 |
|---|---|
| `_calc_atk_buff(ctx)` | 攻击加法层：地形 / 大臣勇武 / 势力 / 士气 / 伏击 / 火攻 / 学派 / 科技 / 兵种技能 |
| `_calc_def_buff(ctx)` | 防御加法层：地形 / 大臣策略 / 势力 / 学派 / 科技 / 建筑 |
| `_get_morale_atk_offset(morale, params)` | 士气攻击偏移：高士气(>=130) +10%，低士气(<50) -10%，崩溃(<20) 归零 |
| `compute_counter_multiplier(atk, def)` | 克制乘算：anti_cavalry 对骑兵强制 1.2，否则查 DataManager 矩阵 |
| `morale_defense_multiplier(morale)` | 崩溃态防御修正：士气 < 阈值时防御 x0.5 |
| `is_ranged_unit(type_id)` | 远程判断：special="ranged" 或 ranged_attack_mode="no_counter" |
| `compute_ranged_spread(lo, hi, terrain, is_ranged)` | 远程精度修正：地形影响命中散布 |
| `should_trigger_counter(def, is_ranged)` | 反击触发：远程攻击不触发反击 |
| `get_momentum_bonus(tiles_moved)` | 动能攻击：每格 +5%，上限 25% |
| `compute_damage(...)` | **完整伤害计算**：攻防加法 -> 崩溃修正 -> 克制 -> 扣防(保底1) -> 随机波动 -> 伏击判定 |
| `compute_counter_attack(...)` | 反击伤害：复用 compute_damage 逻辑 |

**文件**：`scripts/autoload/tactical_skirmish_manager.gd`
