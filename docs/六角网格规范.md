# 《山河策》六角网格坐标系规范

> 阶段0定稿。本文档定义六角网格的坐标系、距离算法和城市辖区规则，供程序实现参考。

---

## 1. 坐标系选择

采用**轴坐标（Axial Coordinates）**，使用 `(q, r)` 两个轴，**flat-top（正南正北）** 朝向——平边朝正南、正北，尖端朝东、西：

```
          N
     _____●_____
    /     ●     \
   /             \
NNW               NNE
  \               /
   \             /
    \_____●_____/
          S
   SSW         SSE
```

- **q 轴**：正东方向，穿过六角形的左右两个尖端（顶点）
- **r 轴**：北北东（NNE，60°）方向，穿过六角形的上下两条平边中心
- **隐含 s 轴**：`s = -q - r`（北北西 NNW，-60°，用于距离计算）

坐标原点 `(0, 0)` 在地图左上角。

## 2. 六个邻居方向

| 方向 | 名称 | q 偏移 | r 偏移 |
|:---:|:---:|:---:|:---:|
| 0 | 北（N） | +1 | 0 |
| 1 | 北北东（NNE） | 0 | +1 |
| 2 | 南南东（SSE） | -1 | +1 |
| 3 | 南（S） | -1 | 0 |
| 4 | 南南西（SSW） | 0 | -1 |
| 5 | 北北西（NNW） | +1 | -1 |

GDScript 实现参考：
```gdscript
const HEX_DIRECTIONS: Array[Vector2i] = [
    Vector2i(1, 0),   # 北
    Vector2i(0, 1),   # 北北东
    Vector2i(-1, 1),  # 南南东
    Vector2i(-1, 0),  # 南
    Vector2i(0, -1),  # 南南西
    Vector2i(1, -1),  # 北北西
]

func get_neighbors(q: int, r: int) -> Array[Vector2i]:
    var result: Array[Vector2i] = []
    for dir in HEX_DIRECTIONS:
        result.append(Vector2i(q + dir.x, r + dir.y))
    return result
```

## 3. 距离公式

六角网格上的曼哈顿距离（也称"六角距离"）：

```
distance(q1, r1, q2, r2) = (|q1 - q2| + |q1 + r1 - q2 - r2| + |r1 - r2|) / 2
```

等价于使用立方坐标 `(q, r, s)` 的曼哈顿距离除以 2：

```
s1 = -q1 - r1
s2 = -q2 - r2
distance = max(|q1-q2|, |r1-r2|, |s1-s2|)
```

GDScript 实现参考：
```gdscript
func hex_distance(q1: int, r1: int, q2: int, r2: int) -> int:
    var dq: int = absi(q1 - q2)
    var dr: int = absi(r1 - r2)
    var ds: int = absi((q1 + r1) - (q2 + r2))
    return maxi(dq, maxi(dr, ds))
```

## 4. 城市辖区定义

城市辖区以城市坐标为中心，**六角距离 ≤ `jurisdiction_radius`** 的所有格子属于该城辖区。

默认辖区半径：**1 格**（来自 `cities.json` 的 `jurisdiction_radius` 字段）。

辖区内的格子数量公式：`1 + 3 * radius * (radius + 1)`，半径1格 = 7格。

GDScript 实现参考：
```gdscript
func get_jurisdiction_tiles(city_q: int, city_r: int, radius: int) -> Array[Vector2i]:
    var tiles: Array[Vector2i] = []
    for q in range(city_q - radius, city_q + radius + 1):
        for r in range(city_r - radius, city_r + radius + 1):
            var s: int = -q - r
            var city_s: int = -city_q - city_r
            if absi(q - city_q) + absi(r - city_r) + absi(s - city_s) <= radius * 2:
                tiles.append(Vector2i(q, r))
    return tiles
```

---

## 5. 地图边界

- 地图尺寸：**30 列 × 20 行**（来自 `cities.json` 的 `map_width` / `map_height`）
- 有效坐标范围：`q ∈ [0, 29]`, `r ∈ [0, 19]`
- 坐标越界的格子视为不可通行

## 6. A* 寻路权重

基于地形移动力消耗的寻路权重：

```gdscript
func get_move_cost(q: int, r: int) -> int:
    var terrain_id: String = get_terrain_at(q, r)
    var terrain: Dictionary = DataManager.get_terrain(terrain_id)
    return terrain["move_cost"]  # -1 表示不可通行
```

寻路时使用 Godot 内置的 `AStarGrid2D`，设置 `cell_shape` 为 `CELL_SHAPE_HEXAGON`。

## 7. TileMapLayer 分层

| 层级 | 名称 | 用途 |
|:---:|:---|:---|
| 0 | terrain_base | 地形底图（平原/森林/山地/河流） |
| 1 | terrain_overlay | 关隘/渡口/栈道 /箭楼标记 |
| 2 | units | 单位图标 |
| 3 | culture | 文化覆盖染色层（半透明） |
| 4 | fog | 战争迷雾 |

## 8. 坐标与像素转换

Godot TileMap 自动处理六角网格的坐标到像素转换。偏移量由 TileSet 的 `tile_size` 决定：

- 建议图块尺寸：32×32 像素（与美术管线一致）
- 六角图块的渲染偏移由 Godot TileMap 的 `hexagonal_flat` 模式自动计算
