## 关键决策

### 决策 16：战斗伤害公式采用「攻击减防御」模型

**选定方案**：`net_dmg = max(base_dmg - defense, 1)`
**理由**：
- 保底伤害为1，避免高防单位完全免疫
- 将军武力每点 +0.3% 攻击，统率每点 +0.1% 防御，形成攻防博弈
- 民心修正采用线性插值（0民心=70%战力，100民心=100%战力）
**已落地**：`docs/数值公式.md` §1

---

### 决策 17：兵种相克采用战国克制表（含弩兵强化机制）

**选定方案**：7×7 克制矩阵，基于战国历史兵种特点
**核心克制循环**：步兵 → 弓/弩 → 骑兵 → 战车 → 步兵
**关键设计**：
- 弩兵是弓兵强化版：主动攻击时克制系数比弓兵高 0.1（对骑兵 1.4 vs 1.3，对战车 1.3 vs 1.2）
- 弩兵被攻击时与弓兵相同（步兵克弩/弓均为 1.2）
- 骑兵克步兵（1.3）、弓/弩克骑兵（1.3/1.4）
- 战车克步兵（1.15）、骑兵克战车（1.25）
- 攻城器械为功能性单位，对城墙伤害 ×3.0，不参与常规克制循环
- 水军为独立单位，主要在水域地形生效
**已落地**：`docs/数值公式.md` §1.4、`data/units.json` 兵种属性

---

### 决策 18：资源产出公式采用「基础×修正」乘法模型

**选定方案**：`production = base * building_bonus * season_mod * special_mod * faction_mod`
**理由**：
- 乘法模型便于策划调整单一维度而不影响其他
- 季节修正直接作用于产出，直观易懂
- 特产加成（产铁城 +30%、盐池 +20%、产马城骑兵训练 +30%）作为独立乘数
**已落地**：`docs/数值公式.md` §2、`data/cities.json` 特产字段

---

### 决策 19：民心变化采用「基础漂移+多因素叠加」模型

**选定方案**：`morale_change = base_drift + tax_effect + building + event + season + faction`
**关键参数**：
- 税率30%为基准（民心不增不减），每高1%民心-0.5
- 民心<30时每回合15%概率触发民变
- 儒家民心上限+20（即最高120）
**已落地**：`docs/数值公式.md` §3

---

### 决策 20：文化扩散采用「邻格渗透+距离衰减」模型

**选定方案**：`culture_spread = source_culture * 0.05 * distance_decay`
**衰减系数**：邻格 1.0、2格 0.5、3格 0.25
**文化胜利条件**：70%城市覆盖+维持10回合
**已落地**：`docs/数值公式.md` §4

---

### 决策 21：六角网格采用轴坐标 (q, r) + 立方距离算法

**选定方案**：
- 坐标系：轴坐标 (q, r)，隐含 s = -q - r
- 距离公式：`max(|dq|, |dr|, |ds|)`
- 6个邻居方向：东(+1,0)、东南(0,+1)、西南(-1,+1)、西(-1,0)、西北(0,-1)、东北(+1,-1)
- 城市辖区：六角距离 ≤ jurisdiction_radius（默认3格，共19格）
**已落地**：`docs/六角网格规范.md`

---

### 决策 22：城市数据包含 14 座城（七国各 2 城）

**选定方案**：每国 1 首都 + 1 附属城，阶段4扩展更多城市
**城市分布**（按历史地理）：
- 秦：咸阳（首都，产铁）、雍城
- 赵：邯郸（首都，产马）、代郡（产马）
- 齐：临淄（首都，盐池）、即墨
- 楚：郢（首都）、寿春
- 魏：大梁（首都）、安邑（产铁）
- 燕：蓟（首都）、辽阳（产马）
- 韩：新郑（首都）、上党（产铁）
**已落地**：`data/cities.json`

---

### 决策 23：DataManager 采用「启动加载+内存缓存+只读访问」模式

**选定方案**：
- `_ready()` 时加载所有 JSON 到 Dictionary
- 所有 getter 方法返回 Dictionary/Array（值类型，非引用）
- `get_faction_variant()` 自动合并基础属性和 stat_overrides
- 预留阶段1~3的数据加载接口（tech_tree、events、buildings、factions、diplomacy）
**已落地**：`docs/接口文档.md`

---

### 决策 24：阶段0只实现核心三表，其他数据表留接口占位

**范围**：
- ✅ 阶段0实现：`terrain.json`、`units.json`、`cities.json`
- ⏳ 阶段1+扩展：`tech_tree.json`、`events.json`、`buildings.json`、`factions.json`、`diplomacy.json`
**理由**：阶段0目标是验证核心循环，科技/事件/外交数据在对应阶段填充
**已落地**：`docs/接口文档.md` 预留注释
