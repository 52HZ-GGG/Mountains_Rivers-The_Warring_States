## 关键决策

### 决策 45：外交系统范围为全部7种外交动作

**选定方案**：
- 阶段2一次性实现全部7种外交动作：宣战、停战、互不侵犯、军事通行权、商路、赠礼、结盟
- 不做子集裁剪，全部落地

**理由**：
- 7种动作相互关联（如盟约影响宣战、通行权影响军事），拆分会导致系统不完整
- 数据驱动架构保证新增动作只需扩展JSON配置，代码量可控
- 阶段2交付目标是"可与AI进行基本政治交易"，7种动作是最低要求
**已落地**：`scripts/autoload/diplomacy_system.gd`

---

### 决策 46：好感度初始值基于历史关系矩阵

**选定方案**：
- 好感度范围 -100 ~ +100，初始值通过 `factions.json` 的 `initial_relations` 矩阵配置
- 秦楚(-20)、秦赵(-15)、齐燕(-10)等基于战国历史恩怨
- 接壤国家额外 -10（固定值）
- 双向独立：A对B和B对A的好感度独立计算

**理由**：
- 统一初始0会让所有国家开局关系无差异，缺乏历史代入感
- 历史关系矩阵让玩家开局就能感知到合纵连横的背景
- 双向独立支持不对称关系（如秦恨楚，楚可能不那么恨秦）
**已落地**：`data/factions.json` initial_relations、`scripts/autoload/diplomacy_system.gd`

---

### 决策 47：国际声望采用五档模糊指示器

**选定方案**：
- 声望范围 0~100，初始值 50
- UI显示五档模糊等级：极低(<20)、低(20-39)、中(40-60)、高(61-80)、极高(>80)
- 不显示精确数值，通过事件文本暗示

**理由**：
- 完全隐藏声望会让玩家无法做出有效外交决策
- 直接显示精确数值会削弱代入感，变成纯数值游戏
- 五档模糊指示器是折中方案：给玩家方向感，但保留不确定性
**已落地**：`scripts/autoload/diplomacy_system.gd` get_reputation_level()

---

### 决策 48：AI性格采用四维独立参数（1-5档）

**选定方案**：
- 四个维度：好战度、贪婪度、诚信度、外交度
- 每个维度 1-5 档，在 `factions.json` 中配置
- 七国分配：秦(4,3,2,1)、赵(4,2,3,2)、齐(1,2,4,4)、楚(2,4,2,3)、魏(3,3,2,2)、燕(3,1,4,2)、韩(1,2,3,4)

**理由**：
- 四维参数比单一"性格标签"更灵活，支持细粒度行为差异
- 1-5档比0-100更直观，策划调整方便
- 独立参数（非分类）允许组合出更多样的AI行为
**已落地**：`data/factions.json` ai_personality

---

### 决策 49：AI决策概率由好战度决定（好战度×15%）

**选定方案**：
- AI每回合有概率触发外交决策评估
- 概率 = 好战度 × 15%（好战度4 = 60%，好战度1 = 15%）
- 不是每回合都评估，避免AI过于活跃

**理由**：
- 每回合评估会让AI外交动作过于频繁，玩家应接不暇
- 好战度高的国家（如秦、赵）更频繁地考虑外交/军事行动
- 好战度低的国家（如齐、韩）更被动，符合历史形象
**已落地**：`scripts/ai/diplomacy_ai.gd` _should_evaluate()

---

### 决策 50：AI宣战目标采用综合评分公式

**选定方案**：
- 目标评分 = 距离分×0.3 + 国力分×0.3 + 好感度分×0.4
- 距离分：两城六角格距离越近越高（归一化到0~1）
- 国力分：目标越弱越高（基于国力评分公式）
- 好感度分：好感度越低越高
- 好感度>50时不主动宣战（好感度保护）

**理由**：
- 远交近攻是战国核心外交策略，距离权重体现这一思想
- 欺软怕硬是AI的基本行为逻辑，国力分确保AI不攻击过强目标
- 好感度权重最高(0.4)确保关系差的国家优先被攻击
- 好感度保护防止AI背刺盟友，增加外交可信度
**已落地**：`scripts/ai/diplomacy_ai.gd` _calculate_target_score()

---

### 决策 51：均势反应通过国力评分触发

**选定方案**：
- 国力评分 = 城市数×50 + 军队数量×3 + 科技数×15 + 民心×1 + 文化值×1
- 当玩家国力 > 所有AI平均国力 × 1.5 时触发联盟
- 好感度<0的AI尝试与其他AI结盟对抗玩家

**理由**：
- 城市数是最直观的国力指标，权重最高(50)
- 军队数量反映军事实力，但权重低于城市（军队可快速重建）
- 1.5倍阈值避免玩家刚占1座城就被围攻
- 只有好感度<0的AI才会加入联盟，避免无理由围攻
**已落地**：`scripts/autoload/diplomacy_system.gd` get_power_score()、`scripts/ai/diplomacy_ai.gd` _check_coalition_trigger()

### 决策 52：停战谈判采用分数系统（最多3轮）

**选定方案**：
- 谈判条件有分数：赔款=金额×0.01、割地=每座城2分、称臣=固定3分
- AI接受阈值 = 基础3分 + 好战度×0.3 + 贪婪度×0.2
- 战胜方先开价，最多3轮谈判
- 还价在临界点上加价

**理由**：
- 分数系统比简单阈值更灵活，支持多种条件组合
- 性格影响阈值：好战/贪婪的AI更难说服，符合直觉
- 3轮限制避免无限谈判拖慢节奏
- 战胜方先开价体现战争胜利的优势
**已落地**：`scripts/ai/diplomacy_ai.gd` _calculate_negotiation_score()、`scenes/ui/diplomacy/negotiation_dialog.gd`

---

### 决策 53：赠礼采用三档制

**选定方案**：
- 小礼：50金 → 好感度+10
- 中礼：150金 → 好感度+20
- 大礼：300金 → 好感度+30
- 三档在 `diplomacy.json` 中配置

**理由**：
- 固定档位比线性公式更直观，玩家容易理解
- 三档提供足够的选择空间（省钱vs速成）
- 大礼300金是显著投入，体现"诚意"
**已落地**：`data/diplomacy.json` gift_tiers

---

### 决策 54：条约可选持续3/5/10回合

**选定方案**：
- 互不侵犯条约：可选3/5/10回合
- 停战协定：可选3/5/10回合
- 盟约：无限期直到主动解约
- 开战后5回合内不可停战（针对同一对国家）

**理由**：
- 可选持续时间让玩家根据战略需求灵活选择
- 短约(3回合)适合临时停战，长约(10回合)适合长期和平
- 盟约无限期体现"同盟"的严肃性
- 5回合开战冷却防止"宣战-立即停战"的 exploits
**已落地**：`data/diplomacy.json` treaty_params

---

### 决策 55：军事通行权按次付费+数量限制

**选定方案**：
- 通行权费用 = 每支部队10金 × 部队数量上限
- 购买时指定允许的部队数量上限（最多5支部队）
- 通行权持续5回合
- 附庸国自动享有宗主国通行权

**理由**：
- 按次付费比按回合付费更灵活，玩家可按需购买
- 数量限制防止无限过境，保持战略平衡
- 5回合持续时间与条约系统对齐
**已落地**：`data/diplomacy.json` treaty_params、`scripts/autoload/diplomacy_system.gd`

---

### 决策 56：背盟/违约惩罚为固定值+debuff

**选定方案**：
- 声望-30、对目标好感-50、所有国家好感-15、民心-10
- 额外debuff：部队攻击力×0.9，持续3回合
- 三种盟约破裂方式：主动解约、被攻击不参战、好感度降到阈值以下

**理由**：
- 固定惩罚比动态惩罚更可预测，玩家能提前评估违约代价
- 攻击力debuff体现"背信弃义"的军事代价
- 三种破裂方式覆盖主要违约场景
**已落地**：`data/diplomacy.json` betrayal_effects、`scripts/autoload/diplomacy_system.gd`

---

### 决策 57：附庸国机制包含进贡+外交从属+三种脱离方式

**选定方案**：
- 附庸国每回合进贡15%资源（粮食+金钱）给宗主国
- 附庸国不能独立宣战/结盟，但可被其他国家赠礼和宣战
- 对附庸国宣战=宗主国不参战则声望-20、民心-15（重度惩罚）
- 三种脱离方式：第三方拉拢、声望脱离(宗主声望<30)、趁乱脱离(宗主在战争中)

**理由**：
- 15%进贡比例体现附庸关系的经济代价
- 外交从属但可被赠礼，为"拉拢脱离"创造条件
- 宗主国不保护附庸有重度惩罚，鼓励玩家信守承诺
- 三种脱离方式给附庸国多种摆脱路径，避免"永久附庸"
**已落地**：`data/diplomacy.json` vassal、`scripts/autoload/diplomacy_system.gd`

---

### 决策 58：商路基础收入20金+声望修正

**选定方案**：
- 商路基础收入：每回合20金（双方各得）
- 声望>70时收入+10%
- 保留计算公式接口，后续可扩展动态计算

**理由**：
- 固定收入简化实现，阶段2不做复杂的贸易距离计算
- 声望修正体现"信誉好=贸易好"的逻辑
- 预留接口方便阶段3扩展为基于城市数和距离的动态计算
**已落地**：`data/diplomacy.json` trade_route、`scripts/autoload/diplomacy_system.gd`

---

### 决策 59：中立城市可通过赠礼归附（附庸关系）

**选定方案**：
- 中立城好感度>80时可请求归附（花费200金）
- 归附后中立城成为附庸国，不是直接吞并
- 其他国家好感度-20（对归附方）
- 附庸国可被其他国家赠礼和宣战

**理由**：
- 附庸关系比直接吞并更有策略深度，需要持续维护
- 其他国家好感度下降体现"扩张引发恐惧"
- 可被赠礼为"拉拢脱离"创造条件
**已落地**：`data/diplomacy.json` neutral_city、`scripts/autoload/diplomacy_system.gd`

### 决策 60：关隘开放采用独立分数系统

**选定方案**：
- 关隘分数 = 好感度×0.4 + 金钱/100×0.3 + 声望/100×0.3
- 分数≥60时可开放关隘
- 与停战谈判分数系统独立

**理由**：
- 关隘开放是和平时期的贸易行为，与战争谈判逻辑不同
- 好感度权重最高(0.4)体现关系的重要性
- 60分阈值要求中等以上关系才能开放
**已落地**：`data/diplomacy.json` gate_open_score_formula、`scripts/autoload/diplomacy_system.gd`

---

### 决策 61：好感度每5回合衰减1点向0回归

**选定方案**：
- 每5回合检查一次，好感度向0回归1点
- 正好感度-1，负好感度+1
- 声望不衰减

**理由**：
- 好感度衰减防止"一次赠礼永久盟友"，鼓励持续外交投入
- 5回合间隔避免衰减过快，给玩家足够反应时间
- 声望不衰减体现"名声一旦建立就难以改变"
**已落地**：`data/diplomacy.json` opinion_decay、`scripts/autoload/diplomacy_system.gd`

---

### 决策 62：AI资源采用完整追踪+难度修正

**选定方案**：
- AI国家和玩家一样有完整的资源系统（粮食/金钱/铁矿/民心/人口/军队）
- AI建筑系统与玩家相同，性格驱动建设方向
- 难度修正：简单(-30%)、普通(0%)、困难(+30%)、地狱(+60%)
- 高难度额外初始金钱：困难+100金、地狱+300金

**理由**：
- 完整资源追踪让AI行为更真实，不出现"AI无限出兵"的情况
- 性格驱动建设让好战型AI优先兵营，保守型AI优先农田
- 难度修正让游戏可调节，满足不同玩家需求
**已落地**：`data/diplomacy.json` difficulty、`scripts/autoload/game_manager.gd`

---

### 决策 63：声望四个阈值全部实现

**选定方案**：
- 极低声望(<20)：触发反玩家联盟事件
- 低声望(<30)：商路收入-50%
- 高声望(>70)：商路收入+10%
- 极高声望(>80)：结盟成功率+30%

**理由**：
- 四个阈值覆盖声望系统的完整影响链
- 极低声望的联盟惩罚是最强的声望惩罚机制
- 高声望的贸易奖励鼓励玩家维护声誉
**已落地**：`data/diplomacy.json` reputation_thresholds/reputation_effects

**选定方案**：

## 新增 Schema 定义

### factions.json

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| id | String | 国家唯一ID |
| name | String | 中文名 |
| color | String | UI颜色（十六进制） |
| description | String | 描述 |
| ai_personality | Dictionary | AI性格：aggression/greed/honesty/diplomacy（1-5档） |
| default_school | String/null | 默认学派（阶段3用） |
| diplomacy_modifier | Dictionary | 外交修正（预留） |

### diplomacy.json

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| gift_tiers | Array | 赠礼档位：name/cost_gold/opinion_change |
| action_effects | Dictionary | 各外交动作的声望和好感度变化 |
| betrayal_effects | Dictionary | 违约惩罚参数 |
| treaty_params | Dictionary | 条约参数：持续时间、战争冷却、通行权费用 |
| trade_route | Dictionary | 商路参数：基础收入、声望阈值 |
| vassal | Dictionary | 附庸参数：进贡比例、脱离方式和惩罚 |
| opinion_decay | Dictionary | 好感度衰减：间隔回合数、衰减量 |
| gate_open_score_formula | Dictionary | 关隘开放分数公式权重 |
| negotiation | Dictionary | 谈判参数：最大轮次、各条件分数、接受阈值 |
| ceasefire_conditions | Dictionary | 停战条件：最小战争回合、AI停战阈值 |
| neutral_city | Dictionary | 中立城参数：归附阈值、费用、好感度影响 |
| power_score | Dictionary | 国力评分各维度权重 |
| ai_decision | Dictionary | AI决策参数：评估概率、目标选择权重 |
| difficulty | Dictionary | 难度设置：资源修正、初始金钱加成 |
| reputation_thresholds | Dictionary | 声望阈值：very_low/low/high/very_high |
| reputation_effects | Dictionary | 各声望等级的效果 |

### balance_params.json 新增 diplomacy 节

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| opinion_min | int | 好感度下限 (-100) |
| opinion_max | int | 好感度上限 (100) |
| initial_opinion | int | 默认初始好感度 (0) |
| reputation_min | int | 声望下限 (0) |
| reputation_max | int | 声望上限 (100) |
| initial_reputation | int | 初始声望 (50) |
| border_distance_threshold | int | 接壤距离阈值 (4) |

## 程序接口约定

DiplomacySystem 提供以下公共接口：

| 方法 | 说明 |
|:---|:---|
| `get_opinion(a, b) -> int` | 获取A对B的好感度 |
| `get_reputation(id) -> int` | 获取国家声望 |
| `get_reputation_level(id) -> String` | 获取声望模糊等级 |
| `are_at_war(a, b) -> bool` | 是否在战争 |
| `are_allied(a, b) -> bool` | 是否同盟 |
