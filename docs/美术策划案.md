## 四、动画制作指南

### 4.1 需要做动画的内容

| 内容 | 动画类型 | 优先级 | 帧数建议 |
|:---|:---|:---:|:---:|
| 兵种待机 | 帧动画 | P1 | 2~4 |
| 兵种移动 | 帧动画 | P1 | 4~6 |
| 兵种攻击 | 帧动画 | P1 | 4~6 |
| 兵种受击 | 帧动画 | P2 | 2~3 |
| 兵种死亡 | 帧动画 | P2 | 4~6 |
| 战斗特效 | 帧动画 | P2 | 6~12 |
| 季节切换 | 过渡动画 | P3 | 逐帧或 Tween |
| 文化光环 | Shader 动画 | P3 | 程序实现 |
| UI 面板弹出 | 补间动画 | P3 | Godot Tween |

### 4.2 Aseprite 动画工作流

1. 在 Aseprite 中创建 32×32 画布
2. 使用 Tag 命名动画（idle / move / attack / hurt / death）
3. 每个 Tag 对应一组帧
4. 导出为**水平条状精灵表**（Sprite Sheet），Godot 的 SpriteFrames 可自动导入
5. 文件命名：`unit_[faction]_[type]_[variant].png`（同一文件内含所有动画帧）

### 4.3 动画优先级建议

**阶段 1~2（现在）**：只做静态图标，确保地图上能区分兵种
**阶段 3~4**：做步兵/骑兵/弓兵 3 套基础动画模板
**阶段 5**：补充所有兵种动画 + 特效动画

## 五、素材转制工作流

现有 `photos/` 目录中的 AI 生成图是高分辨率的，需要转制为像素风格：

### 5.1 地形图块转制

1. 参考 AI 生成图的构图和色彩
2. 在 Aseprite 中新建 32×32 画布
3. 用六角形选区限定绘制区域
4. 用 32 色调色板重绘
5. 导出 PNG，确保六角形外透明

### 5.2 兵种图标转制

1. 参考 AI 生成图的造型
2. 在 Aseprite 中新建 32×32 画布
3. 绘制 16×24 左右的角色（留出上下边距）
4. 保持各兵种视觉差异明显（武器/盔甲轮廓）
5. 导出 PNG

### 5.3 UI 元素转制

1. 面板类：参考 AI 生成图，用羊皮纸/竹简纹理重绘
2. 按钮类：4 态（默认/悬停/按下/禁用）
3. 图标类：16×16 或 24×24，简洁明了

## 六、交付标准

### 6.1 文件规范

- 所有 PNG 必须符合命名规则（小写、下划线、无空格）
- 32×32 图块边缘清晰，**无半透明像素**（像素画硬边）
- 六角形图块的透明区域必须完全透明（alpha = 0）
- 文化染色层不得在 PNG 中固定颜色

### 6.2 目录规范

所有最终素材放入 `assets/` 对应子目录，不要直接用 `photos/` 中的 AI 图。

### 6.3 验收清单

- [ ] 命名符合 `tile_[type]_[variant].png` 规范
- [ ] 尺寸为 32×32 像素
- [ ] 调色板不超过 32 色
- [ ] 六角形图块外缘透明
- [ ] 在 Godot TileMap 中显示正常
- [ ] 文化覆盖层叠加后不产生视觉冲突

## 八、与程序的协作接口

### 8.1 程序需要美术提供的

| 需求 | 格式 | 放置位置 |
|:---|:---|:---|
| 地形 TileSet | PNG 精灵表 + Godot TileSet 资源 | `assets/tilesets/` |
| 兵种 SpriteFrames | PNG 精灵表 | `assets/sprites/units/` |
| UI 纹理 | 单张 PNG | `assets/ui/` |
| 字体文件 | TTF/OTF | `assets/fonts/` |
| Shader 文件 | .gdshader | `assets/shaders/` |

### 8.2 程序会处理的

- TileMap 的六角网格渲染
- 文化覆盖层的颜色注入（从 `schools.json` 读取 `theme_color`）
- UI 面板的布局和交互逻辑
- 动画播放控制（AnimationPlayer / SpriteFrames）
- Tween 补间动画
